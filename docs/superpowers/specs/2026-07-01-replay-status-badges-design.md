# Bug 复现页状态徽章设计

## 目标

增强 Bug 复现页右上角“已确认、候选簇、待最小化”三项统计的视觉层级和状态辨识度，同时保持当前紧凑布局及真实数据绑定。

## 视觉方案

- 已确认：绿色语义，表示已经验证并归档的缺陷证据。
- 候选簇：琥珀色语义，表示等待人工审核的候选问题。
- 待最小化：蓝色语义，表示需要继续处理的候选程序。
- 每项使用独立的淡色背景、细边框、左侧状态点和醒目的数字。
- 三项保持同一高度与间距，不扩大页头，不引入额外图标资源。

## 数据与行为

- 继续读取 `confirmed.length`、`candidateClusterStats.total` 和 `candidateClusterStats.needsMinimize`。
- 不新增接口、不缓存统计值、不写死数量。
- 仅调整模板类名和 CSS；页面交互及移动端隐藏规则保持不变。

## 验证

- 组件测试验证三个统计项具有独立语义样式类，并继续显示动态数量。
- 浏览器检查桌面页头没有换行、溢出或高度突增。
- 运行完整前端测试与生产构建。
