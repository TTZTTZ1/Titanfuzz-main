# TensorGuard 蓝色前端视觉重构设计

## 1. 目标与范围

将已确认的“冰川钴蓝”视觉方案应用到 TensorGuard 正式前端，覆盖总览、单 API 运行和 Bug 复现三个主页。

本次只重构展示层，不修改：

- Web API 路由及返回结构。
- DOM 元素 ID、任务状态机和轮询逻辑。
- Qwen、InCoder、Driver 和 Bug 复现子进程。
- Results 发布、候选审核和报告生成规则。
- 环境、GPU、API 数量和 Bug 数量的真实数据来源。

蓝色版成为默认正式界面。绿色版不提供运行时主题切换，只保留于 Git 历史和 `docs/design-previews/green-overview.html` 作为回退参考。

## 2. 视觉方向

### 2.1 核心配色

- 页面背景：从 `#f8fbff` 经 `#edf4ff` 到 `#f8f5ff` 的低对比度线性渐变。
- 导航外壳：偏蓝石墨色 `#131c2d`。
- 主操作与选中态：钴蓝 `#246bfd`。
- 主文字：`#172033`；次要文字：`#68758d`。
- 边框：冷蓝灰 `#dbe3f0`；表面：高透明度白色。
- 图表辅色：柔和紫 `#8269d6` 和青蓝 `#1598bd`，不用紫色作主色。
- 成功：`#168367`；警告：`#b36f16`；异常：`#c64756`。

语义状态色不被蓝色替换，确保运行中、成功、失败和待审核仍可以快速区分。

### 2.2 层级与质感

- 常规圆角统一为 6px–7px，不使用大面积胶囊形状。
- 用间距、边框和字重建立层级，阴影仅用于主面板和浮层。
- 指标区使用连续的信息条，减少独立小卡片。
- 标题和正文使用系统无衬线字体，API、Job 参数、返回码和日志使用等宽字体。
- 动画限于 150–200ms 的选中、展开和状态过渡，禁用持续发光和装饰性动画。

## 3. 页面应用

### 3.1 总览页

- 顶部保留页面标题、环境信息和“运行单 API”主操作。
- API 总数、框架分布、已确认问题和环境状态放入一个连续指标条。
- 框架覆盖、测试链路和证据表继续读取现有真实数据，只替换颜色和排版。
- 不在总览页制造虚假时间趋势或并不存在的运行指标。

### 3.2 单 API 运行页

- API 搜索、框架、GPU 和模式选择器使用统一的蓝色焦点态。
- 约束检查、Qwen、InCoder、Driver 流程保留原状态机，运行中阶段用钴蓝，成功、失败和跳过使用各自语义色。
- Qwen、InCoder 和 Driver 标签的可点击规则不变，只重绘分段控件。
- 主图表使用蓝色主线、紫色辅线和青蓝 GPU 线；数据缺口仍保持断线，不补零。
- Results 分类、实时日志、任务摘要和数据一致性警告保持现有功能。

### 3.3 Bug 复现页

- 保留左侧 Bug 列表、右侧证据详情的主从式结构。
- 选中 Bug 用浅蓝背景和蓝色边线表示，异常现象仍使用红色语义标记。
- “实际行为”与“预期行为”保持对照区，代码和输出保持高对比度终端表面。
- 复现、停止和生成报告按钮只重绘样式，不改变禁用条件、返回结果和报告不覆盖规则。

## 4. 代码边界

主要修改：

- `webapp/static/styles.css`：设计变量、全局外壳、三页布局、图表、状态和响应式样式。
- `webapp/static/index.html`：只在必要时调整语义容器和布局分组，保留所有现有 ID。
- `webapp/static/charts.js`：如有必要，仅替换图表色板和悬浮视觉，不修改数据计算。
- `webapp/static/app.js`：原则上不改逻辑；只允许为新布局增加不影响数据的 class 切换。

不修改 `webapp/server.py`、`scripts/run_one_api_demo.py`、Results 目录和已确认 Bug 数据。

## 5. 兼容与验证

- 现有 Python 后端、前端合约、图表和 Markdown 测试必须全部通过。
- 增加静态合约检查，确认原有 DOM ID 和三个主视图仍存在。
- 使用真实 Web 服务验证总览、单 API 和 Bug 复现页的导航、空状态和环境抽屉。
- 在约 1440px 桌面和 760px 窄屏下进行视觉检查，确认不重叠、不溢出、文字可读。
- 检查 `prefers-reduced-motion` 和打印报告样式，确保视觉重构不影响 PDF 报告输出。
- 本地缺少 GPU 时，界面应显示真实的“未检测到 GPU”，不使用设计稿示例数据。

## 6. 完成标准

1. 打开正式 Web 服务时默认显示蓝色版，不再显示绿色主题。
2. 三个主页具有统一蓝色视觉语言，同时保留各自功能和数据状态。
3. 页面不包含写死的 GPU、CUDA、框架版本、API 数量或 Bug 数量。
4. 所有现有自动化测试通过，三个主页的关键交互可用。
5. 绿色版可通过 Git 历史或设计预览找回，正式代码不维护双主题分支。
