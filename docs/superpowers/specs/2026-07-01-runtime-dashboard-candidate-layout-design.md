# 运行面板与候选审核布局设计

## 范围

本次改动只调整单 API 运行页和 Bug 复现页的候选审核视图。任务并发、后端子进程、已确认 Bug 复现视图和数据接口保持不变。

## 单 API 运行页

### 运行控制

- 运行模式、计算设备、候选状态和运行按钮使用相同外框高度、圆角与基线。
- 四项内容均在各自固定区域内垂直居中，状态文本过长时不改变整体宽度。

### 实时日志

- 阶段标签使用“种子生成、演化变异、差异检测”，不绑定具体模型品牌。
- 日志容器尺寸由页面网格约束，长日志不能扩大页面。
- 日志使用软换行和纵向滚动；不提供横向滚动。
- 原始日志内容不被修改，换行仅影响浏览器展示。

### 阶段图表

- 图表标题区展示当前阶段的全部序列，不再截取前两项。
- 每个序列使用带线段色标的紧凑指标徽章，同时作为图例和最新值展示。
- 使用稳定语义配色：候选/已检测为蓝色，有效为绿色，异常为琥珀色，崩溃/差异为红色，超时为灰色。
- ECharts 线条与标题区色标使用同一颜色映射，跨阶段的“有效”始终为绿色。

### 信息区比例

- 左侧图表适度缩短，右侧日志列加宽。
- GPU 监控、实时摘要和 Results 构成减少多余内边距与最小高度，但保留现有数据和交互。

## 候选审核页

- 候选代表源码与人工整理 `repro.py` 在上方等宽并排。
- 两个代码框横向更宽、纵向更矮，尺寸固定；代码支持横向和纵向滚动，不自动折行。
- 审核与收录移动到代码框下方并横跨整行。
- 长源码路径允许换行，不扩大网格。
- 已确认 Bug 页面保持现状。

## 验证

- 组件测试覆盖日志换行策略、模型无关阶段标签、完整图例、稳定颜色和候选审核布局类。
- 完整 Vitest 与生产构建通过。
- 浏览器验证当前桌面视口没有横向溢出，日志长行不会扩大页面，候选审核页结构与设计一致。
