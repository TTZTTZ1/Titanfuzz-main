`SyncBatchNorm` is a PyTorch module that extends standard batch normalization to synchronize batch statistics across multiple GPUs in distributed training. It computes mean and variance over mini-batches from all processes in a specified process group, ensuring consistent normalization. Key points:

- **Purpose**: Enables stable training in distributed settings by synchronizing batch statistics.
- **Usage**: Designed for use with `DistributedDataParallel` (single GPU per process). Convert existing `BatchNorm*D` layers using `SyncBatchNorm.convert_sync_batchnorm()` before wrapping with DDP.
- **Behavior**: Synchronization occurs only during training (`model.train()`); disabled in evaluation mode.
- **Parameters**: Includes standard batch norm args (`num_features`, `eps`, `momentum`, etc.) plus an optional `process_group` to define synchronization scope.
- **Limitation**: Currently supports only one GPU per process.
