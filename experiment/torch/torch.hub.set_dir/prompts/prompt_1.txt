### **Key Features**

1. **Publishing Models**  
   - Add a `hubconf.py` file to a GitHub repository.  
   - Define **entrypoints** (Python functions) that return models or tools (e.g., tokenizers).  
   - Specify dependencies in a `dependencies` list (e.g., `['torch']`).  
   - Include docstrings for help messages and examples.  
   - Models must be on a branch or tag (not a random commit).

2. **Loading Models**  
   - **List available models**:  
     ```python
     torch.hub.list('owner/repo')
     ```
   - **Get help for a model**:  
     ```python
     torch.hub.help('owner/repo', 'model_name')
     ```
   - **Load a model**:  
     ```python
     model = torch.hub.load('owner/repo', 'model_name', *args, **kwargs)
     ```
     Can load from GitHub (`source='github'`) or a local directory (`source='local'`).

3. **Utilities**  
   - `torch.hub.download_url_to_file()`: Download files from a URL.  
   - `torch.hub.load_state_dict_from_url()`: Load model weights from a URL.  
   - `torch.hub.get_dir()` / `set_dir()`: Manage the cache directory for downloaded models.

### **Important Notes**

- **Trust & Security**: Use `trust_repo` to control code execution from untrusted repositories.  
- **Caching**: Models are cached locally; use `force_reload=True` to refresh.  
- **Limitations**:  
  - Avoid loading two branches of the same repo in the same Python process.  
  - Import conflicts may occur if different repos use the same sub-package names.
