**PyTorch Hub** is a repository for pre-trained models that facilitates research reproducibility by allowing easy sharing and loading of models.

### **Key Functions**

- **`torch.hub.list(github)`**: Lists all available entrypoints (models) in a specified GitHub repository.
- **`torch.hub.help(github, model)`**: Displays the docstring and usage information for a specific model.
- **`torch.hub.load(repo_or_dir, model, *args, **kwargs)`**: Loads a model (or other callable) from a GitHub repo or local directory. Arguments are passed to the model's entrypoint function for instantiation.
- **`torch.hub.load_state_dict_from_url(url, ...)`**: Loads a model's state dictionary (weights) from a URL.
- **`torch.hub.download_url_to_file(url, dst)`**: Downloads a file from a URL to a local path.

### **Publishing Models to Hub**

To publish a model, add a `hubconf.py` file to your GitHub repository. This file defines **entrypoints** (Python functions) that return models or tools.

**Example `hubconf.py` entrypoint:**
```python
dependencies = ['torch']  # Required packages
from mymodel import mymodel as _mymodel

def mymodel(pretrained=False, **kwargs):
    """Model description and example usage."""
    model = _mymodel(**kwargs)
    if pretrained:
        checkpoint = 'https://.../model.pth'
        model.load_state_dict(torch.hub.load_state_dict_from_url(checkpoint))
    return model
```

**Important Notes for Publishers:**
- Models must be on a branch or tag, not a random commit.
- Use `dependencies` to list required packages.
- Pretrained weights (<2GB) can be attached to GitHub releases; larger files should use external URLs.
- Helper functions should be prefixed with `_` to hide them from `torch.hub.list()`.

### **Loading and Using Models**

1.  **Explore:** Use `torch.hub.list()` to see available models and `torch.hub.help()` for documentation.
2.  **Load:** Call `torch.hub.load()` with the repo and model name. Additional arguments (`*args`, `**kwargs`) are passed to the model's entrypoint for configuration.
3.  **Inspect:** After loading, use `dir(model)` to see available methods and `help(model.method)` for details.

### **Cache and Security**

- **Cache Location:** Downloaded models are saved in a cache directory (default: `~/.cache/torch/hub`). Use `torch.hub.set_dir()` to change this.
- **Force Reload:** Use `force_reload=True` to refresh the cache.
- **Security (`trust_repo`):** Controls trust for repositories. Options:
    - `"check"`: Verify against a trusted list (default in v2.0).
    - `False`: Prompt user for confirmation.
    - `True`: Trust and load without prompt.
    - `None`: Issues a warning (backward compatibility).

### **Limitations**

- Importing models from different repos with conflicting sub-package names (e.g., both have a `model` package) can cause errors. Workaround: manually clean `sys.modules`.
- **Cannot load two different branches of the same repo in the same Python process.**

### **Example Usage**

```python
# List models in torchvision
entrypoints = torch.hub.list("pytorch/vision")

# Get help for resnet18
print(torch.hub.help("pytorch/vision", "resnet18"))

# Load a pretrained ResNet-50
model = torch.hub.load("pytorch/vision", "resnet50", weights="ResNet50_Weights.IMAGENET1K_V1")
```

PyTorch Hub streamlines model sharing and loading, promoting reproducible research with minimal setup.
