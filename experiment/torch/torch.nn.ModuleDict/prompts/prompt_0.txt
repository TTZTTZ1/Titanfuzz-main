`ModuleDict` is a PyTorch container that stores submodules in a dictionary, ensuring they are properly registered and accessible to all `Module` methods. It behaves like an ordered dictionary, preserving insertion order and the order of merged mappings when using `update()` with ordered types like `OrderedDict`, `dict` (Python 3.6+), or another `ModuleDict`.

**Key Features:**
- **Indexing**: Can be indexed like a regular Python dictionary.
- **Order Preservation**: Maintains order based on insertion and ordered merges.
- **Module Registration**: Automatically registers contained modules for PyTorch's module tracking.

**Common Methods:**
- `clear()`: Remove all items.
- `items()`, `keys()`, `values()`: Return iterables of key-value pairs, keys, or values.
- `pop(key)`: Remove and return a module by key.
- `update(modules)`: Update with new key-module pairs, overwriting existing keys and preserving order if the input is ordered.

**Example Usage:**
```python
self.ops = nn.ModuleDict({"conv": nn.Conv2d(10, 10, 3), "pool": nn.MaxPool2d(3)})
x = self.ops["conv"](x)  # Access and use modules by key
```

Use `ModuleDict` to manage named, dynamically accessible submodules within a neural network.
