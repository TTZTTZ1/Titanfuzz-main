- `optimizer`: The optimizer to wrap.
- `start_factor`: Initial multiplier for the learning rate (default: 1/3).
- `end_factor`: Final multiplier after linear progression (default: 1.0).
- `total_iters`: Number of steps over which the multiplier transitions from `start_factor` to `end_factor` (default: 5).
- `last_epoch`: Index of the last epoch (default: -1, initializes learning rate).

**Functionality:**
- Multiplies the base learning rate by a linearly increasing factor until `total_iters` is reached.
- After `total_iters`, the multiplier remains at `end_factor`.
- Works alongside other learning rate adjustments.

**Example Usage:**

- `step()`: Updates the learning rate each epoch.
- `get_last_lr()`: Returns the most recent learning rates.
- `get_lr()`: Computes the next learning rates (used internally).

This scheduler is useful for warm-up phases or gradual learning rate increments.
