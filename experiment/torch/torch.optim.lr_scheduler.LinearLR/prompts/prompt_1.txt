The `LinearLR` scheduler linearly increases the learning rate from a starting factor to an ending factor over a specified number of iterations.

**Key Parameters:**
- `start_factor`: Initial multiplier for the learning rate (default: 1/3).
- `end_factor`: Final multiplier for the learning rate (default: 1.0).
- `total_iters`: Number of iterations over which the multiplier transitions from `start_factor` to `end_factor` (default: 5).

**How It Works:**
- For each epoch up to `total_iters`, the learning rate is scaled by a linearly interpolated factor between `start_factor` and `end_factor`.
- After `total_iters` epochs, the multiplier remains at `end_factor` (typically 1.0, so the original learning rate is used).

**Example Usage:**
```python
scheduler = LinearLR(optimizer, start_factor=0.05, total_iters=40)
for epoch in range(100):
    train(...)
    validate(...)
    scheduler.step()
```
This gradually ramps up the learning rate from 5% to 100% over 40 epochs, then maintains the base learning rate thereafter.

**Notes:**
- Works alongside other learning rate modifications.
- Use `scheduler.step()` after the optimizer's `step()`.
- Call `get_last_lr()` to retrieve the current learning rates.
