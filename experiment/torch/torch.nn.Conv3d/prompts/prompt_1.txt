`torch.nn.Conv3d` applies a 3D convolution over an input with shape `(N, C_in, D, H, W)` to produce an output `(N, C_out, D_out, H_out, W_out)`.  

**Key parameters:**
- `in_channels`: Input channels  
- `out_channels`: Output channels  
- `kernel_size`: Size of convolving kernel (int or tuple of 3 ints)  
- `stride`: Stride for cross-correlation (default: 1)  
- `padding`: Padding added to all six sides (int, tuple, or `'valid'`/`'same'`)  
- `dilation`: Spacing between kernel elements (default: 1)  
- `groups`: Controls input-output connections (default: 1)  
- `bias`: Whether to add a learnable bias (default: `True`)  
- `padding_mode`: `'zeros'`, `'reflect'`, `'replicate'`, or `'circular'`  

**Output size formulas** (for each dimension `D`, `H`, `W`):  
\[
\text{out} = \left\lfloor \frac{\text{in} + 2 \times \text{padding} - \text{dilation} \times (\text{kernel\_size} - 1) - 1}{\text{stride}} + 1 \right\rfloor
\]

**Notes:**
- Supports complex data types and various padding modes.  
- `groups=1` (default): all inputs connected to all outputs.  
- `groups=in_channels` with `out_channels = K * in_channels`: depthwise convolution.  
- `padding='same'` maintains input spatial dimensions when `stride=1`.  

**Example:**
```python
m = nn.Conv3d(16, 33, (3, 5, 2), stride=(2, 1, 1), padding=(4, 2, 0))
input = torch.randn(20, 16, 10, 50, 100)

```
