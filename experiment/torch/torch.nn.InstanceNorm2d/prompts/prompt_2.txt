`InstanceNorm2d` applies instance normalization to 4D inputs (e.g., images with shape `(N, C, H, W)`), normalizing each channel independently for each sample in a batch. It computes mean and variance per channel per instance, stabilizing training by reducing internal covariate shift.

**Key parameters:**
- `num_features`: Number of channels `C` in the input.
- `affine`: If `True`, adds learnable scale (`γ`) and shift (`β`) parameters per channel.
- `track_running_stats`: If `True`, tracks running mean/variance during training for evaluation (uncommon in instance norm; typically kept `False`).

**Default behavior:**  
Uses instance-specific statistics in both training and evaluation unless `track_running_stats=True`.

**Use case:**  
Common in style transfer and image generation tasks, where preserving instance-specific features is important. Unlike `LayerNorm`, it operates on channeled spatial data and usually does not apply affine transformation by default.
