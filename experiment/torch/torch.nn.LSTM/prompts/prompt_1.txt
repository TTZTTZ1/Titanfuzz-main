The `torch.nn.LSTM` module implements a multi‑layer Long Short‑Term Memory (LSTM) recurrent neural network. It processes input sequences by applying the standard LSTM equations for input, forget, cell, and output gates.

**Key parameters:**
- `input_size`: Number of features in the input.
- `hidden_size`: Number of features in the hidden state.
- `num_layers`: Number of stacked LSTM layers (default: 1).
- `batch_first`: If `True`, input/output tensors are shaped as `(batch, seq, feature)` instead of `(seq, batch, feature)`.
- `bidirectional`: If `True`, uses a bidirectional LSTM.
- `dropout`: Dropout probability applied between LSTM layers (except the last).
- `proj_size`: If > 0, uses LSTM with projections, reducing hidden state dimension to `proj_size`.

**Inputs:**
- `input`: Tensor of shape `(L, N, input_size)` (or `(N, L, input_size)` if `batch_first=True`), where `L` is sequence length, `N` is batch size.
- `h_0`, `c_0`: Optional initial hidden and cell states; default to zeros.

**Outputs:**
- `output`: Tensor containing the output features `(h_t)` for each time step.
- `h_n`, `c_n`: Final hidden and cell states for each element in the sequence.

**Key notes:**
- Supports packed sequences for variable-length inputs.
- When `bidirectional=True`, outputs are concatenated forward/backward states.
- All weights are initialized uniformly from `U(-√k, √k)` where `k = 1/hidden_size`.
- Non‑deterministic behavior can occur on CUDA/cuDNN; environment variables can enforce determinism.
- `proj_size` must be smaller than `hidden_size`.
