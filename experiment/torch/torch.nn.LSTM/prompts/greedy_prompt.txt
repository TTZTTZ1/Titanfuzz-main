The `torch.nn.LSTM` module implements a multi‑layer Long Short‑Term Memory (LSTM) recurrent neural network. It processes sequential data by applying LSTM cells across each time step, supporting stacked layers, dropout, bidirectional processing, and optional projection layers.

**Key Parameters:**
- `input_size`: Number of features in the input.
- `hidden_size`: Number of features in the hidden state.
- `num_layers`: Number of stacked LSTM layers (default: 1).
- `batch_first`: If `True`, input/output tensors have shape `(batch, seq, feature)` instead of `(seq, batch, feature)`.
- `bidirectional`: If `True`, uses bidirectional LSTM (default: `False`).
- `dropout`: Dropout probability applied between LSTM layers (except the last).
- `proj_size`: If > 0, uses LSTM with projections, reducing hidden state dimension to `proj_size`.

**Inputs:**
- `input`: Tensor of shape `(L, N, H_in)` (or `(N, L, H_in)` if `batch_first=True`), where  
  `L` = sequence length, `N` = batch size, `H_in` = `input_size`.
- `h_0`, `c_0`: Optional initial hidden and cell states. If not provided, default to zeros.

**Outputs:**
- `output`: Tensor containing the output features `(h_t)` for each time step.
- `h_n`, `c_n`: Final hidden and cell states for each element in the sequence.

**Key Notes:**
- Supports variable‑length sequences via `PackedSequence`.
- All weights are initialized uniformly from `U(−√k, √k)` where `k = 1/hidden_size`.
- Bidirectional LSTMs concatenate forward and backward hidden states.
- Non‑deterministic behavior can occur on some CUDA/cuDNN versions; environment variables can enforce determinism.
- `proj_size` must be smaller than `hidden_size`.

**Example:**
```python
rnn = nn.LSTM(10, 20, 2)          # input_size=10, hidden_size=20, 2 layers
input = torch.randn(5, 3, 10)     # (seq_len=5, batch=3, input_size=10)
h0 = torch.randn(2, 3, 20)        # (num_layers=2, batch=3, hidden_size=20)
c0 = torch.randn(2, 3, 20)
output, (hn, cn) = rnn(input, (h0, c0))
```
