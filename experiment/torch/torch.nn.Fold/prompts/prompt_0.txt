`Fold` reconstructs a multi-dimensional tensor from sliding local blocks (e.g., image patches) by summing overlapping values. It is commonly used as the inverse operation of `Unfold`, though overlapping regions are summed rather than simply copied back.

**Key Parameters:**
- `output_size`: Spatial dimensions of the output tensor.
- `kernel_size`: Size of each sliding block.
- `stride`, `padding`, `dilation`: Control block extraction and overlap.

**Key Notes:**
- Input shape: `(N, C × ∏(kernel_size), L)` or `(C × ∏(kernel_size), L)`, where `L` is the total number of blocks.
- Output shape: `(N, C, *output_size)` or `(C, *output_size)`.
- For 2D data, this operation is often called `col2im`.
- `Fold` and `Unfold` are not exact inverses if blocks overlap; instead:  
  `fold(unfold(input)) == divisor * input`, where `divisor` is a shape/dtype-dependent tensor.

**Example:**
```python
fold = nn.Fold(output_size=(4, 5), kernel_size=(2, 2))
input = torch.randn(1, 12, 12)  # 12 = 3 channels × (2×2) kernel, 12 blocks
output = fold(input)  # Shape: (1, 3, 4, 5)
```
