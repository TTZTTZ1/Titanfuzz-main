**MultiStepLR** is a PyTorch learning rate scheduler that reduces the learning rate by a specified factor (`gamma`) at predefined epoch milestones.

### Key Features:
- **Purpose**: Decays the learning rate when training reaches specific epochs.
- **Parameters**:
  - `optimizer`: The optimizer to wrap.
  - `milestones`: A list of increasing epoch indices (e.g., `[30, 80]`).
  - `gamma`: Multiplicative decay factor (default: 0.1).
  - `last_epoch`: The last epoch index (default: -1, sets initial learning rate).
- **Behavior**: Learning rate is multiplied by `gamma` each time an epoch in `milestones` is reached. If an epoch appears multiple times in `milestones`, the decay is applied repeatedly (i.e., scaled by `gamma^n`).

### Example:

  - `lr = 0.05` for epochs < 30
  - `lr = 0.005` for 30 ≤ epoch < 80
  - `lr = 0.0005` for epoch ≥ 80

### Important Notes:
- Call `scheduler.step()` after each epoch (typically after `optimizer.step()`).
- Use `get_last_lr()` to retrieve the current learning rates.
- The scheduler can work alongside other learning rate modifications.
