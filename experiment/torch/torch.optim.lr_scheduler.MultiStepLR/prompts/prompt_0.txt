**MultiStepLR** is a PyTorch learning rate scheduler that reduces the learning rate by a fixed factor (`gamma`) at specified epoch milestones.

**Key Features:**
- **Decay schedule**: Learning rate is multiplied by `gamma` each time the training epoch reaches a value in the `milestones` list.
- **Cumulative decay**: If an epoch appears multiple times in `milestones`, the learning rate is scaled by `gamma` raised to the power of the number of occurrences.
- **Integration**: Works alongside other learning rate modifications.

**Parameters:**
- `optimizer`: The optimizer to wrap.
- `milestones` (list): Increasing list of epoch indices at which to decay the learning rate.
- `gamma` (float, default=0.1): Multiplicative decay factor.
- `last_epoch` (int, default=-1): Starting epoch index.

**Usage Example:**
```python
scheduler = MultiStepLR(optimizer, milestones=[30, 80], gamma=0.1)
for epoch in range(100):
    train(...)
    validate(...)
    scheduler.step()  # Call after optimizer.step()
```
In this example, the learning rate decays to 0.1× at epoch 30, and to 0.01× (0.1²) at epoch 80.

**Key Methods:**
- `step()`: Updates the learning rate (call once per epoch).
- `get_last_lr()`: Returns the most recent learning rates.
- `state_dict()` / `load_state_dict()`: Save and restore scheduler state.

**Note:** The scheduler should be stepped after the optimizer's update within each training epoch.
