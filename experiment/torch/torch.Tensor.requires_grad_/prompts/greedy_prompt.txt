`requires_grad_()` is an in‑place method that sets a tensor’s `requires_grad` attribute to `True` (or `False` if specified), enabling or disabling gradient tracking for autograd. It returns the same tensor after modification.

**Key points:**
- Used to start recording operations on a tensor that initially had `requires_grad=False`.
- Common for tensors loaded from data or after preprocessing, when you later need gradients.
- Call it before performing operations you want to differentiate.

**Example:**
```python
weights = torch.tensor([0.1, 0.2, 0.3])
weights.requires_grad_()   # Enable gradient tracking
out = weights.pow(2).sum()
out.backward()             # Gradients are now computed
```
