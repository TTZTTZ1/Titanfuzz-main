`requires_grad_()` is an in‑place method that sets a tensor’s `requires_grad` attribute to `True` (or `False` if specified), enabling or disabling gradient tracking for autograd. It returns the modified tensor.

**Main purpose:**  
To start recording operations on a tensor that initially had `requires_grad=False`—for example, after loading data or preprocessing—so that subsequent operations can be differentiated.

**Parameters:**  
- `requires_grad` (bool, default `True`): Whether autograd should track operations on the tensor.

**Example use:**  
```python
weights = torch.tensor([0.1, 0.2, 0.3], requires_grad=False)
weights.requires_grad_()   # Now gradients will be tracked
out = weights.sum()
out.backward()             # weights.grad is computed
```
