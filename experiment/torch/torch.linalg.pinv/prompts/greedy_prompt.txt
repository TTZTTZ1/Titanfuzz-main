---GENERAL SUMMARY START---
`torch.linalg.pinv` computes the **Moore–Penrose pseudoinverse** of a matrix via SVD (or eigenvalue decomposition for Hermitian/symmetric matrices). It supports batched inputs and various dtypes (float, double, complex).

**Key parameters:**
- `A`: Input tensor of shape `(*, m, n)`.
- `atol` / `rtol`: Tolerance thresholds for discarding small singular values.
- `hermitian`: If `True`, uses a more efficient method assuming `A` is Hermitian (complex) or symmetric (real).

**Notes:**
- Prefer `torch.linalg.lstsq` for solving linear systems, as it’s faster and more stable than explicitly computing the pseudoinverse.
- Uses `torch.linalg.svd` (or `torch.linalg.eigh` if `hermitian=True`), with the same numerical and gradient-related caveats.
- For CUDA, synchronizes device with CPU.

**Example:**
```python
A = torch.randn(3, 5)
A_pinv = torch.linalg.pinv(A)  # shape (5, 3)
```
