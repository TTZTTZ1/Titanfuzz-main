**Function:** `torch.linalg.pinv`  
**Purpose:** Computes the Moore-Penrose pseudoinverse of a matrix (or batch of matrices).  

**Key Features:**
- Supports real and complex floating-point tensors, including batched inputs.
- Uses SVD by default; if `hermitian=True`, it uses eigenvalue decomposition (assuming the matrix is Hermitian/symmetric).
- Singular values/eigenvalues below `max(atol, σ₁·rtol)` are truncated to zero for numerical stability.
- Default `rtol` is `max(m, n) * eps` (dtype-dependent) if not specified.

**Parameters:**
- `A`: Input tensor of shape `(*, m, n)`.
- `atol` / `rtol`: Absolute/relative tolerance thresholds (float or tensor).
- `hermitian`: If `True`, assumes `A` is Hermitian (complex) or symmetric (real).
- `out` (optional): Output tensor.

**Notes:**
- Prefer `torch.linalg.lstsq` for solving linear systems (`A.pinv() @ B`), as it is faster and more stable.
- Derivatives may be unstable due to internal use of SVD/eigenvalue decomposition.
- Synchronizes CUDA device with CPU.

**Example:**
```python
A = torch.randn(3, 5)
A_pinv = torch.linalg.pinv(A)  # Pseudoinverse of shape (5, 3)
```
