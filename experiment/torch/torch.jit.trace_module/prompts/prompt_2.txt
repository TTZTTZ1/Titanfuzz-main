`torch.jit.trace_module` traces multiple methods of a PyTorch module to create a single optimized `ScriptModule`. Unlike `torch.jit.trace`, which only traces the `forward` method, `trace_module` allows you to specify a dictionary of method names and corresponding example inputs. This is useful for capturing and compiling custom methods beyond `forward` into the final scripted module.

**Key parameters:**
- `mod`: The module containing methods to trace.
- `inputs`: A dict mapping method names (e.g., `'forward'`, `'method2'`) to example inputs for tracing each method.

**Optional arguments** include validation controls (`check_trace`, `check_inputs`, `check_tolerance`) and an indicator for keyword-argument inputs (`example_inputs_is_kwarg`).

**Example usage:**
```python
inputs = {
    "forward": example_forward_input,
    "weighted_kernel_sum": example_weight,
}
traced_module = torch.jit.trace_module(net, inputs)
```
The resulting `ScriptModule` contains traced versions of all specified methods, retaining the original module's submodules and parameters.
