The `torch.nn.Module` class is the foundational base class for all neural network modules in PyTorch. It provides a structured way to define, organize, and manage neural networks.

**Key Features:**

- **Nested Structure:** Modules can contain other modules, allowing hierarchical and reusable network designs.
- **Automatic Registration:** Submodules assigned as attributes are automatically registered, enabling parameter management and device transfers.
- **Parameter & Buffer Management:** Includes methods to register, access, and iterate over parameters (trainable) and buffers (non-trainable state like batch norm statistics).
- **State Persistence:** `state_dict()` and `load_state_dict()` allow easy saving and loading of model states.
- **Device & Data Type Management:** Methods like `.to()`, `.cuda()`, `.cpu()`, `.float()`, `.half()` facilitate moving models across devices and casting data types.
- **Training/Evaluation Modes:** `.train()` and `.eval()` toggle behaviors of certain layers (e.g., Dropout, BatchNorm).
- **Hooks:** Support for forward/backward hooks to intercept and modify computations during training.
- **Gradient Management:** `.zero_grad()` resets gradients; `.requires_grad_()` controls autograd recording.
- **Module Inspection:** Utilities like `.children()`, `.modules()`, `.named_parameters()` help inspect and manipulate the module tree.
- **Compilation:** `.compile()` integrates with `torch.compile` for optimized forward passes.

**Basic Usage:**
Subclass `nn.Module`, define layers in `__init__`, and implement the `forward` method to specify the computation.

**Example:**
```python
import torch.nn as nn

class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(10, 5)
    def forward(self, x):
        return self.linear(x)
```
