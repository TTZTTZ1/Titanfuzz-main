The `torch.nn.Module` class is the foundational building block for all neural network models in PyTorch. It provides a structured way to define, organize, and manage the components of a neural network.

**Key Features:**
- **Hierarchical Structure:** Modules can contain other modules, enabling nested, tree-like architectures. Submodules assigned as attributes are automatically registered.
- **Parameter & Buffer Management:** Handles model parameters (trainable) and buffers (non-trainable state like BatchNorm running statistics) with methods for access, dtype casting, and device movement.
- **State Persistence:** `state_dict()` and `load_state_dict()` allow saving and loading model checkpoints.
- **Hooks:** Supports forward/backward hooks for custom logic during execution and gradient computation.
- **Training Utilities:** Includes methods for setting training/evaluation mode (`train()`, `eval()`), zeroing gradients (`zero_grad()`), and freezing parameters (`requires_grad_()`).
- **Device & Dtype:** Methods like `to()`, `cuda()`, `cpu()` move parameters/buffers across devices and cast dtypes.

**Core Methods:**
- `forward()`: Must be overridden to define the model's computation.
- `parameters()`, `buffers()`: Iterate over model parameters and buffers.
- `children()`, `modules()`: Navigate the module hierarchy.
- `apply()`: Recursively apply a function to all submodules.
- `get_submodule()`, `get_parameter()`: Access nested components by name.

**Usage Example:**
```python
class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(10, 5)  # Automatically registered
    def forward(self, x):
        return self.linear(x)
```

In essence, `nn.Module` encapsulates model definition, state management, and execution, forming the backbone of PyTorch's neural network API.
