The `torch.nn.Module` class is the foundational base class for all neural network modules in PyTorch. It provides a structured way to define, organize, and manage neural network components.

**Key Features and Usage:**

- **Nested Structure:** Modules can contain other modules, forming a tree-like hierarchy. Submodules assigned as attributes are automatically registered, enabling parameter management and device transfers.
- **Core Methods:**
  - `forward()`: Must be overridden to define the computation performed during each forward pass.
  - `to()`: Moves parameters and buffers to a specified device or casts their data types.
  - `state_dict()` / `load_state_dict()`: Save and load the module's state (parameters and buffers).
  - `train()` / `eval()`: Switch between training and evaluation modes (affects modules like Dropout and BatchNorm).
  - `parameters()` / `named_parameters()`: Iterate over model parameters, commonly used for optimizer setup.
  - `zero_grad()`: Resets parameter gradients before backpropagation.
- **Module Management:**
  - `add_module()` / `register_parameter()` / `register_buffer()`: Explicitly add submodules, parameters, or buffers.
  - `children()` / `modules()`: Iterate over immediate children or all nested modules.
  - `get_submodule()` / `get_parameter()`: Retrieve specific nested components by name.
- **Hooks:** Supports forward/backward hooks (`register_forward_hook`, `register_full_backward_hook`) for customizing or inspecting computations.
- **Device and Type Conversion:** Methods like `cuda()`, `cpu()`, `float()`, `double()` allow in-place movement and type casting of parameters and buffers.
- **Utility Methods:** `apply()` for recursively applying functions to submodules (e.g., weight initialization), and `requires_grad_()` to toggle gradient recording.

**Example:**
```python
import torch.nn as nn

class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(10, 5)  # Automatically registered submodule

    def forward(self, x):
        return self.linear(x)
```

In summary, `nn.Module` encapsulates neural network layers, parameters, and operations, providing a consistent interface for building, training, and deploying models in PyTorch.
