---GENERAL SUMMARY START---
The `StepLR` learning rate scheduler decays the learning rate by a fixed factor (`gamma`) at regular intervals (`step_size` epochs).  

**Key Features:**  
- Multiplies the learning rate by `gamma` every `step_size` epochs.  
- Works with any PyTorch optimizer.  
- The `step()` method should be called after each epoch (typically after validation).  
- If `last_epoch = -1`, the initial learning rate is used; otherwise, scheduling continues from `last_epoch`.  

**Example Behavior:**  
With `lr = 0.05`, `step_size = 30`, `gamma = 0.1`:  
- Epochs 0–29: lr = 0.05  
- Epochs 30–59: lr = 0.005  
- Epochs 60–89: lr = 0.0005  

**Main Methods:**  
- `step()`: Updates the learning rate for the next epoch.  
- `get_last_lr()`: Returns the current learning rate(s).  
- `state_dict()` / `load_state_dict()`: Save and restore scheduler state.  

**Note:**  
Decay can be combined with other learning rate adjustments outside the scheduler.
---GENERAL SUMMARY END---
