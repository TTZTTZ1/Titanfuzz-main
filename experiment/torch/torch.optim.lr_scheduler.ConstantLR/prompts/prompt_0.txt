The `ConstantLR` learning rate scheduler multiplies the learning rate by a specified constant factor for a set number of initial training steps (epochs), then restores it to the original value.

**Key Features:**
- **Factor scaling:** Multiplies the learning rate by `factor` (default: 1/3) for the first `total_iters` epochs.
- **Automatic restoration:** After `total_iters` epochs, the learning rate returns to its original value (scaled by 1/`factor`).
- **Integration:** Works alongside other learning rate adjustments from outside the scheduler.

**Parameters:**
- `optimizer`: The optimizer to wrap.
- `factor` (float, default=1./3): Multiplier applied to the learning rate during the initial phase.
- `total_iters` (int, default=5): Number of epochs to apply the factor.
- `last_epoch` (int, default=-1): Starting epoch index; -1 initializes from the base learning rate.

**Usage Example:**
```python
scheduler = ConstantLR(optimizer, factor=0.5, total_iters=40)
for epoch in range(100):
    train(...)
    validate(...)
    scheduler.step()  # Call after optimizer.step()
```
In this example, the learning rate is halved for the first 40 epochs, then returns to its original value for the remaining epochs.
