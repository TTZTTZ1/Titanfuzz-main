**ConstantLR** is a PyTorch learning rate scheduler that multiplies the learning rate by a constant factor for a specified number of initial training steps, then restores it to the original value.

**Key Features:**
- **Factor scaling**: Multiplies the learning rate by `factor` (default: 1/3) for the first `total_iters` steps.
- **Automatic restoration**: After `total_iters` steps, the learning rate returns to its original value (scaled by 1/`factor`).
- **Integration**: Works alongside other learning rate modifications from outside the scheduler.

**Parameters:**
- `optimizer`: The optimizer to wrap.
- `factor` (float, default=1./3): Multiplier applied to the learning rate during the initial phase.
- `total_iters` (int, default=5): Number of steps during which the factor is applied.
- `last_epoch` (int, default=-1): Starting epoch index.

**Usage Example:**
```python
scheduler = ConstantLR(optimizer, factor=0.5, total_iters=40)
for epoch in range(100):
    train(...)
    validate(...)
    scheduler.step()  # Call after optimizer.step()
```

**Behavior:**
- For epochs 0–39: learning rate = original lr × 0.5
- From epoch 40 onward: learning rate returns to original value

**Key Methods:**
- `step()`: Updates the learning rate (call after optimizer.step()).
- `get_last_lr()`: Returns the most recent learning rates.
- `get_lr()`: Computes the next learning rate (internal use).
- `state_dict()` / `load_state_dict()`: Save and restore scheduler state.

This scheduler is useful for implementing warm-up phases or temporary learning rate reductions at the start of training.
