---GENERAL SUMMARY START---
`torch.fmod` computes the element-wise remainder of division, following C++’s `std::fmod`. The result retains the sign of the dividend (`input`) and has a magnitude less than the divisor (`other`).

**Key points:**
- Supports broadcasting, type promotion, and integer/float inputs.
- Not supported for complex numbers.
- Returns `NaN` for division by zero with floats; raises an error or returns undefined values for integer division by zero.
- Differs from `torch.remainder` (Python’s modulus) in how rounding is handled.

**Example:**
```python
torch.fmod(torch.tensor([-3., -2, -1, 1, 2, 3]), 2)
# tensor([-1., -0., -1.,  1.,  0.,  1.])
```
