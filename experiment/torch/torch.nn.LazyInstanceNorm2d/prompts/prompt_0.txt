`LazyInstanceNorm2d` is a PyTorch module for 2D instance normalization that automatically infers the number of input channels (`num_features`) from the first forward pass. It inherits from `InstanceNorm2d` but delays parameter initialization until the input size is known.

**Key Features:**
- **Lazy Initialization:** Automatically sets `num_features` from `input.size(1)`.
- **Lazy Parameters:** Weight, bias, running_mean, and running_var are initialized lazily.
- **Inherits Behavior:** Functions identically to `InstanceNorm2d` after initialization.

**Parameters:**
- `eps` (float): Numerical stability term (default: 1e-5).
- `momentum` (float): For running mean/variance computation (default: 0.1).
- `affine` (bool): If `True`, includes learnable scale/shift parameters (default: `True`).
- `track_running_stats` (bool): If `True`, tracks running statistics (default: `True`).

**Input/Output Shape:**  
`(N, C, H, W)` or `(C, H, W)` → Same shape as input.

**Use Case:**  
Useful when the input channel size is unknown at model definition time (e.g., dynamic architectures). After the first forward pass, it becomes a standard `InstanceNorm2d` module.

**Note:**  
See `LazyModuleMixin` for limitations of lazy modules.
