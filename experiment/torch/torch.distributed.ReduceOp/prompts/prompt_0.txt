The PyTorch `torch.distributed` package enables multi-process parallel training across multiple machines or GPUs. Here's a concise overview of its key components and usage:

## **Core Concepts**
- **Backends**: Supports `gloo` (CPU/GPU), `nccl` (CUDA GPU), `xccl` (XPU GPU), and `mpi`. NCCL is recommended for GPU training, Gloo for CPU.
- **Initialization**: Must call `init_process_group()` or `init_device_mesh()` before any distributed operations.
- **Process Groups**: Default "world" group includes all processes; custom groups can be created with `new_group()`.

## **Key Functions**
- **Collective Operations**: `all_reduce`, `broadcast`, `all_gather`, `scatter`, `reduce_scatter`, `all_to_all`, `barrier`
- **Point-to-Point**: `send`/`recv` (synchronous), `isend`/`irecv` (asynchronous)
- **Object Collectives**: `broadcast_object_list`, `all_gather_object` (for Python objects, but limited performance)
- **Utilities**: `get_rank()`, `get_world_size()`, `get_backend()`

## **Initialization Methods**
1. **Environment Variables** (default): Set `MASTER_ADDR`, `MASTER_PORT`, `WORLD_SIZE`, `RANK`
2. **TCP**: Specify a master address: `tcp://10.1.1.20:23456`
3. **Shared File System**: Use a shared file: `file:///mnt/nfs/sharedfile`

## **DeviceMesh** (Higher-Level Abstraction)
- Manages process groups for N-dimensional parallelism
- Created via `init_device_mesh(device_type, mesh_shape)`
- Enables communication on specific mesh dimensions

## **Key Features**
- **Synchronous/Asynchronous Operations**: Most collectives support `async_op=True` for non-blocking execution
- **Distributed DataParallel**: Built on top for model parallel training
- **Key-Value Stores**: `TCPStore`, `FileStore`, `HashStore` for process coordination
- **Debugging Tools**: `monitored_barrier()`, `TORCH_DISTRIBUTED_DEBUG` environment variable, `breakpoint()` utility
- **Launch Utilities**: `torch.distributed.launch` and `torch.multiprocessing.spawn` for process management

## **Best Practices**
- Use NCCL backend for CUDA GPU training
- Call `destroy_process_group()` for cleanup
- Set `torch.cuda.set_device()` before NCCL operations
- Be cautious with object collectives due to performance limitations
- Use `DeviceMesh` for complex multi-dimensional parallelism setups

The package enables efficient distributed training with advantages over `DataParallel`, including per-process optimizers and independent Python interpreters to avoid GIL contention.
