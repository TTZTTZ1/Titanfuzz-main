# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
`BatchNorm1d` applies batch normalization to 2D or 3D inputs, normalizing across the feature dimension (C) for each mini-batch. It stabilizes training by standardizing inputs: subtracting the mean and dividing by the standard deviation, then scaling and shifting with learnable parameters γ and β (if `affine=True`).

**Key parameters:**
- `num_features`: Number of features/channels (C)
- `eps`: Small constant for numerical stability (default: 1e-5)
- `momentum`: Weight for updating running statistics (default: 0.1)
- `affine`: Enables learnable γ and β (default: True)
- `track_running_stats`: Tracks moving mean/variance for evaluation (default: True)

**Behavior:**
- **Training:** Uses batch statistics; updates running mean/variance if `track_running_stats=True`
- **Evaluation:** Uses running statistics (if tracked) or batch statistics (if not)

**Input/Output shapes:**  
- 2D: `(N, C)` → `(N, C)`  
- 3D: `(N, C, L)` → `(N, C, L)`  

**Example:**
```python
m = nn.BatchNorm1d(100)  # With learnable parameters
input = torch.randn(20, 100)
output = m(input)
```
