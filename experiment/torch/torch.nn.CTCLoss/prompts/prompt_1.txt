`torch.nn.CTCLoss` computes the Connectionist Temporal Classification loss, which aligns variable-length input sequences (e.g., audio frames) with target sequences (e.g., text labels) without requiring explicit segmentation. It sums probabilities over all possible alignments, assuming a many-to-one mapping.

**Key Parameters:**
- `blank`: Blank label index (default 0).
- `reduction`: Loss reduction method (`'none'`, `'mean'`, `'sum'`).
- `zero_infinity`: Whether to zero infinite losses (occurs when inputs are too short).

**Input Shapes:**
- `log_probs`: `(T, N, C)` or `(T, C)` — log-softmax probabilities over classes (including blank).
- `targets`: `(N, S)` (padded) or concatenated 1D tensor.
- `input_lengths`: Lengths of each input sequence (`≤ T`).
- `target_lengths`: Lengths of each target sequence.

**Usage Notes:**
- Supports both batched and unbatched inputs.
- For CuDNN acceleration: use concatenated targets, `input_lengths = T`, `target_lengths ≤ 256`, `torch.int32` for lengths, and `torch.float32` for log probabilities.
- Enables gradient flow through input log probabilities.

**Example:**
```python
ctc_loss = nn.CTCLoss()
loss = ctc_loss(log_probs, targets, input_lengths, target_lengths)
loss.backward()
```
