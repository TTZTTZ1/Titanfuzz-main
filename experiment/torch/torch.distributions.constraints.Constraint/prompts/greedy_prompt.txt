The `torch.distributions` package provides a comprehensive framework for working with probability distributions in PyTorch, enabling stochastic computation graphs and gradient estimation for optimization. Here's a concise summary of its key aspects:

## **Core Concepts**
- **Parameterizable distributions**: Create distributions with learnable parameters (e.g., `Normal(loc, scale)`)
- **Two gradient estimation methods**:
  - **Score function/REINFORCE**: Uses `sample()` + `log_prob()` (policy gradients)
  - **Pathwise derivative/Reparameterization**: Uses `rsample()` (VAEs, differentiable sampling)

## **Main Components**

### **1. Distribution Classes**
- **Base class**: `Distribution` - abstract base with methods like:
  - `sample()`, `rsample()` - sampling
  - `log_prob()` - log probability density
  - `entropy()`, `mean`, `variance` - statistics
  - `cdf()`, `icdf()` - cumulative distribution functions

- **Common distributions**:
  - Continuous: `Normal`, `Beta`, `Gamma`, `Exponential`, `Dirichlet`
  - Discrete: `Bernoulli`, `Categorical`, `Binomial`, `Poisson`
  - Multivariate: `MultivariateNormal`, `LowRankMultivariateNormal`
  - Specialized: `LKJCholesky`, `Wishart`, `VonMises`

### **2. Key Features**
- **Batch/event shapes**: Support for batched parameters and samples
- **ExponentialFamily**: Special class for distributions in exponential family form
- **TransformedDistribution**: Create new distributions via transforms of base distributions
- **MixtureSameFamily**: Mixture distributions with same component type
- **Independent**: Reinterpret batch dims as event dims

### **3. Transforms System**
- **Transform classes**: `AffineTransform`, `ExpTransform`, `SigmoidTransform`, etc.
- **Composition**: `ComposeTransform` chains multiple transforms
- **Constraint handling**: Transforms map between constrained and unconstrained spaces

### **4. Constraints & Registry**
- **Constraints**: Define valid parameter ranges (e.g., `simplex`, `positive`, `interval`)
- **ConstraintRegistry**: Maps constraints to transforms for optimization
  - `transform_to()`: For unconstrained optimization
  - `biject_to()`: For bijective transforms (HMC)

### **5. KL Divergence**
- `kl_divergence()`: Compute KL divergence between distributions
- Extensible via `register_kl()` decorator

## **Typical Usage Patterns**

### **REINFORCE (Policy Gradient)**
```python
probs = policy_network(state)
m = Categorical(probs)
action = m.sample()
loss = -m.log_prob(action) * reward
loss.backward()
```

### **Reparameterization Trick**
```python

```

