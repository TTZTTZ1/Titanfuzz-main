`torch.fft.irfft` computes the inverse real-valued Fast Fourier Transform, converting a one-sided Hermitian frequency spectrum (as produced by `rfft`) back to a real-valued time-domain signal. It requires the input to satisfy Hermitian symmetry for a real output.

**Key Parameters:**
- `input`: One-sided Hermitian spectrum.
- `n`: Output signal length (crucial for odd-length signals to ensure proper inversion).
- `dim`: Dimension along which to compute the transform.
- `norm`: Normalization mode (`"backward"` default, `"forward"`, or `"ortho"`).

**Important Notes:**
- Always specify `n` (original signal length) to guarantee correct reconstruction, especially for odd-length signals.
- The function ignores imaginary components in real-valued required frequencies (e.g., zero-frequency).
- Default `n` assumes an even-length output: `2 * (input.size(dim) - 1)`.

**Example:**
```python
t = torch.linspace(0, 1, 5)
T = torch.fft.rfft(t)
reconstructed = torch.fft.irfft(T, n=t.numel())  # Specify n for exact inversion
```
