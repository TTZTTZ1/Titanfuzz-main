`torch.fft.irfft` computes the inverse real-valued Fast Fourier Transform, converting a one-sided Hermitian frequency spectrum (as produced by `rfft`) back to a real-valued time-domain signal. It requires the input to satisfy Hermitian symmetry, with imaginary parts at certain frequencies (like zero-frequency) being ignored.

**Key parameters:**
- `input`: One-sided Hermitian spectrum.
- `n`: Output signal length (crucial for odd-length signals to ensure correct inversion).
- `dim`: Dimension for the transform.
- `norm`: Normalization mode (`"backward"` default, `1/n` normalization).

**Important notes:**
- Always specify `n` (original signal length) for accurate reconstruction, especially with odd-length signals.
- Default `n` assumes even output: `2*(input.size(dim)-1)`.
- Supports half-precision on CUDA (SM53+) with power-of-2 signal lengths.

**Example:**
```python
t = torch.linspace(0, 1, 5)
T = torch.fft.rfft(t)                 # Forward transform
reconstructed = torch.fft.irfft(T, n=t.numel())  # Inverse with explicit length
```
