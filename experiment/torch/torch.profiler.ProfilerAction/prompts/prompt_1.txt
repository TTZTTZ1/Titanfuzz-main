The PyTorch Profiler (`torch.profiler`) is a performance analysis tool for collecting detailed metrics during model training and inference. It helps identify expensive operations, analyze kernel activity, and visualize execution traces.

**Key Components:**

1. **`profile` Context Manager** – The main interface for profiling code blocks. It supports configurable activities (CPU, CUDA, XPU), memory tracking, shape recording, and FLOPs estimation.

2. **Profiling Schedule** – Use `schedule()` to define cycles of waiting, warming up, and active recording, enabling periodic profiling in long-running jobs.

3. **Output & Export**  
   - `key_averages()`: Aggregates events by operator, shape, or stack.  
   - `export_chrome_trace()`: Exports traces in Chrome JSON format.  
   - `tensorboard_trace_handler()`: Directly outputs traces for TensorBoard visualization.

4. **Advanced Features**  
   - **Execution Trace Observer**: Enables graph-based workload representation.  
   - **ITT APIs** (`torch.profiler.itt`): Intel Instrumentation and Tracing Technology for low-level event marking.  
   - **Dynamic Control**: `toggle_collection_dynamic()` allows enabling/disabling activities during profiling.

**Basic Usage Example:**
```python
with torch.profiler.profile(activities=[ProfilerActivity.CPU, ProfilerActivity.CUDA]) as prof:
    model(input)
print(prof.key_averages().table(sort_by="cuda_time_total"))
```

**Note:** Legacy `torch.autograd.profiler` is deprecated; use `torch.profiler` for new projects. Enabling options like `record_shapes` or `with_stack` adds overhead.
