**DistributedDataParallel (DDP)** is a PyTorch module for distributed data parallelism across multiple GPUs or machines. It synchronizes gradients across model replicas to enable efficient multi-process training.

### Key Features:
- **Data Parallelism**: Each process holds a full model replica; gradients are averaged across processes via all-reduce.
- **Performance**: Faster than `DataParallel` for single-node multi-GPU training.
- **Mixed Precision**: Supports training with mixed `fp16`/`fp32` parameters.
- **Flexible Backends**: Works with NCCL (recommended for GPUs), Gloo, MPI.

### Basic Usage:
1. **Initialize** `torch.distributed` with `init_process_group()`.
2. **Wrap model**:
   ```python
   model = DistributedDataParallel(model, device_ids=[rank], output_device=rank)
   ```
3. **Ensure each process** uses a unique GPU (set via `torch.cuda.set_device()` or `torch.accelerator.set_device_index()`).

### Important Parameters:
- `process_group`: Custom process group for communication (default: entire world).
- `broadcast_buffers`: Syncs buffers (e.g., BatchNorm stats) before forward pass.
- `find_unused_parameters`: Marks unused parameters for reduction (useful for dynamic graphs).
- `static_graph`: Optimizes performance for static training graphs.
- `bucket_cap_mb`: Controls gradient bucket size for overlapping communication with backward pass.

### Key Methods:
- `no_sync()`: Context manager to disable gradient synchronization (accumulate gradients locally).
- `join()`: Handles uneven inputs across processes by shadowing collective operations.
- `register_comm_hook()`: Customize gradient aggregation (e.g., compression, custom all-reduce).

### Warnings:
- **Model Consistency**: All processes must have identical models and parameter registration order.
- **No Parameter Changes**: Do not add/remove parameters after wrapping with DDP.
- **Gradient Accumulation**: Works only with gradient accumulation in `.grad` attributes (not `torch.autograd.grad`).
- **Checkpointing**: Use `use_reentrant=False` for gradient checkpointing compatibility.

### Example:
```python
torch.distributed.init_process_group(backend='nccl', world_size=4)
model = DistributedDataParallel(model)
```

DDP is essential for scalable multi-GPU training, providing efficient gradient synchronization while supporting advanced features like custom communication hooks and uneven input handling.
