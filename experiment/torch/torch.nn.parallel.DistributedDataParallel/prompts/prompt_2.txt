**DistributedDataParallel (DDP)** is a PyTorch module that enables distributed training by synchronizing gradients across multiple GPUs or nodes. It wraps a model and performs gradient all-reduce automatically during backpropagation, ensuring all model replicas stay synchronized.

### **Key Features:**
- **Data Parallelism**: Each GPU/process holds a full model replica; gradients are averaged across processes.
- **Efficiency**: Faster than `DataParallel` for single-node multi-GPU training.
- **Flexible Backends**: Works with NCCL (recommended for GPUs), Gloo, or MPI.

### **Setup Requirements:**
- Initialize `torch.distributed` via `init_process_group()` before creating DDP.
- Use one process per GPU, setting the device explicitly (e.g., `torch.cuda.set_device(i)`).
- Ensure identical models and parameter order across all processes.

### **Important Parameters:**
- `device_ids`: Specifies GPU(s) for the process (single-device modules only).
- `broadcast_buffers`: Syncs buffers (e.g., BatchNorm stats) before forward pass.
- `find_unused_parameters`: Marks unused parameters for reduction (useful for dynamic graphs).
- `static_graph`: Optimizes performance for static training graphs.
- `bucket_cap_mb`: Controls gradient bucket size for overlapping communication with computation.

### **Usage Example:**
```python
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP

dist.init_process_group(backend='nccl', world_size=world_size, init_method='...')
model = DDP(model, device_ids=[local_rank])
```

### **Key Methods:**
- `no_sync()`: Disables gradient synchronization within the context (for gradient accumulation).
- `join()`: Handles uneven inputs across processes by shadowing collective operations.
- `register_comm_hook()`: Allows custom gradient aggregation (e.g., compression, gossip algorithms).

### **Notes & Warnings:**
- **Checkpointing**: Use `torch.save`/`torch.load` with proper `map_location`.
- **Mixed Precision**: Supports mixed `fp16`/`fp32` parameters.
- **Constraints**: Model parameters must not change after DDP wrapping; all processes must have identical models.
- **Gradient Reduction**: Gradients are averaged, not summed—adjust learning rates accordingly for batch size equivalence.

DDP is essential for scalable multi-GPU training, providing efficient synchronization while maintaining flexibility for advanced use cases.
