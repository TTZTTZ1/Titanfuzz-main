**DistributedDataParallel (DDP)** is a PyTorch module for distributed training that synchronizes gradients across multiple GPUs or nodes. It wraps a model to enable data parallelism by performing gradient all-reduce across processes, ensuring consistent model updates.

### **Key Features:**
- **Gradient Synchronization**: Automatically averages gradients across all processes during backward passes.
- **Process Management**: Requires `torch.distributed.init_process_group()` to be initialized beforehand.
- **Performance**: Faster than `DataParallel` for single-node multi-GPU training.
- **Mixed Precision**: Supports training with mixed `fp16`/`fp32` parameters.
- **Flexible Backends**: Recommends `nccl` backend for GPU-based training.

### **Basic Usage:**
```python
import torch
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP

# Initialize distributed process group
dist.init_process_group(backend='nccl', world_size=world_size, init_method='...')
torch.cuda.set_device(local_rank)

# Wrap model with DDP
model = DDP(model, device_ids=[local_rank], output_device=local_rank)
```

### **Important Parameters:**
- `device_ids`: Specifies GPU device(s) for the process (single-device modules only).
- `broadcast_buffers`: Synchronizes buffers (e.g., BatchNorm stats) at the start of each forward pass.
- `find_unused_parameters`: Marks unused parameters for reduction (useful for dynamic graphs).
- `static_graph`: Optimizes performance for static computation graphs.
- `bucket_cap_mb`: Controls gradient bucket size for overlapping communication with computation.

### **Context Managers:**
- `no_sync()`: Disables gradient synchronization within the context, allowing gradient accumulation.
- `join()`: Handles uneven inputs across processes by shadowing collective operations.

### **Communication Hooks:**
Custom aggregation strategies can be implemented via `register_comm_hook()` for advanced gradient reduction algorithms.

### **Key Notes:**
- **Input Responsibility**: Users must manually partition data (e.g., using `DistributedSampler`).
- **Model Consistency**: All processes must have identical models and parameter registration order.
- **Checkpointing**: Use `consume_prefix_in_state_dict_if_present()` to load DDP checkpoints into non-DDP models.
- **Limitations**: Does not support `torch.autograd.grad()`; requires gradients accumulated in parameter `.grad` attributes.

### **Example with Training Loop:**
```python
for inputs, labels in dataloader:
    outputs = model(inputs)
    loss = criterion(outputs, labels)
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()
```

DDP is essential for scalable multi-GPU training, providing efficient gradient synchronization while maintaining flexibility for custom distributed workflows.
