`torch.fft.rfft` computes the 1D Fourier transform of a **real-valued** input tensor. It returns only the positive frequencies (up to the Nyquist frequency), exploiting Hermitian symmetry to avoid redundant complex outputs. Key parameters include:

- **input**: Real-valued tensor.
- **n**: Optional length for zero-padding or trimming.
- **dim**: Dimension along which to compute the FFT (default: last dimension).
- **norm**: Normalization mode (`"backward"`, `"forward"`, or `"ortho"`).

**Example**:
```python
t = torch.arange(4)
torch.fft.rfft(t)  # Output: [6.+0.j, -2.+2.j, -2.+0.j]
```
Compared to `torch.fft.fft`, which returns all frequencies, `rfft` omits symmetric negative frequencies for efficiency.
