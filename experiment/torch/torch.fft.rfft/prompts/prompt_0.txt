- `input`: Real-valued tensor.
- `n`: Optional length for zero-padding or trimming.
- `dim`: Dimension along which to compute the FFT (default: last dimension).
- `norm`: Normalization mode (`"backward"`, `"forward"`, or `"ortho"`).

**Example:**
```python

torch.fft.rfft(t)  # Output: tensor([ 6.+0.j, -2.+2.j, -2.+0.j])
```
This returns half the frequencies compared to `torch.fft.fft`, omitting the conjugate-symmetric negative frequencies.
