`MultiLabelMarginLoss` is a PyTorch loss function for multi-class, multi-label classification tasks using a hinge loss (margin-based) approach. It computes the loss between predicted scores and target labels, where each sample can have multiple correct classes.

**Key points:**
- **Purpose**: Optimizes classification where each input can belong to multiple classes simultaneously.
- **Input/Output shapes**:  
  - Input: `(C)` or `(N, C)` (N = batch size, C = number of classes)  
  - Target: Same shape as input, with target class indices and padding by `-1`  
  - Output: Scalar loss (or per-sample if `reduction='none'`)
- **How it works**: For each sample, it sums over all pairs of target and non-target classes, applying `max(0, 1 - (score[target] - score[non-target]))`, then averages by the number of classes.
- **Target format**: Labels are provided as indices, with `-1` padding after the last relevant label to handle variable numbers of target classes per sample.
- **Reduction options**: Supports `'mean'`, `'sum'`, or `'none'` (via the `reduction` parameter).

**Example**:  
```python
loss_fn = nn.MultiLabelMarginLoss()
x = torch.tensor([[0.1, 0.2, 0.4, 0.8]])  # 4 classes
y = torch.tensor([[3, 0, -1, 1]])         # Targets: class 3 and 0 (padding with -1)
loss = loss_fn(x, y)  # Computes margin loss
```
