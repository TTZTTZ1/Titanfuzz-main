# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
`torch.jit.trace` converts a Python function or `nn.Module` into a TorchScript representation by recording tensor operations during a single execution with provided example inputs. It produces either a `ScriptFunction` (for standalone functions) or a `ScriptModule` (for modules), which can be optimized and run independently of Python.

**Key characteristics:**
- Ideal for pure tensor operations without data-dependent control flow (e.g., conditionals, loops).
- Traces only the operations executed during the example run; the same graph runs for all inputs.
- Supports both positional (`example_inputs`) and keyword (`example_kwarg_inputs`) example arguments.
- Includes options for validation (`check_trace`), strictness control (`strict`), and tolerance settings.

**Limitations:**
- Cannot capture dynamic behavior (e.g., varying control flow, training/eval mode differences).
- May produce incorrect traces if the function/module behavior changes based on input or state.

**Example usage:**
```python
traced = torch.jit.trace(func, example_inputs)  # For functions
traced = torch.jit.trace(module, example_inputs)  # For modules
```
---GENERAL SUMMARY END---
