`torch.jit.trace` is a PyTorch function that converts a Python function or `nn.Module` into a TorchScript representation by recording operations performed during a forward pass with example inputs. It is ideal for pure tensor operations without data-dependent control flow (e.g., conditionals, loops) or external dependencies.

**Key Points:**
- **Usage**: Provide a callable (`func`) and example inputs (`example_inputs` or `example_kwarg_inputs`).
- **Output**: Returns a `ScriptModule` (for modules) or `ScriptFunction` (for standalone functions) containing the traced graph.
- **Limitations**: Does not capture dynamic control flow; the traced graph is static and runs the same operations for all inputs.
- **Verification**: By default, checks consistency between original and traced outputs (`check_trace=True`).
- **When to Use**: Suitable for deterministic models with fixed operations; for dynamic behavior, use `torch.jit.script` instead.

**Example:**
```python
traced = torch.jit.trace(model, example_input)
```
