The `torch.optim.Rprop` class implements the Resilient Backpropagation (Rprop) algorithm, an adaptive learning rate optimizer that adjusts step sizes individually for each parameter based on gradient sign changes.

**Key Features:**
- **Adaptive Step Sizes**: Each parameter has its own step size (η) that adjusts based on whether consecutive gradients have the same or opposite signs.
- **Gradient Sign-Based Updates**: Uses only the sign of gradients for parameter updates, ignoring magnitude.
- **Step Size Adjustment Rules**:
  - If gradient signs are consistent (g_prev * g_current > 0): Increase step size multiplicatively (η ← min(η * η_plus, Γ_max))
  - If gradient signs change (g_prev * g_current < 0): Decrease step size multiplicatively (η ← max(η * η_minus, Γ_min)) and reset current gradient to zero
  - If gradient is zero: Keep step size unchanged

**Parameters:**
- `params`: Parameters to optimize
- `lr`: Initial learning rate (default: 0.01)
- `etas`: (etaminus, etaplus) multiplicative adjustment factors (default: (0.5, 1.2))
- `step_sizes`: (min_step, max_step) allowed step size range (default: (1e-6, 50))
- `capturable`, `foreach`, `maximize`, `differentiable`: Advanced optimization flags

**Typical Use Case:**
```python
optimizer = torch.optim.Rprop(model.parameters(), lr=0.01)
```

**Best For**: Problems where gradient magnitudes are unreliable or noisy, as Rprop relies only on gradient signs. Particularly effective for batch learning rather than online/mini-batch scenarios.

**Note**: Unlike momentum-based methods, Rprop doesn't use gradient momentum and can be sensitive to mini-batch training due to its sign-based adaptation.
