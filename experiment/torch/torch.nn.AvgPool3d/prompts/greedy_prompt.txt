`AvgPool3d` applies 3D average pooling over an input tensor of shape `(N, C, D, H, W)` or `(C, D, H, W)`. It reduces spatial dimensions by computing the average value within sliding 3D windows of size `kernel_size`.

**Key parameters:**
- `kernel_size`: size of the pooling window (int or tuple of 3 ints)
- `stride`: step size for sliding window (defaults to `kernel_size`)
- `padding`: zero-padding added to all three sides (int or tuple)
- `ceil_mode`: uses ceil (True) or floor (False) for output shape calculation
- `count_include_pad`: includes padding in average calculation when True
- `divisor_override`: optional custom divisor instead of `kernel_size` product

**Output shape:**  
`(N, C, D_out, H_out, W_out)` or `(C, D_out, H_out, W_out)`, where each dimension is computed as:  
`floor((input_size + 2×padding - kernel_size) / stride + 1)` (or ceil if `ceil_mode=True`).

**Note:** When `ceil_mode=True`, windows starting in the right padding region are ignored. Padding should not exceed half the kernel size.

**Example:**
```python
m = nn.AvgPool3d(3, stride=2)          # Square window
m = nn.AvgPool3d((3, 2, 2), stride=(2, 1, 2))  # Non-square window
```
