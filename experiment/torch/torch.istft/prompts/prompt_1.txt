`torch.istft` performs the inverse short-time Fourier transform, reconstructing a time-domain signal from its complex STFT representation. It is designed as the inverse of `torch.stft`, using least-squares estimation to recover the original signal.

**Key Parameters:**
- **input**: Complex tensor of shape `(B?, N, T)` (batch, frequencies, frames), as output by `stft(..., return_complex=True)`.
- **n_fft**: FFT size.
- **hop_length**: Frame step (default: `n_fft // 4`).
- **window**: Window function used in the forward STFT (required for correct reconstruction).
- **center**: Whether the STFT input was padded (affects length handling).
- **length**: Optional original signal length; if provided, output is padded/trimmed to this size.
- **return_complex**: If `True`, returns a complex tensor; default is `False` (real-valued output).

**Important Notes:**
- The function checks the nonzero overlap condition (NOLA) for numerical stability.
- If `center=True`, padding is added during STFT, and `istft` automatically trims left padding. Right padding handling may require the `length` parameter to avoid signal loss.
- From PyTorch 2.1 onward, not providing a `window` triggers a warning (future versions will require it).
---GENERAL SUMMARY END---
