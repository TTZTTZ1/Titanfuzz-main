`torch.istft` performs the inverse short-time Fourier transform, reconstructing a time-domain signal from its complex STFT representation. It is designed to be the inverse of `torch.stft`.

**Key Parameters:**
- **input**: Complex tensor of shape `(B?, N, T)` from `stft(..., return_complex=True)`.
- **n_fft**: FFT size used in the forward STFT.
- **hop_length**: Frame step (default: `n_fft // 4`).
- **window**: Window function used in the forward STFT (required from PyTorch 2.1 onward).
- **center**: Whether the STFT was centered (default: `True`).
- **length**: Optional original signal length to control output size/trimming.

**Functionality:**
- Reconstructs the least-squares estimate of the original signal.
- Automatically checks the NOLA (nonzero overlap add) condition.
- If `center=True`, it handles padding; the `length` parameter can be used to specify the exact output length.
- Output shape is `(B?, length)` (real-valued by default, unless `return_complex=True`).

**Note:** The input must be complex-valued (real inputs are no longer supported since v2.0), and the same window used in `stft` should be provided.
---GENERAL SUMMARY END---
