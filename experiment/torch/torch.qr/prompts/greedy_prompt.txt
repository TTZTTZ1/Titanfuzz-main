`torch.qr` computes the QR decomposition of a matrix (or batch of matrices) into an orthogonal matrix **Q** and an upper triangular matrix **R**, such that `input = Q @ R`.  

**Key details:**  
- **Deprecated:** Use `torch.linalg.qr` instead.  
- **`some=True` (default):** Returns the thin/reduced factorization (Q: m×k, R: k×n, where k = min(m, n)).  
- **`some=False`:** Returns the complete factorization (Q: m×m, R: m×n).  
- **Backpropagation caution:** Currently well-defined only when the first min(m, n) columns of the input are linearly independent.  
- **Platform-dependent:** Uses LAPACK (CPU) or MAGMA (CUDA), which may produce different valid decompositions across devices/platforms.  

**Example replacement:**  
- `Q, R = torch.qr(A)` → `Q, R = torch.linalg.qr(A)`  
- `Q, R = torch.qr(A, some=False)` → `Q, R = torch.linalg.qr(A, mode="complete")`
---GENERAL SUMMARY END---
