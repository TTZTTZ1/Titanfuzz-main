The `torch.optim.Adadelta` class implements the Adadelta optimization algorithm, an adaptive learning rate method designed to reduce reliance on manual hyperparameter tuning.

**Key Features:**
- **Adaptive Learning Rates:** Automatically adjusts per-parameter learning rates based on a window of past gradients and updates.
- **No Manual Learning Rate Scheduling:** The algorithm inherently handles learning rate adjustments, often eliminating the need for a separate scheduler.
- **Robust to Noisy Gradients:** Well-suited for problems with sparse or noisy gradients.

**Core Algorithm Parameters:**
- `params` (iterable): Parameters to optimize (required).
- `lr` (float/Tensor, default=1.0): Coefficient scaling the update before application.
- `rho` (float, default=0.9): Decay factor for moving averages of squared gradients and updates. Higher values slow the averaging.
- `eps` (float, default=1e-6): Term added to denominators for numerical stability.
- `weight_decay` (float, default=0): L2 penalty (weight decay).

- **Performance:** Supports a `foreach` implementation for faster execution on CUDA (uses more memory).
- **Advanced Use Cases:** Includes flags for CUDA graph capture (`capturable`), gradient-free maximization (`maximize`), and differentiability through the optimizer step (`differentiable`).
- **Standard Optimizer Methods:** Provides `step()`, `zero_grad()`, `state_dict()`, `load_state_dict()`, and `add_param_group()`.
- **Hooks:** Supports pre- and post-hooks for the `step()`, `state_dict()`, and `load_state_dict()` operations for custom logic.

**Typical Usage:**
```python

```
The algorithm is detailed in the paper *"ADADELTA: An Adaptive Learning Rate Method"* (Zeiler, 2012).
