**LambdaLR** is a PyTorch learning rate scheduler that multiplies the initial learning rate of each parameter group by a user-defined function of the current epoch.

**Key Features:**
- **Purpose:** Dynamically adjusts the learning rate based on a custom function (`lr_lambda`).
- **Inputs:** 
  - `optimizer`: The optimizer to wrap.
  - `lr_lambda`: A function (or list of functions) that takes the epoch index as input and returns a multiplicative factor.
  - `last_epoch`: The index of the last epoch (default: -1, meaning start from initial LR).
- **Usage:** Call `scheduler.step()` after each epoch (typically after `optimizer.step()`).

**Example:**
```python

# Different functions per parameter group

lambda2 = lambda epoch: 0.95 ** epoch
scheduler = LambdaLR(optimizer, lr_lambda=[lambda1, lambda2])
```

**Important Notes:**
- The scheduler updates the learning rate at each epoch via `step()`.
- Use `get_last_lr()` to inspect the current learning rates.
- State saving/loading (`state_dict`, `load_state_dict`) should be done alongside the optimizer's state.
- The `epoch` parameter in `step()` is deprecated; call `step()` without arguments.

**Ideal for:** Custom, non-standard learning rate schedules defined by arbitrary epoch-dependent functions.
