**LambdaLR** is a PyTorch learning rate scheduler that multiplies the initial learning rate by a custom function (or list of functions) at each epoch.

---

### **Key Features**
- **Purpose**: Adjusts the learning rate by applying a multiplicative factor defined by `lr_lambda`.
- **Customization**: Accepts a single function or a list of functions (one per parameter group) that compute the scaling factor based on the epoch index.
- **Initialization**: When `last_epoch=-1`, the initial learning rate is set as the base LR; subsequent updates apply the lambda function.

---

### **Parameters**
- `optimizer`: Wrapped optimizer (e.g., SGD, Adam).
- `lr_lambda`: Function or list of functions mapping `epoch` → multiplicative factor.
- `last_epoch`: Index of the last epoch (default: -1).

---

### **Usage Example**
```python
# Two parameter groups with different schedules
lambda1 = lambda epoch: epoch // 30        # Step-wise
lambda2 = lambda epoch: 0.95 ** epoch      # Exponential decay
scheduler = LambdaLR(optimizer, lr_lambda=[lambda1, lambda2])

for epoch in range(num_epochs):
    train(...)
    validate(...)
    scheduler.step()  # Update LR after each epoch
```

---

### **Important Methods**
- `step()`: Updates the learning rate (call after optimizer.step()).
- `get_last_lr()`: Returns the most recent learning rates.
- `state_dict()` / `load_state_dict()`: Save/restore scheduler state (requires optimizer state too).

---

### **Notes**
- The lambda function receives the epoch index as input.
- If using multiple parameter groups, provide a list of functions matching the number of groups.
- Lambda functions (e.g., `lambda epoch: ...`) are **not saved** in `state_dict`; use callable objects for serialization.
