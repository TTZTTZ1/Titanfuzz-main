The PyTorch Profiler is a performance analysis tool for collecting metrics during model training and inference. It helps identify expensive operations, analyze input shapes, examine stack traces, and visualize execution traces.

**Key Components:**

1. **`torch.profiler.profile`** – Main context manager for profiling
   - Supports CPU, CUDA, and XPU activities
   - Optional scheduling for long-running jobs
   - Can export results for TensorBoard visualization

2. **`_KinetoProfile`** – Lower-level profiler wrapper (experimental)

3. **Core Features:**
   - Shape recording (`record_shapes=True`)
   - Memory profiling (`profile_memory=True`)
   - Stack trace collection (`with_stack=True`)
   - FLOPs estimation (`with_flops=True`)
   - Module hierarchy tracking (`with_modules=True`)

4. **Output Options:**
   - Chrome trace format (`.json`)
   - Memory timeline (deprecated, use `torch.cuda.memory` instead)
   - Stack traces
   - Aggregated performance tables

5. **Scheduling:** Use `torch.profiler.schedule()` to control profiling cycles (wait/warmup/active phases)

6. **Intel ITT APIs:** For instrumenting code with marks and ranges

**Basic Usage:**
```python
with torch.profiler.profile(activities=[ProfilerActivity.CPU, ProfilerActivity.CUDA]) as prof:
    # Code to profile
    model(input)
    
print(prof.key_averages().table(sort_by="cuda_time_total"))
```

The profiler replaces legacy `torch.autograd.profiler` and provides enhanced visualization through TensorBoard integration.
