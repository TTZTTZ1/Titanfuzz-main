The `torch.profiler` module provides tools for profiling PyTorch model performance during training and inference. It collects detailed metrics to help identify expensive operations, analyze device activity, and visualize execution traces.

### Key Features:
- **Activity Tracking**: Supports CPU, CUDA, and XPU profiling.
- **Context Manager**: Use `torch.profiler.profile` as a context manager to profile code blocks.
- **Customizable Schedule**: Control profiling cycles with `schedule()` to skip, warm up, and record specific iterations.
- **Data Export**: Export traces in Chrome JSON format for visualization or use TensorBoard integration.
- **Advanced Metrics**: Optionally record operator input shapes, stack traces, memory usage, and FLOPs estimates.

### Basic Usage:
```python
with torch.profiler.profile(activities=[ProfilerActivity.CPU, ProfilerActivity.CUDA]) as prof:
    # Code to profile
    model(input)
print(prof.key_averages().table(sort_by="cuda_time_total"))
```

### Additional Tools:
- **Intel ITT APIs**: For low-level instrumentation and tracing.
- **Execution Trace Observer**: Enables graph-based workload representation for benchmarks and simulators.

This profiler replaces the legacy `torch.autograd.profiler` and is essential for optimizing PyTorch performance.
