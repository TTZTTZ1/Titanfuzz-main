The `torch.distributions` module provides a comprehensive framework for working with probability distributions in PyTorch. Here's a concise summary of its key aspects:

## **Core Purpose**
- Enables construction of stochastic computation graphs with gradient estimators for optimization
- Follows design patterns similar to TensorFlow Distributions
- Supports two main gradient estimation methods: Score Function (REINFORCE) and Pathwise Derivative (reparameterization trick)

## **Key Components**

### **1. Distribution Base Class**
- Abstract base class for all probability distributions
- Common methods: `sample()`, `rsample()`, `log_prob()`, `cdf()`, `icdf()`, `entropy()`
- Properties: `mean`, `variance`, `mode`, `support`

### **2. Available Distributions**
**Continuous**: Normal, Beta, Gamma, Exponential, Dirichlet, Cauchy, Laplace, StudentT, Gumbel, etc.
**Discrete**: Bernoulli, Binomial, Categorical, Poisson, Geometric, NegativeBinomial, etc.
**Multivariate**: MultivariateNormal, LowRankMultivariateNormal, Dirichlet
**Specialized**: MixtureSameFamily, TransformedDistribution, Independent, Wishart, LKJCholesky

### **3. Key Features**
- **Parameterization**: Most distributions accept either `probs` or `logits` parameters
- **Batch Support**: All distributions support batched parameters
- **Gradient Estimation**:
  - **Score Function**: Uses `sample()` + `log_prob()` (e.g., for REINFORCE)
  - **Pathwise Derivative**: Uses `rsample()` (e.g., for reparameterization trick)
- **Exponential Family**: Specialized base class for distributions in exponential family form

### **4. Transforms & Constraints**
- **Transforms**: Bijective transformations (AffineTransform, ExpTransform, SigmoidTransform, etc.)
- **Constraints**: Define valid parameter ranges (Interval, Simplex, PositiveDefinite, etc.)
- **Constraint Registry**: `transform_to()` and `biject_to()` for converting between constrained/unconstrained spaces

### **5. KL Divergence**
- Pre-registered KL divergences between many distribution pairs
- Extensible via `register_kl()` decorator

## **Common Usage Patterns**

### **REINFORCE (Policy Gradient)**
```python
probs = policy_network(state)
m = Categorical(probs)
action = m.sample()
loss = -m.log_prob(action) * reward
loss.backward()
```

### **Reparameterization Trick (VAEs)**
```python
params = policy_network(state)
m = Normal(*params)
action = m.rsample()  # Differentiable
loss = -reward
loss.backward()
```

### **Working with Constraints**
```python
# Convert constrained parameters to unconstrained space
unconstrained = torch.zeros(100, requires_grad=True)

```

## **Design Principles**
1. **Separation of concerns**: Distributions separate sampling from gradient estimation
2. **Composability**: Transforms can be chained via `TransformedDistribution`
3. **Extensibility**: Users can register new distributions, transforms, and KL divergences
4. **Performance**: Optimized for both forward passes and gradient computations

The module is essential for probabilistic modeling, reinforcement learning, variational inference, and any application requiring differentiable sampling from probability distributions.
