- Share reproducible research artifacts.

- Ensure environment-agnostic execution.

**Important Notes:**
- Relies on `pickle` (insecure—only load trusted packages).
- Modules are isolated; types from packages differ from system types.
- Best practices include avoiding global state and refactoring to minimize dependencies.

`torch.package` is ideal for creating portable, versioned ML packages that run consistently anywhere PyTorch is supported.
