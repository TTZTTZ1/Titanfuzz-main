### Key Features:

- **Sparse Layout Requirement**: Optimizes performance using sparse tensors; assumes all materialized values in the sparse gradient are non-zero.
- **Narrow Use Case**: Best suited when gradients are **natively sparse** (e.g., from certain modules). Converting dense gradients to sparse layout may negate performance benefits.

### Important Notes:
- **Not for Semantically Sparse Gradients**: If gradients are dense but contain many zeros, use standard Adam instead, unless you can avoid materializing sparse values.
- **Performance**: Efficiency gains depend on the sparsity of gradients; more masked values lead to better performance.
- **Experimental Alternative**: MaskedTensors could offer a clearer implementation but are still experimental.

### Methods:

- `zero_grad()`: Resets gradients (default sets to `None` for memory efficiency).
- `state_dict()` / `load_state_dict()`: Save and load optimizer state.
- Additional hooks for customizing state dict and step behavior.

### When to Use:
- Use with modules that **natively produce sparse gradients** (like `nn.Embedding(sparse=True)`).
- Avoid if you need to manually convert dense gradients to sparse format, as overhead may outweigh benefits.
