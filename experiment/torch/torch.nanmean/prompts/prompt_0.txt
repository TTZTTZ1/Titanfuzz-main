`torch.nanmean` computes the mean of tensor elements while ignoring NaN values. It works only with floating-point or complex data types and behaves like `torch.mean` when no NaNs are present.  

**Parameters:**
- `input`: Input tensor (float or complex).
- `dim`: Dimension(s) to reduce; if `None`, reduces all dimensions.
- `keepdim`: If `True`, retains reduced dimensions with size 1.

**Keyword arguments:**
- `dtype`: Casts input to this dtype before computation.
- `out`: Output tensor.

**Key behavior:**
- Ignores NaN values during computation.
- If all elements in a reduced dimension are NaN, the result is NaN.
- With `keepdim=True`, output dimensions match input except reduced dimensions become size 1.

**Example:**
```python
x = torch.tensor([[torch.nan, 1, 2], [1, 2, 3]])
x.nanmean()          # tensor(1.8000)
x.nanmean(dim=0)     # tensor([1.0000, 1.5000, 2.5000])
```
