`InstanceNorm1d` applies instance normalization to 2D (unbatched) or 3D (batched) inputs, normalizing each channel independently across each sample in a batch. It is commonly used for tasks like stylization on sequential or time-series data.

**Key parameters:**
- `num_features`: Number of input channels/features.
- `affine`: If `True`, adds learnable scale (γ) and shift (β) parameters per channel.
- `track_running_stats`: If `True`, tracks running mean/variance during training for evaluation (uncommon in instance norm; usually kept `False`).

**Behavior:**
- Computes mean and variance **per sample and per channel**.
- By default, uses batch statistics in both training and evaluation (unless `track_running_stats=True`).
- Similar to `LayerNorm`, but applied per channel rather than per sample, and typically without affine transformation unless specified.

**Input/Output shape:**  
`(N, C, L)` or `(C, L)` → same shape output.
