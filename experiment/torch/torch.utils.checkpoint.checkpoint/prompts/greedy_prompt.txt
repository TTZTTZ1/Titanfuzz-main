The `torch.utils.checkpoint` module provides **activation checkpointing**, a technique that trades compute for memory by recomputing intermediate activations during the backward pass instead of storing them.

### Key Functions:
- **`checkpoint(function, *args, use_reentrant=False, **kwargs)`**  
  Checkpoints a model or part of it. Use `use_reentrant=False` (recommended) for better flexibility and support for `torch.autograd.grad`, keyword arguments, and nested structures.

- **`checkpoint_sequential(functions, segments, input, use_reentrant=False, **kwargs)`**  
  Checkpoints a sequential model by splitting it into segments, reducing memory usage.

### Important Notes:
- **RNG State**: By default, the RNG state is preserved for deterministic output (e.g., with dropout). Set `preserve_rng_state=False` to improve performance if determinism isn’t required.
- **Reentrant vs. Non-reentrant**:  
  - **Non-reentrant** (`use_reentrant=False`): Records autograd graph, supports `torch.autograd.grad`, and allows early stopping of recomputation.  
  - **Reentrant** (`use_reentrant=True`): Runs under `torch.no_grad()`, has limitations (e.g., requires at least one input/output with `requires_grad=True`).
- **Selective Checkpointing**:  
  Use `create_selective_checkpoint_contexts` with a policy function to control which operations are recomputed, optimizing the compute-memory trade-off.

### Warnings:
- Ensure the `function` behaves identically in forward and backward passes to avoid incorrect gradients.
- Explicitly pass `use_reentrant` (required from PyTorch 2.9 onward).

### Example:
```python
out = checkpoint(fn, x, y, use_reentrant=False)
```
This reduces memory at the cost of recomputing `fn` during backpropagation.
