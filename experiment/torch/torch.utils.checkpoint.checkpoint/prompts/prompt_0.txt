The `torch.utils.checkpoint` module provides **activation checkpointing**, a technique that trades computation for memory by recomputing intermediate activations during the backward pass instead of storing them.

### Key Functions:

1. **`checkpoint(function, *args, **kwargs)`**  
   - **Purpose**: Checkpoints a model or part of it to reduce memory usage.  
   - **How it works**: During the forward pass, only the inputs are saved. In backward, the function is re-run to recompute tensors needed for gradients.  
   - **Important parameters**:  
     - `use_reentrant`: Must be specified explicitly. Use `False` (recommended) for more flexibility (supports `autograd.grad`, keyword arguments, etc.).  
     - `preserve_rng_state`: Controls RNG state handling for deterministic output (default `True`).  
     - `context_fn`, `determinism_check`, `debug`, `early_stop`: Additional controls for the non-reentrant variant.

2. **`checkpoint_sequential(functions, segments, input, **kwargs)`**  
   - **Purpose**: Checkpoints a sequential model by splitting it into segments.  
   - **How it works**: Each segment except the last is checkpointed, saving memory at the cost of recomputation.

### Key Notes:
- **Reentrant vs. Non-reentrant**:  
  - **Non-reentrant** (`use_reentrant=False`): Records autograd graph, supports more backward APIs, and allows early stopping of recomputation.  
  - **Reentrant** (`use_reentrant=True`): Runs under `torch.no_grad()`, has limitations (e.g., requires at least one input/output with `requires_grad=True`).  
- **RNG State**: By default, the RNG state is preserved for deterministic results (can be disabled with `preserve_rng_state=False`).  
- **Selective Checkpointing**: Use `create_selective_checkpoint_contexts` with a policy function to control which operations are recomputed.  
- **Warnings**:  
  - Ensure the function behaves identically in forward and backward passes.  
  - Explicitly pass `use_reentrant` (required since PyTorch 2.9).

### Example Usage:
```python
# Basic checkpointing
output = torch.utils.checkpoint.checkpoint(model, x, use_reentrant=False)

# Sequential model
output = torch.utils.checkpoint.checkpoint_sequential(model, chunks=3, input=x, use_reentrant=False)
```

This module is essential for training large models where memory savings outweigh the cost of extra computation.
