`torch.Tensor.resize_` is an in-place method that changes the tensor's size, preserving existing data but leaving new memory uninitialized. If the new size requires more elements, the underlying storage expands; if fewer, storage remains unchanged.

**Key points:**
- **Low-level operation**: Ignores strides and treats storage as C-contiguous (unlike `view()` or `reshape()`).
- **Memory handling**: New memory is uninitialized unless deterministic algorithms are enabled (then filled with NaN or max integer values).
- **Parameters**:
  - `sizes`: Target size (tuple or variable arguments).
  - `memory_format`: Memory layout (default: contiguous).

**Example:**
```python
x = torch.tensor([[1, 2], [3, 4], [5, 6]])
x.resize_(2, 2)  # Shape becomes (2, 2), data preserved where possible
```

**Warning**: Prefer `view()` or `reshape()` for most use cases, as `resize_` can lead to unexpected strides and memory interpretation.
