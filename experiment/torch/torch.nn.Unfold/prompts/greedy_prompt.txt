`torch.nn.Unfold` extracts sliding local blocks from a batched input tensor, commonly used for operations like custom convolutions. It flattens each kernel-sized block across spatial dimensions into columns, producing a 3D output of shape `(N, C × ∏(kernel_size), L)`, where `L` is the total number of blocks.

**Key parameters:**
- `kernel_size`: Size of sliding blocks.
- `stride`: Step size for sliding.
- `padding`: Implicit zero-padding added to input.
- `dilation`: Spacing between kernel points.

**Notes:**
- For 2D spatial inputs, this operation is equivalent to `im2col`.
- Works with 4D inputs (batched image-like tensors) only.
- Combined with `Fold`, they relate as `fold(unfold(input)) == divisor * input`, where `divisor` depends on input shape/dtype.

**Example use case:**  
Unfold + matrix multiplication + Fold can replicate a convolution operation, as shown in the documentation.
