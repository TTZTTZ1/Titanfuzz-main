**Key Parameters:**
- `kernel_size`: Size of sliding blocks.
- `stride`: Step size for sliding.
- `padding`: Implicit zero-padding added to input.
- `dilation`: Spacing between kernel points.

**Input/Output Shape:**
- **Input:** `(N, C, *)` where `*` denotes spatial dimensions.
- **Output:** `(N, C × ∏(kernel_size), L)`, with `L` as the total number of extracted blocks.

**Notes:**
- Primarily supports 4D inputs (batched images).
- Combined with matrix multiplication, it can simulate convolution.
- `Fold` and `Unfold` are not exact inverses if blocks overlap; they relate by a scaling factor (`divisor`).

**Example Use Case:**  
Useful for custom convolution implementations or patch-based processing by transforming image data into flattened block representations.
