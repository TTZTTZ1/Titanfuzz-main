**LazyModuleMixin** enables PyTorch modules to defer parameter initialization until the first forward pass, inferring shapes from input data.  

**Key points:**  
- **Purpose:** Avoids specifying dimensions like `in_features` manually; parameters start as `UninitializedParameter` and become regular `Parameter` after first forward.  
- **Usage workflow:**  
  1. Construct network with lazy modules (e.g., `LazyLinear`).  
  2. Move to desired device/dtype.  
  3. Perform a "dry run" (forward pass with dummy input) to initialize parameters.  
  4. Use normally (e.g., attach optimizer, train).  
- **Caveats:**  
  - Initialization order may differ vs non‑lazy modules, affecting random seeding.  
  - Loaded state dicts with initialized parameters persist; dry runs won’t re‑initialize them.  
- **Methods:**  
  - `has_uninitialized_params()`: Check if any parameters remain uninitialized.  
  - `initialize_parameters()`: Explicitly trigger shape inference via input.  

**Example:**  
```python
lazy_mlp = LazyMLP()  # contains LazyLinear layers
lazy_mlp = lazy_mlp.cuda()
lazy_mlp(torch.ones(10, 10).cuda())  # dry run initializes parameters
optim = torch.optim.SGD(lazy_mlp.parameters(), lr=0.01)  # now usable
```
