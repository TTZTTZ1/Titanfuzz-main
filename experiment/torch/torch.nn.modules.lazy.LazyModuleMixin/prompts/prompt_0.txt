- **Lazy Initialization**: Parameters remain as `UninitializedParameter` until the first forward call, after which they become regular `Parameter` objects.
- **Automatic Shape Inference**: Parameter shapes are derived from the first input's dimensions.
- **Convenience**: Simplifies module construction by avoiding explicit size calculations.

3. **Perform a "dry run"** by passing a sample input through the network to initialize all lazy modules.
4. **Use as normal** after initialization, including attaching optimizers.

### Important Notes:
- **Initialization Order**: Lazy modules initialize after non-lazy ones, which may affect parameter initialization order and reproducibility.
- **Serialization**: Lazy modules can be saved/loaded via `state_dict()`. They can load both initialized and uninitialized parameters, but once initialized, parameters won't be re-initialized by subsequent dry runs.

### Methods:

- `initialize_parameters()`: Explicitly triggers parameter initialization using input batch properties.

Lazy modules streamline network design when input dimensions are unknown at construction time, while maintaining full compatibility with standard PyTorch workflows after initialization.
