- **Purpose:** Adjusts learning rates by multiplying them with a factor determined by `lr_lambda` at each epoch.
- **Parameters:**
  - `optimizer`: The wrapped optimizer.
  - `lr_lambda`: A function (or list of functions) that takes the epoch index as input and returns a multiplicative factor.
  - `last_epoch`: The index of the last epoch (default: -1, which sets initial learning rate as the base LR).
- **Usage:** After each epoch, call `scheduler.step()` to update the learning rates.
- **Example:**
  ```python

  scheduler = MultiplicativeLR(optimizer, lr_lambda=lmbda)
  for epoch in range(100):
      train(...)
      validate(...)
      scheduler.step()
  ```

**Important Methods:**
- `get_last_lr()`: Returns the most recent learning rates for each parameter group.
- `get_lr()`: Computes the next learning rates (used internally by `step()`).
- `state_dict()` / `load_state_dict()`: Save and load scheduler state.

**Note:** Call `scheduler.step()` after the optimizer’s `step()` during training.
