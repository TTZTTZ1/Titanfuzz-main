The `torch.optim.SGD` class implements stochastic gradient descent (SGD) with optional momentum, weight decay, and Nesterov acceleration. It is a versatile optimizer widely used for training neural networks.

**Key Features:**
- **Basic SGD:** Updates parameters using gradients scaled by a learning rate.
- **Momentum:** Accumulates a velocity buffer to smooth updates and accelerate convergence.
- **Nesterov Momentum:** A variant that applies momentum before the gradient step for improved stability.
- **Weight Decay:** Applies L2 regularization to prevent overfitting.
- **Maximization:** Supports gradient ascent by setting `maximize=True`.

**Important Implementation Notes:**
- PyTorch's momentum implementation differs from some frameworks: the learning rate is applied to the velocity term *after* momentum accumulation, not before.
- The momentum buffer is initialized with the first gradient (not zeros), so dampening is applied starting from the second step.

**Performance Options:**
- `foreach`: Uses a faster, memory-intensive vectorized implementation (default on CUDA if unspecified).
- `fused`: Uses an even faster fused kernel for supported dtypes (requires explicit enabling).

**Common Workflow:**
1. Initialize with model parameters and hyperparameters (lr, momentum, etc.).
2. Call `zero_grad()` to clear gradients.
3. Compute loss and call `backward()`.
4. Call `step()` to update parameters.

**Additional Functionality:**
- Supports parameter groups for different hyperparameters per layer.
- Provides hooks for customizing state saving/loading and step execution.
- Compatible with PyTorch's learning rate schedulers.

**Typical Use Case:**
