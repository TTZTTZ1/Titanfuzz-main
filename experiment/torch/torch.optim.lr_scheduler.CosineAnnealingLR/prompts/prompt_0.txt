The `CosineAnnealingLR` learning rate scheduler in PyTorch adjusts the learning rate according to a cosine annealing schedule, which gradually reduces the learning rate from its initial value to a specified minimum over a defined number of steps.

**Key Features:**
- **Schedule**: The learning rate follows a cosine curve, decreasing smoothly from the initial learning rate (`eta_max`) to a minimum (`eta_min`) over `T_max` steps.
- **Formula**:  
  \(\eta_t = \eta_{\min} + \frac{1}{2}(\eta_{\max} - \eta_{\min}) \left(1 + \cos\left(\frac{T_{cur} \pi}{T_{\max}}\right)\right)\)  
  where \(T_{cur}\) is the current step count.
- **No Restarts**: Unlike SGDR (which includes warm restarts), this implementation performs monotonic cosine annealing without restarts.

**Parameters:**
- `optimizer`: The optimizer to wrap.
- `T_max`: Maximum number of steps (epochs/iterations) in a cycle.
- `eta_min`: Minimum learning rate (default: 0).
- `last_epoch`: The index of the last epoch (default: -1).

**Usage Example:**
```python
scheduler = CosineAnnealingLR(optimizer, T_max=num_epochs)
for epoch in range(num_epochs):
    train(...)
    validate(...)
    scheduler.step()  # Update learning rate after each epoch
```

**Important Notes:**
- Call `scheduler.step()` after each epoch (or optimizer update) to adjust the learning rate.
- Use `get_last_lr()` to retrieve the current learning rate(s).
- The scheduler does **not** include periodic restarts; for restarts, use `CosineAnnealingWarmRestarts`.

This scheduler is useful for training deep learning models where a smooth, gradual reduction in learning rate can help convergence.
