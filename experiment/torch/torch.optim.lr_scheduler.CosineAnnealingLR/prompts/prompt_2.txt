**CosineAnnealingLR** is a PyTorch learning rate scheduler that adjusts the learning rate according to a cosine annealing schedule, gradually decreasing it from an initial value to a specified minimum over a fixed number of steps.

### Key Features:
- **Schedule**: The learning rate follows a cosine curve, smoothly decreasing from the initial value (`base_lr`) to `eta_min` over `T_max` steps.
- **Formula**:  
  \( \eta_t = \eta_{\min} + \frac{1}{2}(\eta_{\max} - \eta_{\min}) \left(1 + \cos\left(\frac{T_{\text{cur}} \pi}{T_{\max}}\right)\right) \)  
  where \( T_{\text{cur}} \) is the current step count.
- **No Restarts**: Unlike SGDR, this implementation does **not** include periodic restarts; \( T_{\text{cur}} \) increases monotonically.

### Parameters:
- `optimizer`: The optimizer to wrap.
- `T_max`: Maximum number of steps (epochs/iterations) in a cycle.
- `eta_min`: Minimum learning rate (default: 0).
- `last_epoch`: The index of the last epoch (default: -1).

### Usage Example:
```python
scheduler = CosineAnnealingLR(optimizer, T_max=num_epochs)
for epoch in range(num_epochs):
    train(...)
    validate(...)
    scheduler.step()  # Update learning rate after each epoch
```

### Important Notes:
- Call `scheduler.step()` **after** `optimizer.step()`.
- Use `get_last_lr()` to retrieve the current learning rate(s).
- The scheduler provides a smooth decay, useful for fine-tuning or training over many epochs without abrupt LR changes.

This scheduler is ideal for scenarios where a gradual, smooth reduction in learning rate is desired, such as in long training runs or when fine-tuning models.
