The `torch.nn.Embedding` module is a lookup table that maps integer indices to dense vectors of fixed size. It is commonly used for word embeddings in NLP tasks.

**Key Features:**
- Stores embeddings for a fixed dictionary size (`num_embeddings`) with each embedding of dimension `embedding_dim`.
- Input is a tensor of indices; output is the corresponding embedding vectors.
- Supports optional padding index (`padding_idx`) to ignore gradient updates for specific indices.
- Can renormalize embeddings exceeding a specified `max_norm`.
- Includes a class method `from_pretrained()` to initialize embeddings from a pre-existing tensor.

**Usage Example:**
```python

input = torch.LongTensor([[1, 2], [3, 4]])
output = embedding(input)  # shape: (2, 2, 3)
```

**Important Notes:**
- Only certain optimizers (e.g., `optim.SGD`, `optim.SparseAdam`) support sparse gradients when `sparse=True`.
- When `max_norm` is set, the weight tensor is modified in-place during the forward pass, which may affect gradient computation.
