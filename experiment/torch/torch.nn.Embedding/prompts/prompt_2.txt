The `torch.nn.Embedding` module is a lookup table that maps integer indices to dense vectors of fixed size. It is commonly used for word embeddings in NLP tasks.

**Key Features:**
- Stores embeddings for a fixed vocabulary size (`num_embeddings`) with each embedding of dimension `embedding_dim`.
- Input is a tensor of indices; output is the corresponding embedding vectors.
- Supports optional padding index (`padding_idx`) to ignore gradient updates for a specific index (e.g., for padding tokens).
- Includes options for gradient scaling by frequency (`scale_grad_by_freq`), sparse gradients (`sparse`), and renormalizing embeddings exceeding a maximum norm (`max_norm`).

**Initialization:**
- Weights are initialized from \(\mathcal{N}(0, 1)\).
- Can also be created from pretrained embeddings using `from_pretrained()`.

**Usage Example:**
```python
embedding = nn.Embedding(10, 3)  # 10 embeddings of size 3
input = torch.LongTensor([[1, 2], [3, 4]])
output = embedding(input)  # Shape: (2, 2, 3)
```

**Important Notes:**
- When `max_norm` is used, the weight tensor is modified in-place during forward passes, which may affect gradient computation.
- Sparse gradients are only supported by specific optimizers (e.g., `optim.SGD`, `optim.SparseAdam`).
