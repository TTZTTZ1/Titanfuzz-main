The `torch.nn.Embedding` module is a lookup table that maps integer indices to dense vectors of fixed size. It is commonly used for word embeddings in NLP tasks.

**Key Features:**
- Stores embeddings for a fixed vocabulary size (`num_embeddings`) with each embedding of dimension `embedding_dim`.
- Input is a tensor of indices; output is the corresponding embedding vectors.
- Supports optional padding index (`padding_idx`) to ignore gradient updates for a specific index.
- Can renormalize embeddings exceeding a specified maximum norm (`max_norm`).
- Supports sparse gradients and gradient scaling by token frequency.

**Important Notes:**
- Only certain optimizers (e.g., `SGD`, `SparseAdam`, `Adagrad`) support sparse gradients.
- When `max_norm` is used, the weight tensor is modified in-place during the forward pass, which may affect gradient computation if the weight is used elsewhere.

**Class Method:**
- `from_pretrained()`: Creates an embedding layer from a pre-existing weight tensor, with an option to freeze the embeddings.

**Example Usage:**
```python
embedding = nn.Embedding(10, 3)  # 10 embeddings of size 3
input = torch.LongTensor([[1, 2], [3, 4]])
output = embedding(input)  # shape: (2, 2, 3)
```
