The `torch.Tensor.to()` method converts a tensor’s data type (dtype), device (e.g., CPU/GPU), or both. It returns a new tensor with the specified properties unless the tensor already matches them, in which case the original tensor is returned (unless `copy=True`).

**Key behaviors:**
- If the target dtype is an integer type and the original tensor requires gradients (`requires_grad=True`), the returned tensor will have `requires_grad=False`.
- Converting floating-point values to integer types truncates the fractional part (C++ conversion rules apply).
- The `non_blocking` option can be used for asynchronous transfers (e.g., CPU → GPU).
- The `copy` flag forces creation of a new tensor even if no conversion is needed.

**Common usage patterns:**
1. **Convert dtype only:**  
   `tensor.to(dtype)`
2. **Convert device and optionally dtype:**  
   `tensor.to(device, dtype)`
3. **Match another tensor’s properties:**  
   `tensor.to(other_tensor)`

**Example:**
```python
x = torch.randn(2, 2)                     # float32, CPU
x.to(torch.float64)                       # Change dtype
x.to('cuda:0', dtype=torch.float64)       # Change device and dtype
x.to(other_tensor)                        # Match other tensor's dtype/device
```
