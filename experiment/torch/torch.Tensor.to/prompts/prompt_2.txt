**Key behaviors:**
- If the target dtype is an integer type and the original tensor requires gradients (`requires_grad=True`), the returned tensor will have `requires_grad=False`.
- Converting floating-point values to integers truncates the fractional part (undefined if truncated values overflow).
- The `non_blocking` flag enables asynchronous conversion when possible (use with caution).

**Three main calling signatures:**
1. `tensor.to(dtype, ...)` – Changes only the data type.
2. `tensor.to(device, dtype=None, ...)` – Changes device and optionally dtype.
3. `tensor.to(other_tensor, ...)` – Matches the dtype and device of another tensor.

**Parameters:**
- `dtype` (torch.dtype): Target data type.
- `device` (torch.device): Target device (e.g., 'cuda', 'cpu').
- `non_blocking` (bool): If True, attempts asynchronous conversion.
- `copy` (bool): If True, forces a new tensor even if no conversion is needed.
- `memory_format` (torch.memory_format): Desired memory layout (default: preserve).
