**Key features:**
- **Three calling signatures:**
  1. `.to(dtype, ...)` – changes only the data type.
  2. `.to(device, dtype=None, ...)` – changes device and optionally dtype.
  3. `.to(other_tensor, ...)` – copies dtype and device from another tensor.
- **Automatic inference:** If dtype or device are omitted, they are inferred from the arguments or kept as the original.
- **No-op behavior:** If the tensor already has the target dtype and device, the same tensor is returned (unless `copy=True`).
- **Gradient handling:** Converting to an integer dtype will disable gradients (`requires_grad=False`).
- **Optional parameters:** `non_blocking` (for async transfers), `copy` (force a new tensor), and `memory_format` (control memory layout).

**Example use cases:**
- Move a tensor to GPU: `tensor.to('cuda')`
- Change precision: `tensor.to(torch.float16)`
- Match another tensor's settings: `tensor.to(other_tensor)`
