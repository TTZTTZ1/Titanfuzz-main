`torch.stft` performs a Short-Time Fourier Transform (STFT) on an input signal, analyzing its frequency content over time using sliding windows.

**Key Parameters:**
- **input**: 1D time series or 2D batch of sequences
- **n_fft**: Size of the Fourier transform (window length after padding)
- **hop_length**: Step size between windows (default: `n_fft // 4`)
- **window**: Window function tensor (e.g., `torch.hann_window()`); required from version 2.1 onward
- **center**: Pad input so frames are centered (default: `True`)
- **return_complex**: Must be explicitly set; `True` returns complex tensor, `False` returns real tensor with separate real/imaginary dimensions (deprecated)

**Output Shape:**
- Complex output: `(*, N, T)` where `N` is frequency bins, `T` is time frames
- Real output: `(*, N, T, 2)` with real/imaginary components

**Important Notes:**
- Real inputs with `onesided=True` (default) return only unique frequencies due to conjugate symmetry
- From PyTorch 2.0, `return_complex=True` is strongly preferred for real inputs
- Window specification is required in future versions to avoid rectangular window artifacts
---GENERAL SUMMARY END---
