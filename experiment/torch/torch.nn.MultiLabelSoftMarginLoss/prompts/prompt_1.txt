**Summary: `torch.nn.MultiLabelSoftMarginLoss`**

**Purpose:**  
A loss function for multi-label classification problems, where each sample can belong to multiple classes simultaneously. It optimizes a one-versus-all binary classification loss using max-entropy (logistic loss).

**Key Formula:**  
For each class \( i \) in a sample:

\[
\text{loss}(x, y) = -\frac{1}{C} \sum_i \left[ y[i] \cdot \log(\sigma(x[i])) + (1 - y[i]) \cdot \log(1 - \sigma(x[i])) \right]
\]
where \( \sigma(x[i]) = \frac{1}{1 + \exp(-x[i])} \) (sigmoid), \( C \) is the number of classes, and \( y[i] \in \{0, 1\} \).

**Parameters:**
- `weight` (Tensor, optional): Per-class rescaling weight (size \( C \)).
- `reduction` (str): Specifies output reduction: `'none'`, `'mean'`, or `'sum'` (default: `'mean'`).  
  (Note: `size_average` and `reduce` are deprecated.)

**Input/Output Shapes:**
- **Input:** \((N, C)\), where \( N \) = batch size, \( C \) = number of classes.
- **Target:** \((N, C)\), with binary labels (0 or 1) of same shape as input.
- **Output:** Scalar if reduction is `'mean'` or `'sum'`; otherwise \((N)\) if `'none'`.

**Usage:**  
Suitable for tasks where multiple independent binary classifications are performed per sample (e.g., tagging). The loss applies sigmoid activation internally, so input logits do not require prior normalization.
