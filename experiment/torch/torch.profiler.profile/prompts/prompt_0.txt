The `torch.profiler` module provides tools for profiling PyTorch model performance during training and inference. It enables detailed collection of metrics to analyze operator costs, input shapes, stack traces, device kernel activity, and execution traces.

### Key Components:
- **`profile` context manager**: Main entry point for profiling, supporting configurable activities (CPU, CUDA, XPU, etc.), schedules, and trace handlers.
- **`schedule()`**: Creates a profiling schedule to control when profiling is active (e.g., wait, warmup, active phases).
- **`tensorboard_trace_handler()`**: Outputs traces for visualization in TensorBoard.
- **`_KinetoProfile`**: Lower-level profiler wrapping autograd events.
- **`ProfilerAction` & `ProfilerActivity`**: Enums for controlling profiling behavior and device activities.

### Core Features:
- **Activity Tracking**: Profile operations on CPU, CUDA, and other accelerators.
- **Advanced Options**: Record input shapes, memory allocation, stack traces, FLOPs estimation, and module hierarchies.
- **Trace Export**: Export results in Chrome JSON format, memory timelines, and stack traces.
- **Dynamic Control**: Toggle activity collection during profiling with `toggle_collection_dynamic()`.
- **Scheduled Profiling**: Profile specific iterations in long-running jobs to reduce overhead.
- **Execution Trace Integration**: Supports PyTorch Execution Trace Observers for graph-based workload representation.

### Usage Example:
```python
with torch.profiler.profile(
    activities=[ProfilerActivity.CPU, ProfilerActivity.CUDA],
    schedule=torch.profiler.schedule(wait=1, warmup=1, active=2, repeat=1),
    on_trace_ready=torch.profiler.tensorboard_trace_handler('./log')
) as prof:
    for step in range(total_steps):
        training_iteration()
        prof.step()
```

### Additional Utilities:
- **Intel ITT APIs**: `itt.mark()`, `itt.range_push()`, and `itt.range_pop()` for manual instrumentation.

The profiler helps identify performance bottlenecks, optimize models, and visualize execution patterns, replacing the legacy `torch.autograd.profiler`.
