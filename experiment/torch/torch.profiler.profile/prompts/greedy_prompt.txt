# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
The PyTorch Profiler is a performance analysis tool for collecting metrics during model training and inference. It helps identify expensive operations, analyze input shapes, examine stack traces, and visualize execution traces.

**Key Components:**

1. **`torch.profiler.profile`** – Main context manager for profiling:
   - Supports CPU, CUDA, and XPU activities.
   - Optional scheduling for long-running jobs.
   - Can export traces for TensorBoard or Chrome.

2. **`_KinetoProfile`** – Lower-level profiler wrapper (experimental).

3. **`ProfilerAction` & `schedule()`** – Control profiling cycles (wait, warmup, active phases).

4. **`tensorboard_trace_handler()`** – Exports traces for TensorBoard visualization.

**Common Usage:**
```python
with torch.profiler.profile(activities=[ProfilerActivity.CPU, ProfilerActivity.CUDA]) as prof:
    # Code to profile
    model(input)
print(prof.key_averages().table(sort_by="cuda_time_total"))
```

**Features:**
- Track operator shapes, memory usage, and FLOPs.
- Record stack traces and module hierarchies.
- Export results as Chrome traces, memory timelines, or stack files.
- Integrate with Intel ITT for low-level instrumentation.

**Note:** Legacy `torch.autograd.profiler` is deprecated; use this API instead.
---GENERAL SUMMARY END---
