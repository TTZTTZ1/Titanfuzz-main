**OneCycleLR** is a PyTorch learning rate scheduler that implements the 1cycle policy, which dynamically adjusts the learning rate (and optionally momentum) over a training cycle to achieve faster convergence.

### **Key Features:**
- **Cycle Structure**: Learning rate starts low, increases to a maximum (`max_lr`), then decreases to a minimum (far below the initial rate).
- **Per-Batch Updates**: Must call `scheduler.step()` after each batch (not epoch).
- **Momentum Cycling**: If enabled (`cycle_momentum=True`), momentum varies inversely with the learning rate between `base_momentum` and `max_momentum`.
- **Total Steps**: Defined either directly via `total_steps` or inferred from `epochs × steps_per_epoch`.

### **Core Parameters:**
- `max_lr`: Peak learning rate in the cycle.
- `div_factor`: Initial learning rate = `max_lr / div_factor` (default 25).
- `final_div_factor`: Minimum learning rate = initial learning rate / `final_div_factor` (default 1e4).
- `pct_start`: Proportion of cycle spent increasing LR (default 30%).
- `anneal_strategy`: `'cos'` (cosine) or `'linear'` annealing.
- `three_phase`: If `True`, uses three phases (as in original paper); default `False` uses two phases (fastai style).

### **Usage Example:**
```python
optimizer = torch.optim.SGD(model.parameters(), lr=0.001)
scheduler = OneCycleLR(
    optimizer, 
    max_lr=0.01, 
    epochs=10, 
    steps_per_epoch=len(data_loader)
)

for epoch in range(10):
    for batch in data_loader:
        loss = train_batch(...)
        optimizer.step()
        scheduler.step()  # Update LR after each batch
```

### **Important Notes:**
- **Not chainable** with other schedulers.
- Use `get_last_lr()` to inspect current learning rates.
- Resume training via `last_epoch` parameter (counts batches, not epochs).

This scheduler is designed to accelerate training by using high learning rates briefly, often leading to super-convergence.
