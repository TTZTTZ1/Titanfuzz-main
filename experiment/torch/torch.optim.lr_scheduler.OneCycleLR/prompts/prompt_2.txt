The `OneCycleLR` scheduler implements the 1cycle learning rate policy, which dynamically adjusts the learning rate and optionally momentum over a training cycle.  

**Key features:**  
- **Learning rate schedule**: Starts from `max_lr/div_factor`, increases to `max_lr`, then decreases to `min_lr = initial_lr/final_div_factor`.  
- **Cycle phases**: By default uses two phases (ascent then descent), but can be set to three phases with `three_phase=True`.  
- **Momentum cycling**: If `cycle_momentum=True`, momentum varies inversely to the learning rate between `base_momentum` and `max_momentum`.  
- **Step-based updates**: Must call `scheduler.step()` after each batch, not each epoch.  

**Required parameters:**  
Provide either `total_steps` directly, or specify `epochs` and `steps_per_epoch` to infer total steps.  

**Example usage:**  
```python
scheduler = OneCycleLR(optimizer, max_lr=0.01, 
                       steps_per_epoch=len(data_loader), epochs=10)
for epoch in range(10):
    for batch in data_loader:
        train_batch(...)
        optimizer.step()
        scheduler.step()
```  

**Note:** This scheduler is not chainable and is designed for single-cycle training.
