**OneCycleLR** is a PyTorch learning rate scheduler that implements the 1cycle policy, which rapidly increases the learning rate to a maximum, then gradually decreases it to a minimum over a single training cycle.

### Key Features:
- **Learning Rate Schedule**: Starts from `max_lr/div_factor`, rises to `max_lr` over `pct_start` of the cycle, then decays to `min_lr = initial_lr/final_div_factor`.
- **Momentum Cycling** (optional): When `cycle_momentum=True`, momentum varies inversely to the learning rate between `base_momentum` and `max_momentum`.
- **Batch-Level Updates**: Must call `scheduler.step()` after each batch, not each epoch.
- **Cycle Length**: Specify either `total_steps` directly, or provide `epochs` and `steps_per_epoch` to infer it.

### Main Parameters:
- `max_lr`: Peak learning rate in the cycle.
- `total_steps` / (`epochs` + `steps_per_epoch`): Defines cycle duration.
- `pct_start`: Proportion of cycle spent increasing LR (default 0.3).
- `anneal_strategy`: `'cos'` (cosine) or `'linear'` annealing.
- `div_factor` / `final_div_factor`: Determine initial and final LRs.
- `three_phase`: If `True`, uses a symmetric three-phase schedule instead of the default two-phase.

### Typical Usage:
```python
scheduler = OneCycleLR(optimizer, max_lr=0.01, 
                       steps_per_epoch=len(dataloader), epochs=10)
for epoch in range(10):
    for batch in dataloader:
        train_batch(...)
        optimizer.step()
        scheduler.step()  # Update after each batch
```

**Note**: This scheduler is not chainable and is designed for single-cycle training regimes aimed at achieving faster convergence.
