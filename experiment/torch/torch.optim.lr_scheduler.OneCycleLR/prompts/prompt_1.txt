# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
The `OneCycleLR` scheduler implements the 1cycle learning rate policy, which adjusts the learning rate (and optionally momentum) after each batch rather than each epoch. It increases the learning rate from an initial value to a maximum, then decreases it to a minimum lower than the starting point.

**Key Features:**
- **Cycle Phases:** By default uses two phases (ascent then descent), but can be set to three phases with `three_phase=True`.
- **Total Steps:** Must be specified either directly via `total_steps` or derived from `epochs × steps_per_epoch`.
- **Momentum Cycling:** When `cycle_momentum=True`, momentum varies inversely to the learning rate between `base_momentum` and `max_momentum`.
- **Annealing:** Supports cosine (`'cos'`) or linear (`'linear'`) annealing strategies.
- **Learning Rate Range:**  
  - Initial LR = `max_lr / div_factor`  
  - Minimum LR = initial LR / `final_div_factor`

**Usage Example:**
```python
scheduler = OneCycleLR(optimizer, max_lr=0.01, steps_per_epoch=len(data_loader), epochs=10)
for epoch in range(10):
    for batch in data_loader:
        train_batch(...)
        optimizer.step()
        scheduler.step()  # Call after each batch
```

**Important Notes:**
- Not chainable with other schedulers.
- `step()` must be called after each batch (not each epoch).
- Use `get_last_lr()` to inspect current learning rates.
---GENERAL SUMMARY END---
