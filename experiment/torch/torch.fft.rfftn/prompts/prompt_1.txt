`torch.fft.rfftn` computes the **N-dimensional discrete Fourier transform (DFT) for real-valued input tensors**. Since the FFT of a real signal is Hermitian-symmetric, this function omits the redundant negative frequencies in the last dimension, returning a compact, complex-valued output.

**Key Parameters:**
- `input`: Real-valued input tensor.
- `s`: Optional tuple specifying the signal size in each transformed dimension (for padding/trimming).
- `dim`: Dimensions to transform (defaults to all dimensions, or last `len(s)` dimensions if `s` is provided).
- `norm`: Normalization mode (`"forward"`, `"backward"`, or `"ortho"`).

**Example:**
```python
t = torch.rand(10, 10)
rfftn = torch.fft.rfftn(t)  # Output shape: [10, 6]
```

This function is more efficient than `fftn` for real data, as it reduces the last dimension by approximately half while preserving all non-redundant frequency components.
