`torch.nn.Conv1d` applies a 1D convolution over an input signal. It is commonly used for processing sequential data such as time series or audio.

**Key Parameters:**
- `in_channels`: Number of input channels
- `out_channels`: Number of output channels
- `kernel_size`: Size of the convolution kernel
- `stride`: Step size for convolution (default: 1)
- `padding`: Amount of padding (default: 0; can be 'valid' or 'same')
- `dilation`: Spacing between kernel elements (default: 1)
- `groups`: Controls input-output connections (default: 1)
- `bias`: Whether to add a learnable bias (default: True)

**Output Size Formula:**
\[
L_{out} = \left\lfloor\frac{L_{in} + 2 \times \text{padding} - \text{dilation} \times (\text{kernel\_size} - 1) - 1}{\text{stride}} + 1\right\rfloor
\]

**Key Features:**
- Supports depthwise convolution when `groups = in_channels`
- Works with complex data types
- Multiple padding modes: 'zeros', 'reflect', 'replicate', 'circular'
- 'same' padding maintains input length when stride=1

**Example:**
```python
conv = nn.Conv1d(16, 33, 3, stride=2)  # 16 input channels, 33 output channels, kernel size 3
input = torch.randn(20, 16, 50)        # (batch_size, channels, sequence_length)
output = conv(input)                   # Output shape: (20, 33, 24)
```
