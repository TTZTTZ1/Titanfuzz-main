- **No autograd history:** Equivalent to `t.detach().clone()` for an existing tensor `t`.

- **Flexible device placement:** Uses the input tensor's device if provided, otherwise defaults to the current device.
- **Common use cases:** Converting Python data to tensors, creating scalar tensors, or explicitly copying tensors without gradients.

**Alternatives to consider:**
- Use `torch.as_tensor()` to preserve autograd and avoid copies when possible.
- Use `torch.from_numpy()` to share memory with NumPy arrays.
- For existing tensors, prefer `.clone()`, `.detach()`, or `.requires_grad_()` for clearer intent.

