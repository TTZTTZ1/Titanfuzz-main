**CyclicLR** is a PyTorch learning rate scheduler that implements **Cyclical Learning Rates (CLR)**, cycling the learning rate between a lower bound (`base_lr`) and an upper bound (`max_lr`) over training iterations.

### Key Features:
- **Cycles per batch**: Adjusts LR after each batch (not epoch).
- **Built-in policies**:  
  - `triangular`: Linear up/down cycle.  
  - `triangular2`: Halves amplitude each cycle.  
  - `exp_range`: Scales amplitude exponentially by `gamma^cycle_iterations`.
- **Custom scaling**: Optional `scale_fn` for user-defined cycle shapes.
- **Momentum cycling**: If `cycle_momentum=True`, momentum varies inversely to LR between `base_momentum` and `max_momentum`.

### Main Parameters:
- `optimizer`: Wrapped optimizer (e.g., SGD).
- `base_lr` / `max_lr`: Lower/upper LR bounds.
- `step_size_up`: Iterations for increasing half of cycle (default: 2000).
- `step_size_down`: Iterations for decreasing half (defaults to `step_size_up`).
- `mode`: Policy type (`'triangular'`, `'triangular2'`, `'exp_range'`).
- `cycle_momentum`: Whether to cycle momentum (default: `True`).

### Usage Example:
```python
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)
scheduler = torch.optim.lr_scheduler.CyclicLR(
    optimizer, 
    base_lr=0.01, 
    max_lr=0.1,
    step_size_up=10
)

for batch in data_loader:
    train_batch(...)
    optimizer.step()
    scheduler.step()  # Update LR after each batch
```

### Notes:
- Use `scheduler.step()` **after each batch**.
- `last_epoch` tracks batch count, not epoch count.
- Momentum cycling is applied automatically when enabled.

This scheduler helps escape poor local minima and can reduce the need for extensive LR tuning.
