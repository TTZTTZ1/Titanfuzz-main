`CyclicLR` is a PyTorch learning rate scheduler that implements **Cyclical Learning Rates (CLR)**, where the learning rate oscillates between a lower bound (`base_lr`) and an upper bound (`max_lr`) over training iterations.  

**Key features:**  
- **Cycle behavior:** The learning rate increases linearly over `step_size_up` iterations, then decreases over `step_size_down` iterations (defaults to `step_size_up`).  
- **Built-in policies:**  
  - `triangular`: Basic triangular cycle.  
  - `triangular2`: Halves the amplitude each cycle.  
  - `exp_range`: Scales amplitude by `gamma^(cycle iterations)`.  
- **Custom scaling:** Optional `scale_fn` allows user-defined scaling.  
- **Momentum cycling:** If `cycle_momentum=True`, momentum varies inversely to the learning rate between `base_momentum` and `max_momentum`.  
- **Per-batch updates:** Must call `scheduler.step()` after each batch (not epoch).  

**Typical usage:**  
```python
scheduler = CyclicLR(optimizer, base_lr=0.001, max_lr=0.1, step_size_up=2000)
for batch in data_loader:
    train_batch(...)
    optimizer.step()
    scheduler.step()
```
