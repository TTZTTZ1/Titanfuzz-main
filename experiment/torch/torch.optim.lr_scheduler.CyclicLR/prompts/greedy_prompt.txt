**CyclicLR** is a PyTorch learning rate scheduler that implements a cyclical learning rate policy, cycling the learning rate between a lower bound (`base_lr`) and an upper bound (`max_lr`) over training iterations.

**Key Features:**
- **Cycles per batch**: Updates the learning rate after each batch (not epoch).
- **Built-in policies**: `triangular` (linear cycle), `triangular2` (amplitude halves each cycle), and `exp_range` (exponential scaling).
- **Custom scaling**: Optional user-defined `scale_fn` for custom cycle shapes.
- **Momentum cycling**: Optionally cycles momentum inversely to the learning rate between `base_momentum` and `max_momentum`.
- **Flexible cycle shape**: Asymmetric cycles via `step_size_up` and `step_size_down`.

**Basic Usage:**
```python
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)
scheduler = torch.optim.lr_scheduler.CyclicLR(
    optimizer,
    base_lr=0.01,      # lower bound
    max_lr=0.1,        # upper bound
    step_size_up=10    # iterations to increase from base_lr to max_lr
)

# Call after each batch
for batch in data_loader:
    train_batch(...)
    optimizer.step()
    scheduler.step()
```

**Key Parameters:**
- `base_lr`/`max_lr`: Cycle boundaries
- `step_size_up`/`step_size_down`: Steps for increasing/decreasing halves
- `mode`: Cycle policy (`'triangular'`, `'triangular2'`, `'exp_range'`)
- `cycle_momentum`: Whether to cycle momentum inversely to LR
- `last_epoch`: Batch index for resuming training

**Note**: Since it updates per batch, `last_epoch` tracks total batches, not epochs.
