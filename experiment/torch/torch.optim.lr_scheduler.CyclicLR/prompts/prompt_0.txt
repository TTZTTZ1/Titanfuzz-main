---GENERAL SUMMARY START---
The `CyclicLR` scheduler implements a cyclical learning rate policy that varies the learning rate between a lower (`base_lr`) and upper (`max_lr`) boundary over training iterations. It operates on a per-batch basis, requiring `step()` to be called after each batch update.

**Key Features:**
- **Cycles:** Learning rate oscillates between `base_lr` and `max_lr` with a configurable cycle length (via `step_size_up` and `step_size_down`).
- **Built-in Modes:** Supports `triangular` (basic cycle), `triangular2` (amplitude halves each cycle), and `exp_range` (exponential amplitude decay).
- **Custom Scaling:** Allows a user-defined `scale_fn` for flexible cycle shaping.
- **Momentum Cycling:** Optionally cycles momentum inversely to the learning rate between `base_momentum` and `max_momentum` when `cycle_momentum=True`.
- **Usage:** Must be stepped after each batch, not each epoch.

**Common Use Case:**
```python
scheduler = CyclicLR(optimizer, base_lr=0.001, max_lr=0.01, step_size_up=2000)
for batch in data_loader:
    train_batch(...)
    optimizer.step()
    scheduler.step()  # Update LR after each batch
```

This approach can help models converge faster and escape local minima by periodically varying the learning rate.
