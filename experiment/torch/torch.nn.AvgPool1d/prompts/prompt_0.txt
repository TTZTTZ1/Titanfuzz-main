`AvgPool1d` performs 1D average pooling over an input signal, reducing its length by computing the average within sliding windows.

**Key parameters:**
- `kernel_size`: size of the pooling window
- `stride`: step size for sliding the window (defaults to `kernel_size`)
- `padding`: zeros added to both sides of the input
- `ceil_mode`: uses ceil (instead of floor) for output length calculation
- `count_include_pad`: includes padding in average computation when True

**Input/Output shape:**
- Input: `(N, C, L_in)` or `(C, L_in)`
- Output: `(N, C, L_out)` or `(C, L_out)`
- Output length formula:  
  `L_out = floor((L_in + 2×padding - kernel_size)/stride + 1)`  
  (adjusted when `ceil_mode=True`)

**Example:**
```python
m = nn.AvgPool1d(3, stride=2)
m(torch.tensor([[[1., 2, 3, 4, 5, 6, 7]]]))  # Output: [[[2., 4., 6.]]]
```
This computes averages over windows of size 3 with stride 2:  
(1+2+3)/3=2, (3+4+5)/3=4, (5+6+7)/3=6.
