The `torch.distributions` package provides a comprehensive framework for working with probability distributions in PyTorch, enabling stochastic computation graphs and gradient estimation for optimization. Here's a concise summary of its key aspects:

## Core Concepts
- **Parameterizable Distributions**: Includes common distributions (Normal, Bernoulli, Categorical, etc.) with parameterization via `probs`/`logits` or location/scale parameters.
- **Gradient Estimation**: Supports two main methods for backpropagation through random samples:
  1. **Score Function/REINFORCE**: Uses `sample()` and `log_prob()` for policy gradient methods
  2. **Pathwise Derivative/Reparameterization**: Uses `rsample()` for differentiable sampling (e.g., in VAEs)

## Key Components

### Distribution Base Class
- Abstract base class with methods: `sample()`, `rsample()`, `log_prob()`, `entropy()`, `cdf()`, `icdf()`
- Properties: `mean`, `variance`, `stddev`, `mode`, `support`
- Batch and event shape handling

### Available Distributions
- **Univariate**: Bernoulli, Beta, Binomial, Categorical, Cauchy, Exponential, Gamma, Geometric, Laplace, Normal, Poisson, Uniform, etc.
- **Multivariate**: Dirichlet, Multinomial, MultivariateNormal, LowRankMultivariateNormal, Wishart
- **Transformed**: LogNormal, HalfNormal, HalfCauchy, Gumbel, Weibull, etc.
- **Specialized**: LKJCholesky (correlation matrices), MixtureSameFamily, Relaxed distributions (Gumbel-Softmax)

### Transforms
- Bijective transformations for reparameterization: `AffineTransform`, `ExpTransform`, `SigmoidTransform`, `SoftmaxTransform`, `StickBreakingTransform`, etc.
- Used in `TransformedDistribution` to create complex distributions from simple ones

### Constraints & Registry
- **Constraints**: Define valid parameter ranges (simplex, positive, interval, etc.)
- **Registry System**: `transform_to()` and `biject_to()` for mapping between constrained and unconstrained spaces
- Enables unconstrained optimization of constrained parameters

### KL Divergence
- `kl_divergence()` computes KL divergence between distributions
- Extensive registry of pairwise KL implementations

## Common Usage Patterns

### REINFORCE (Score Function)
```python
probs = policy_network(state)
m = Categorical(probs)
action = m.sample()
loss = -m.log_prob(action) * reward
loss.backward()
```

### Reparameterization Trick
```python

```

### Distribution Creation
```python

```
