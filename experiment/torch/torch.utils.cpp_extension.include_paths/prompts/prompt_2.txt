The `torch.utils.cpp_extension` module provides utilities for building PyTorch C++ and CUDA extensions, supporting both ahead-of-time (via `setup.py`) and just-in-time (JIT) compilation.

### Key Classes and Functions:

1. **`CppExtension`** – Creates a `setuptools.Extension` for C++ extensions.
2. **`CUDAExtension`** – Creates a `setuptools.Extension` for mixed C++/CUDA extensions, automatically handling CUDA paths and libraries.
3. **`SyclExtension`** – Creates a `setuptools.Extension` for SYCL/C++ extensions (e.g., Intel XPU).
4. **`BuildExtension`** – A custom `setuptools` build extension that handles compiler flags and mixed C++/CUDA/SYCL compilation, with optional Ninja backend for faster builds.
5. **`load()`** – JIT compilation: builds and loads an extension from source files at runtime.
6. **`load_inline()`** – JIT compilation from string sources, automatically generating bindings for specified functions.

### Common Use Cases:

- **Ahead-of-time compilation** (for distribution): Use `CppExtension`/`CUDAExtension`/`SyclExtension` with `setup.py` and `BuildExtension`.
- **Just-in-time compilation** (for development/testing): Use `load()` or `load_inline()` to compile and import extensions dynamically.

### Important Notes:

- **CUDA/SYCL support**: Mixed compilation is automatic when `.cu`/`.cuh` (CUDA) or `.sycl` (SYCL) files are included.
- **Compute architectures**: For CUDA, you can specify target architectures via `TORCH_CUDA_ARCH_LIST`. For SYCL, use `TORCH_XPU_ARCH_LIST`.
- **Ninja**: Recommended for faster builds (default in `BuildExtension`). Use `MAX_JOBS` to control parallelism.
- **Python limited API**: PyTorch's Python API (`libtorch_python`) cannot be built with `py_limited_api=True`.
- **Relocatable Device Code (RDC)**: Required for cross-file device symbol references in CUDA; use `-rdc=true` and `dlink=True` in `CUDAExtension`.

### Example Workflow:


**JIT:**
```python

```
