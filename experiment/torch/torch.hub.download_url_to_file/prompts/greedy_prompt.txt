**PyTorch Hub** is a centralized repository for sharing and loading pre-trained models, designed to promote research reproducibility.

### **Key Functions**

- **`torch.hub.list(github)`**: Lists all available entrypoints (models/tools) in a specified GitHub repository.
- **`torch.hub.help(github, model)`**: Shows the docstring and usage instructions for a specific model.
- **`torch.hub.load(repo_or_dir, model, *args, **kwargs)`**: Loads a model or tool from a GitHub repo or local directory. Arguments are passed to the model's entrypoint function for instantiation.
- **`torch.hub.load_state_dict_from_url(url, ...)`**: Downloads and loads a model's state dictionary from a URL.
- **`torch.hub.get_dir()` / `torch.hub.set_dir(d)`**: Gets or sets the local cache directory for downloaded models.

### **Publishing a Model to Hub**

To share a model, add a `hubconf.py` file to your GitHub repo. This file defines callable entrypoints.

**Example `hubconf.py`**:
```python
dependencies = ['torch', 'torchvision']  # Required packages

def my_model(pretrained=False, **kwargs):
    """Model description and example usage."""
    model = create_my_model(**kwargs)
    if pretrained:
        checkpoint = 'https://example.com/model.pth'
        model.load_state_dict(torch.hub.load_state_dict_from_url(checkpoint))
    return model
```

**Key points for publishers**:
- Entrypoints can return models (`nn.Module`) or auxiliary tools.
- Document your function with a clear docstring showing examples.
- Pre-trained weights (<2GB) can be attached to GitHub releases; larger files should use external URLs.
- Models must be published on a branch or tag, not a random commit.

### **Loading and Using a Model**

**Basic workflow**:
```python
# 1. See what's available
entrypoints = torch.hub.list('pytorch/vision')

# 2. Get help for a specific model
print(torch.hub.help('pytorch/vision', 'resnet18'))

# 3. Load a model (with pretrained weights)
model = torch.hub.load('pytorch/vision', 'resnet18', pretrained=True)

# 4. Explore the loaded model
print(dir(model))  # See available methods
```

### **Security and Caching**

- **`trust_repo` parameter**: Controls security prompts for running code from repositories. Set to `"check"`, `True`, or `False` (default will change to `"check"` in v2.0).
- **Caching**: Models are cached locally. Use `force_reload=True` to refresh.
- **Cache location**: Determined by `torch.hub.set_dir()`, `$TORCH_HOME/hub`, or `~/.cache/torch/hub`.

### **Limitations**

- Cannot load two different branches of the same repo in the same Python process.
- Import conflicts may occur if different repos use the same sub-package names.

PyTorch Hub simplifies model sharing and loading, enabling easy access to state-of-the-art models with minimal code.
