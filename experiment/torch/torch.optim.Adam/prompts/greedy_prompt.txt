The `torch.optim.Adam` class implements the Adam optimization algorithm, which adapts learning rates for each parameter using estimates of first and second moments of gradients.

**Key Features:**
- **Adaptive Learning Rates:** Uses moving averages of gradients (momentum) and squared gradients to adjust per-parameter learning rates
- **Bias Correction:** Applies correction terms to initial iterations to counteract initialization bias
- **AMSGrad Variant:** Optional `amsgrad=True` uses the maximum of past squared gradients for more stable convergence
- **Weight Decay Options:** Supports both standard L2 penalty (`decoupled_weight_decay=False`) and AdamW-style decoupled weight decay (`decoupled_weight_decay=True`)
- **Multiple Implementations:** Offers for-loop, foreach (faster on CUDA), and fused (fastest) implementations

**Default Parameters:**
- Learning rate (`lr`): 0.001
- Betas (`betas`): (0.9, 0.999) - coefficients for gradient and squared gradient moving averages
- Epsilon (`eps`): 1e-8 - numerical stability term
- Weight decay: 0
- AMSGrad: False

**Common Usage:**
```python
optimizer = torch.optim.Adam(model.parameters(), lr=0.001, betas=(0.9, 0.999))
```

**Important Notes:**
- The `foreach` implementation is automatically used on CUDA for better performance (uses more memory)
- The `fused` implementation offers the best performance but is newer
- Supports hooks for customizing state dict loading/saving and step execution
- Works with CUDA graphs and torch.compile when `capturable=True`
- MPS backend supports float32 and float16 dtypes

The algorithm is particularly effective for problems with noisy/sparse gradients and is widely used in deep learning applications.
