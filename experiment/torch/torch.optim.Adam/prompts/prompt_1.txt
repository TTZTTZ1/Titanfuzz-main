The `torch.optim.Adam` class implements the Adam optimization algorithm, which adapts learning rates for each parameter using estimates of the first and second moments of gradients.

**Key Parameters:**
- `params`: Iterable of parameters to optimize (required)
- `lr`: Learning rate (default: 0.001)
- `betas`: Coefficients for computing running averages (default: (0.9, 0.999))
- `eps`: Term added to denominator for numerical stability (default: 1e-8)
- `weight_decay`: L2 penalty (default: 0)
- `amsgrad`: Whether to use AMSGrad variant (default: False)
- `foreach`: Use foreach implementation for better performance on CUDA (default: None)
- `fused`: Use fused implementation for maximum performance (default: None)

**Core Algorithm Features:**
- Maintains exponentially decaying averages of past gradients (first moment) and squared gradients (second moment)
- Includes bias correction for initial timesteps
- Supports weight decay (L2 regularization)
- Optional AMSGrad variant for better convergence properties
- Can maximize instead of minimize the objective

**Implementation Notes:**
- Multiple implementations available: for-loop (default), foreach (faster on CUDA), and fused (fastest)
- Supports CUDA graphs and torch.compile via `capturable` parameter
- Can handle decoupled weight decay (AdamW-style) via `decoupled_weight_decay`
- Works with various data types including float32, float16, and bfloat16

**Common Methods:**
- `step()`: Performs a single optimization step
- `zero_grad()`: Resets gradients of optimized tensors
- `state_dict()` / `load_state_dict()`: Save and load optimizer state
- `add_param_group()`: Add parameter groups during training

The optimizer is widely used for training neural networks due to its adaptive learning rate capabilities and robust performance across different problem domains.
