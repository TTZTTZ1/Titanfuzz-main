The `torch.nn.RNN` module implements a multi-layer Elman RNN (recurrent neural network) with either tanh or ReLU activation. It processes input sequences step-by-step, updating a hidden state at each time step based on the current input and previous hidden state.

**Key Features:**
- Supports stacked RNNs via `num_layers`
- Optional bidirectional processing
- Configurable batch dimension ordering (`batch_first`)
- Dropout between layers (except the last)
- Handles both padded sequences and regular tensors

**Core Parameters:**
- `input_size`: Number of input features
- `hidden_size`: Number of hidden state features
- `nonlinearity`: 'tanh' (default) or 'relu'
- `bidirectional`: If True, processes sequence both forward and backward

**Input/Output Shapes:**
- **Input**: `(L, N, H_in)` or `(N, L, H_in)` if `batch_first=True`
- **Hidden state**: `(D*num_layers, N, H_out)`
- **Output**: `(L, N, D*H_out)` (or batch-first equivalent)

Where:
- L = sequence length
- N = batch size
- D = 2 if bidirectional else 1
- H_in = input_size
- H_out = hidden_size

**Usage Example:**
```python
rnn = nn.RNN(10, 20, 2)  # 2-layer RNN
input = torch.randn(5, 3, 10)  # (L, N, H_in)
h0 = torch.randn(2, 3, 20)     # initial hidden state
output, hn = rnn(input, h0)    # forward pass
```

The module handles GPU acceleration with cuDNN optimizations and includes warnings about non-deterministic behavior in certain CUDA configurations.
