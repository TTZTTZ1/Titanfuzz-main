The `torch.nn.RNN` module implements a multi-layer Elman RNN with either `tanh` or `ReLU` activation. It processes input sequences step-by-step, updating a hidden state at each time step using the formula:

\[
h_t = \text{activation}(x_t W_{ih}^T + b_{ih} + h_{t-1} W_{hh}^T + b_{hh})
\]

**Key Parameters:**
- `input_size`: Number of features in the input.
- `hidden_size`: Number of features in the hidden state.
- `num_layers`: Number of stacked RNN layers (default: 1).
- `nonlinearity`: `'tanh'` (default) or `'relu'`.
- `batch_first`: If `True`, input/output shape is `(batch, seq, feature)` instead of `(seq, batch, feature)`.
- `bidirectional`: If `True`, uses a bidirectional RNN (default: `False`).
- `dropout`: Dropout probability applied between RNN layers (except the last).

**Inputs:**
- `input`: Tensor of shape `(L, N, H_in)` if `batch_first=False`, else `(N, L, H_in)`.
- `hx` (optional): Initial hidden state tensor of shape `(D * num_layers, N, H_out)`; defaults to zeros.

**Outputs:**
- `output`: Tensor containing output features `(h_t)` for each time step.
- `h_n`: Tensor containing the final hidden state for each element in the batch.

**Notes:**
- Weights are initialized uniformly from \(\mathcal{U}(-\sqrt{k}, \sqrt{k})\) where \(k = 1 / \text{hidden\_size}\).
- Supports packed sequences for variable-length inputs.
- For bidirectional RNNs, direction 0 is forward and direction 1 is backward.
- Non-determinism may occur with cuDNN/CUDA; environment variables can enforce determinism at a potential performance cost.
---GENERAL SUMMARY END---
