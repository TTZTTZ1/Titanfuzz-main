### Key Functions:

1. **`checkpoint(function, *args, use_reentrant=False, **kwargs)`**  
   - **Purpose**: Runs `function` without storing intermediate activations; recomputes them during backward pass.

     - `early_stop`: Stops recomputation once needed tensors are available.

2. **`checkpoint_sequential(functions, segments, input, **kwargs)`**  
   - **Purpose**: Splits a sequential model into segments and checkpoints each (except the last).  
   - **Usage**: Reduces memory for long sequential chains (e.g., transformer layers).

### Advanced Features:
- **Selective Checkpointing**:  
  Use `create_selective_checkpoint_contexts()` with a policy function to control which operations are saved vs. recomputed (e.g., save expensive ops like matrix multiplies).
- **RNG State Handling**:  
  By default, RNG states are preserved for determinism (e.g., dropout). Disable with `preserve_rng_state=False` for performance if determinism isn’t required.
- **Debugging**:  
  Enable with `set_checkpoint_debug_enabled()` or the `debug` flag to trace forward/recompute steps.

### Important Notes:
- **Reentrant vs. Non-reentrant**:  
  Non-reentrant (`use_reentrant=False`) is preferred—it supports more autograd features and doesn’t require `requires_grad` on inputs/outputs.
- **Warnings**:  
  - Ensure `function` is deterministic (no side effects) between forward/backward.  
  - Explicitly pass `use_reentrant` (required from PyTorch 2.9 onward).  
  - Avoid detached tensors inside checkpointed regions when using reentrant mode.

### Use Case:
Ideal for memory-intensive models (e.g., large CNNs, transformers), where you can trade extra computation for reduced GPU memory usage by checkpointing specific layers or segments.
