   - **Purpose**: Optimized checkpointing for sequential models (e.g., `nn.Sequential`).
   - **How it works**: Splits the model into segments and checkpoints each except the last.

### Key Features & Considerations:

- **Memory vs Compute**: Saves memory at the cost of extra compute during backward passes.
- **RNG State**: By default, preserves RNG states for deterministic behavior (can be disabled for performance).
- **Reentrant vs Non-reentrant**:
  - **Non-reentrant (`use_reentrant=False`)**: More flexible, supports `autograd.grad`, keyword arguments, and early stopping of recomputation.

- **Debugging**: Tools like `set_checkpoint_debug_enabled()` help diagnose recomputation issues.

### Best Practices:
- Explicitly set `use_reentrant=False` for new code.
- Ensure the checkpointed function is deterministic (no side effects) to avoid incorrect gradients.
- Use selective checkpointing to optimize performance by avoiding recomputation of expensive operations.

This functionality is essential for training large models that exceed GPU memory limits, enabling larger batch sizes or model capacities at the expense of increased computation time.
