The `torch.optim.Adagrad` class implements the **Adagrad (Adaptive Gradient Algorithm)** optimizer, which adapts the learning rate for each parameter based on the historical sum of squared gradients.

### **Key Features**
- **Adaptive per-parameter learning rates**: Each parameter's learning rate is scaled inversely by the square root of the cumulative sum of its past squared gradients.
- **Learning rate decay**: Optional decay over time via `lr_decay`.
- **Weight decay**: L2 regularization (default: 0).
- **Numerical stability**: Added `eps` term to denominator.

### **Core Algorithm Steps**
1. Compute gradient \( g_t \).
2. Update learning rate: \( \tilde{\gamma} = \gamma / (1 + (t-1) \cdot \eta) \) (if `lr_decay` > 0).
3. Apply weight decay if \( \lambda \neq 0 \).
4. Accumulate squared gradients: \( \text{state\_sum}_t = \text{state\_sum}_{t-1} + g_t^2 \).
5. Update parameters:
   \[
   \theta_t = \theta_{t-1} - \tilde{\gamma} \frac{g_t}{\sqrt{\text{state\_sum}_t} + \epsilon}
   \]

### **Parameters**
- `params`: Parameters to optimize.
- `lr` (default: 0.01): Learning rate.
- `lr_decay` (default: 0): Learning rate decay factor.
- `weight_decay` (default: 0): L2 penalty.
- `initial_accumulator_value` (default: 0): Initial value for gradient accumulation.
- `eps` (default: 1e-10): Term for numerical stability.
- `foreach` (optional): Use foreach implementation for performance (CUDA).
- `maximize` (default: False): Maximize objective instead of minimizing.
- `differentiable` (default: False): Allow autograd through optimizer step.
- `fused` (optional): Use fused CPU implementation (supports specific dtypes).

### **Common Methods**
- `step(closure)`: Performs a single optimization step.
- `zero_grad(set_to_none=True)`: Resets gradients.
- `state_dict()` / `load_state_dict()`: Save/load optimizer state.
- `add_param_group()`: Add new parameter groups dynamically.

### **Use Case**
Adagrad is suitable for **sparse data** (e.g., NLP, recommendation systems) because it automatically assigns higher learning rates to infrequent parameters. However, its monotonically decreasing learning rate can become too small over time, which led to variants like Adadelta and RMSprop.
