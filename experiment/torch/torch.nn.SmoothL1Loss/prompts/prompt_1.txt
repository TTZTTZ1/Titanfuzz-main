`SmoothL1Loss` is a PyTorch loss function that combines L1 and L2 losses, using a squared term for small errors (below `beta`) and an L1 term for larger errors. This makes it less sensitive to outliers than `MSELoss` and helps prevent exploding gradients.

**Key features:**
- **Formula:** For each element, if `|x - y| < beta`, loss = `0.5 * (x - y)^2 / beta`; otherwise, loss = `|x - y| - 0.5 * beta`.
- **Reduction options:** `'none'`, `'mean'` (default), or `'sum'`.
- **Hyperparameter:** `beta` (default 1.0) controls the threshold between L1 and L2 behavior.
- **Relation to other losses:** Equivalent to `HuberLoss` scaled by `1/beta`. As `beta → 0`, it becomes `L1Loss`; as `beta → ∞`, loss approaches 0.

**Usage:**
```python
loss_fn = nn.SmoothL1Loss(beta=1.0, reduction='mean')
loss = loss_fn(input, target)  # Input and target must have same shape
```

**Typical applications:** Regression tasks where robustness to outliers is needed, such as object detection (e.g., Fast R-CNN).
