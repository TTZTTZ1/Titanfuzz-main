- **Backends**:

- **Debugging Tools**:

  - `torch.distributed.breakpoint()` for interactive debugging on a single rank.

## Common Use Cases:

- **Multi-Node Training**: Scale training across multiple machines.
- **Custom Communication Patterns**: Use collectives or point-to-point ops for custom workflows.

## Best Practices:
- Use NCCL for GPU training, Gloo for CPU.
- Ensure proper GPU isolation (one GPU per process) with NCCL.

- Leverage debugging tools (`TORCH_DISTRIBUTED_DEBUG`, `monitored_barrier`) for troubleshooting.

This package is essential for scaling PyTorch models across multiple devices or nodes, offering both high-level abstractions and low-level control over distributed communication.
