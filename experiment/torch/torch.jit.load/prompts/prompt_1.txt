---GENERAL SUMMARY START---
`torch.jit.load` loads a previously saved TorchScript model (ScriptModule or ScriptFunction) from a file or file-like object. By default, it loads all tensors onto CPU first, then moves them to their original saved devices. Key parameters include:

- **`map_location`**: Optionally remap storage to a different device (e.g., `'cpu'` or `torch.device('cpu')`).
- **`_extra_files`**: Load additional file content stored with the model.
- **`_restore_shapes`**: If `True`, retrace the module using stored inputs.

⚠️ **Security warning**: Only load trusted data, as malicious pickled content can execute arbitrary code.

**Example usage**:
```python
import torch

# Load from file
model = torch.jit.load('model.pt')

# Load onto CPU
model = torch.jit.load('model.pt', map_location='cpu')

# Load with extra files
extra_files = {'config.txt': ''}
model = torch.jit.load('model.pt', _extra_files=extra_files)
print(extra_files['config.txt'])
```
