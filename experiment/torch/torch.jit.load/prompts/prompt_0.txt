# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
`torch.jit.load` loads a previously saved TorchScript model (ScriptModule or ScriptFunction) from a file or file-like object. It automatically loads all tensors to CPU first, then moves them to their original saved devices. Key parameters include:

- `f`: File path or file-like object
- `map_location`: Optionally remap storage devices (e.g., to CPU)
- `_extra_files`: Dictionary to load additional saved metadata files
- `_restore_shapes`: Whether to retrace the model using stored inputs

**Warning:** Only load trusted data, as malicious pickle data can execute arbitrary code.

**Example usage:**
```python
import torch

# Load from file
model = torch.jit.load('model.pt')

# Load to CPU
model = torch.jit.load('model.pt', map_location='cpu')

# Load with extra metadata
extra_files = {'config.txt': ''}
model = torch.jit.load('model.pt', _extra_files=extra_files)
```
