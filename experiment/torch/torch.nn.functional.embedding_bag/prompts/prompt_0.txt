---GENERAL SUMMARY START---
`torch.nn.functional.embedding_bag` efficiently computes aggregated embeddings (sum, mean, or max) for bags of indices without creating intermediate embeddings. It is useful for handling variable-length sequences via offsets or fixed-length 2D inputs.

**Key Parameters:**
- `input`: 1D (with offsets) or 2D tensor of indices.
- `weight`: Embedding matrix of shape `(num_embeddings, embedding_dim)`.
- `offsets`: Required for 1D input; specifies start indices of each bag.
- `mode`: Aggregation method: `"sum"`, `"mean"` (default), or `"max"`.
- `padding_idx`: Optional index to ignore in aggregation and gradient updates.

**Returns:** Aggregated embeddings of shape `(B, embedding_dim)` where `B` is the number of bags.

**Example:**
```python
embedding_matrix = torch.rand(10, 3)
input = torch.tensor([1, 2, 4, 5, 4, 3, 2, 9])
offsets = torch.tensor([0, 4])
output = F.embedding_bag(input, embedding_matrix, offsets)  # shape: (2, 3)
```

**Notes:**
- Supports sparse gradients (except in `"max"` mode).
- `per_sample_weights` allows weighted aggregation.
- Non-deterministic gradients possible on CUDA devices.
