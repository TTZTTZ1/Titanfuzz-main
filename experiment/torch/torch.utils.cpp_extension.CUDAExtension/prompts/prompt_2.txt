1. **Extension Classes:**
   - `CppExtension`: Creates a `setuptools.Extension` for C++.

- **Ahead-of-time (setup.py):**

  ```python
  from torch.utils.cpp_extension import BuildExtension, CppExtension/CUDAExtension

  setup(
      ext_modules=[CppExtension('name', ['src.cpp'], extra_compile_args=['-O2'])],
      cmdclass={'build_ext': BuildExtension}
  )
  ```

- **Just-in-time (JIT):**
  ```python

  module = load(name='ext', sources=['file.cpp', 'kernel.cu'], extra_cflags=['-O2'])
  ```

**Key Features and Notes:**

- **Mixed Compilation:** Automatically detects and compiles `.cu`/`.cuh` (CUDA) and `.sycl` (SYCL) files with appropriate compilers (nvcc/SYCL compiler).

- **Relocatable Device Code (RDC):** Supported for CUDA with `-rdc=true` and `dlink=True`; requires Ninja.
- **Python Limited API:** PyTorch's Python API (`libtorch_python`) is incompatible with `py_limited_api=True`.
- **Dependencies:** Ninja is recommended (and required for some features like RDC). Compiler toolchains (e.g., C++ compiler, CUDA toolkit) must be installed separately for JIT compilation.
- **Environment Variables:** Useful ones include `TORCH_EXTENSIONS_DIR` (build location), `CUDA_HOME` (CUDA path), and `MAX_JOBS` (build parallelism).

This module simplifies integrating custom C++/CUDA/SYCL code into PyTorch, enabling performance-critical operations and kernel development.
