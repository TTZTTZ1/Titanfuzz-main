- Use `init_process_group()` or `init_device_mesh()` to set up the process group.

- **Profiling**: Integrated with `torch.profiler` to profile communication operations.

## **Best Practices**
- Always call `destroy_process_group()` to clean up resources.

- Offload data movement to GPU copy engines (DMA) instead of SMs for better compute/communication overlap.
- Requires symmetric memory allocation and zero‑CTA policy configuration.
