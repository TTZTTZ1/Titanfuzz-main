The `torch.distributions` package provides a comprehensive framework for working with probability distributions in PyTorch. Here's a concise summary of its key functionality:

## Core Purpose
- Enables construction of stochastic computation graphs and gradient estimators for optimization
- Supports two main gradient estimation methods: Score Function (REINFORCE) and Pathwise Derivative (reparameterization trick)

## Key Components

### 1. **Distribution Base Class**
- Abstract base class for all probability distributions
- Common methods: `sample()`, `log_prob()`, `rsample()`, `entropy()`, `cdf()`, `icdf()`
- Properties: `mean`, `variance`, `mode`, `support`

### 2. **Available Distributions**
- **Univariate**: Bernoulli, Beta, Binomial, Categorical, Cauchy, Chi2, Exponential, Gamma, Geometric, Gumbel, Laplace, LogNormal, Normal, Pareto, Poisson, StudentT, Uniform, etc.
- **Multivariate**: Dirichlet, MultivariateNormal, LowRankMultivariateNormal, Wishart, LKJCholesky
- **Specialized**: MixtureSameFamily, TransformedDistribution, Independent, Relaxed distributions

### 3. **Key Features**

**Gradient Estimation:**
- **Score Function**: Uses `sample()` + `log_prob()` for REINFORCE (policy gradients)
- **Pathwise Derivative**: Uses `rsample()` for reparameterization trick (VAEs)

**Transforms & Constraints:**
- `Transform` objects for bijective mappings between spaces
- `Constraint` objects for parameter validation
- Registry system (`biject_to`, `transform_to`) for constraint transformations

**KL Divergence:**
- Pre-registered KL divergences between many distribution pairs
- Extensible via `register_kl()` decorator

### 4. **Common Usage Patterns**

**REINFORCE (Policy Gradient):**
```python
probs = policy_network(state)
m = Categorical(probs)
action = m.sample()
loss = -m.log_prob(action) * reward
loss.backward()
```

**Reparameterization Trick:**
```python
params = policy_network(state)
m = Normal(*params)
action = m.rsample()  # Differentiable sample
loss = -reward
loss.backward()
```

### 5. **Design Principles**
- Follows TensorFlow Distributions design
- Batch-aware: all distributions support batched parameters
- Validation via `arg_constraints`
- Exponential family distributions for efficient entropy/KL computations
- Transformed distributions for complex distributions via composition

The package is essential for probabilistic modeling, reinforcement learning, variational inference, and any application requiring differentiable sampling from probability distributions.
