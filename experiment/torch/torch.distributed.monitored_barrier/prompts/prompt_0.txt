## **Core Features**
- **Multi-process parallelism** across computation nodes
- **Synchronous distributed training** via `DistributedDataParallel`
- **Four built-in backends**: Gloo (CPU/GPU), NCCL (CUDA GPU), XCCL (XPU GPU), MPI (optional)
- **Communication primitives**: point-to-point, collectives, groups

### **3. Process Management**
- **Groups**: Create sub-groups with `new_group()`

- **Shutdown**: Clean up with `destroy_process_group()`

### **4. Key-Value Stores**
- **TCPStore**: TCP-based distributed store
- **FileStore**: File-based store
- **HashStore**: In-memory store for same-process use

## **Backend Selection Guidelines**
- **CUDA GPU training**: Use NCCL (best performance)
- **XPU GPU training**: Use XCCL
- **CPU training**: Use Gloo
- **InfiniBand**: NCCL for GPU, Gloo/MPI for CPU

## **Advanced Features**
- **Copy Engine Collectives**: Offload communication to GPU DMA engines (NCCL 2.28+)
- **Object collectives**: Python object communication (with performance limitations)
- **Third-party backends**: Extensible via registration API
- **Debugging tools**: Monitored barriers, debug levels, HTTP debug server

## **Launch Utilities**
- **`torch.distributed.launch`**: Spawn multiple processes per node (deprecated in favor of `torchrun`)
- **`torch.multiprocessing.spawn`**: Alternative process spawning utility

## **Best Practices**
- Each process maintains its own optimizer (no parameter broadcast needed)
- Separate Python interpreters per process avoid GIL contention
- Proper cleanup with `destroy_process_group()` to prevent hangs
- Use NCCL for GPU training with Ethernet/InfiniBand interconnects

The package enables efficient distributed training with advantages over `DataParallel`, including reduced communication overhead and better Python runtime utilization.
