## **Core Concepts**
- Enables synchronous distributed training with `DistributedDataParallel`
- Supports multiple processes per node, each with independent Python interpreters
- Requires explicit launch of separate script copies for each process

## **Backends**
Four built-in backends with different device support:
- **NCCL**: Recommended for CUDA GPU training (best performance)
- **XCCL**: For XPU GPU training  
- **Gloo**: Recommended for CPU training

- `all_reduce()`, `broadcast()`, `all_gather()`: Collective operations
- `send()`, `recv()`, `isend()`, `irecv()`: Point-to-point communication
- `barrier()`: Synchronize all processes

## **DeviceMesh**
Higher-level abstraction for managing process groups in N-dimensional device topologies:
- `init_device_mesh()`: Create device mesh with specified shape
- Simplifies setup of inter/intra-node process groups

## **Debugging Tools**
- `monitored_barrier()`: Barrier with timeout and rank failure reporting
- `TORCH_DISTRIBUTED_DEBUG`: Environment variable for detailed logging
- `torch.distributed.breakpoint()`: Interactive debugging on specific rank
- HTTP debug server for runtime data collection

## **Best Practices**
- Use NCCL for GPU training, Gloo for CPU
- Ensure exclusive GPU access per process with NCCL
- Call `destroy_process_group()` for cleanup
- Set `async_op=True` for asynchronous operations with overlap
- Use `DeviceMesh` for complex multi-dimensional parallelism setups
