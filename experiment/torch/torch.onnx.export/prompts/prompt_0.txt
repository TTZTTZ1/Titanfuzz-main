# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
The PyTorch `torch.export`-based ONNX exporter converts PyTorch models to ONNX format using the modern `torch.export` engine, which traces the model's tensor computation into a normalized, static graph. This exporter is now the default (`dynamo=True`) and is recommended over the legacy TorchScript-based approach.

### Key Features:
- **AOT Graph Capture**: Eliminates Python control flow, producing a functional ATen operator graph with shape constraints.
- **Reduced Memory Usage**: Optimized export process.
- **Dynamic Shapes**: Supports dynamic dimensions via `dynamic_shapes`.
- **Rich Metadata**: Each ONNX node includes debugging info like stack traces, FX node details, and class hierarchies.
- **Flexible Serialization**: Control over including model weights and treating them as inputs via `ONNXProgram.save()`.

### Basic Usage:
```python
import torch
import torch.onnx

# Export model
onnx_program = torch.onnx.export(
    model,
    example_args,
    dynamo=True,
    input_names=["input"],
    output_names=["output"],
    dynamic_shapes={...}  # Optional
)

# Save to file
onnx_program.save("model.onnx")
```

### Dependencies:
- `onnx` and `onnxscript` packages (install via `pip`).
- Optional: `onnxruntime` for execution.

### Error Handling:
If export fails, rerun with `report=True` to generate a detailed markdown report for debugging.

### Output:
Returns an `ONNXProgram` object containing:
- `model_proto`: The ONNX model (`onnx.ModelProto`).
- Methods to save, optimize, run inference, and inspect the model.

This exporter is designed for modern PyTorch models, offering better performance, flexibility, and debugging support compared to the legacy exporter.
