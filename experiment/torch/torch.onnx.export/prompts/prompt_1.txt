# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
The `torch.onnx.export` function is the primary API for exporting PyTorch models to the ONNX format. It leverages the `torch.export` engine to trace the model's computation graph, eliminating Python control flow and normalizing operators for efficient conversion.

**Key Features:**
- **Modern Export Pipeline**: Enabled by default (`dynamo=True`), it uses `torch.export.ExportedProgram` for a more robust and optimized export process.
- **Dynamic Shapes**: Supports dynamic input dimensions via the `dynamic_shapes` parameter.
- **Metadata Preservation**: Annotates ONNX nodes with PyTorch origin information (stack traces, module hierarchy, etc.) for debugging.
- **Flexible Serialization**: Returns an `ONNXProgram` object that allows control over how the model is saved (e.g., including initializers, external data storage).

**Basic Usage:**
```python
import torch

# Export a model
onnx_program = torch.onnx.export(
    model,
    example_args,
    input_names=["input"],
    output_names=["output"],
    dynamic_shapes={"input": {0: "batch_size"}}
)

# Save to file
onnx_program.save("model.onnx")
```

**Important Parameters:**
- `model`: The PyTorch model or `ExportedProgram` to export.
- `args`: Example inputs for tracing.
- `dynamic_shapes`: Specifies dynamic axes (preferred when `dynamo=True`).
- `input_names`/`output_names`: Names for ONNX graph inputs/outputs.
- `report=True`: Generates a debug report if export fails.

**Output:**
Returns an `ONNXProgram` object containing:
- The ONNX model (`model_proto`)
- The original `ExportedProgram`
- Methods to save, optimize, and run the model

The exporter handles complex inputs (tuples, dicts, lists) and provides extensive metadata to trace back to the original PyTorch code. For large models (>2GB), it automatically supports external weight storage.
