`torch.as_tensor` converts input data (like lists, NumPy arrays, or existing tensors) into a PyTorch tensor, **sharing memory when possible** to avoid copying. It preserves autograd history if the input is already a tensor.

**Key behaviors:**
- Returns the same tensor if `dtype` and `device` already match.
- Copies data only if needed (e.g., when `dtype` or `device` differ).
- Shares memory with NumPy arrays if on the same device (changes reflect in both).
- For CuPy arrays, the tensor is placed on the corresponding CUDA device by default.

**Contrast with `torch.tensor`:**
- `torch.tensor` always creates a new copy and a new leaf tensor in the autograd graph.

**Parameters:**
- `data`: Input data (list, tuple, ndarray, tensor, etc.).
- `dtype`: Desired tensor dtype (infers from data if `None`).
- `device`: Target device (uses data's device if data is a tensor; otherwise uses current device).

**Example:** Converting a NumPy array shares memory (modifying the tensor affects the array), unless a different device is specified, which forces a copy.
