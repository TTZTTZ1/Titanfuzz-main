`torch.nn.RNNCell` is a single Elman RNN cell that processes one time step. It applies either tanh or ReLU nonlinearity to compute the next hidden state from the current input and previous hidden state.

**Key Parameters:**
- `input_size`: Number of features in input `x`
- `hidden_size`: Number of features in hidden state `h`
- `nonlinearity`: 'tanh' (default) or 'relu'
- `bias`: Whether to include bias terms (default: True)

**Usage:**
```python
rnn_cell = nn.RNNCell(input_size, hidden_size)
h_next = rnn_cell(input_t, h_prev)
```

**Key Points:**
- Processes one time step at a time (unlike `nn.RNN` which processes sequences)
- Input shape: `(batch, input_size)` or `(input_size,)`
- Hidden state shape: `(batch, hidden_size)` or `(hidden_size,)`
- Output shape: Same as hidden state shape
- Weights initialized uniformly from `[-√(1/hidden_size), √(1/hidden_size)]`

**Typical Use Case:** Manually iterating through sequence time steps when custom control over the RNN loop is needed.
---GENERAL SUMMARY END---
