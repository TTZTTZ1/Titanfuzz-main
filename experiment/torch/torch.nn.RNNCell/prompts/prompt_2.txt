`torch.nn.RNNCell` is a single Elman RNN cell that processes one time step at a time. It takes an input and the previous hidden state, then computes the next hidden state using either tanh or ReLU activation.

**Key parameters:**
- `input_size`: Number of features in the input
- `hidden_size`: Number of features in the hidden state
- `nonlinearity`: 'tanh' (default) or 'relu'
- `bias`: Whether to include bias terms (default: True)

**Usage:**
- Input shape: `(batch_size, input_size)` or `(input_size,)`
- Hidden state shape: `(batch_size, hidden_size)` or `(hidden_size,)`
- Output: Next hidden state with same shape as input hidden state

**Example workflow:**
```python
rnn_cell = nn.RNNCell(10, 20)  # 10 input features, 20 hidden features
hx = torch.randn(3, 20)        # Initial hidden state for batch of 3
for i in range(sequence_length):
    hx = rnn_cell(input[i], hx)  # Process one time step
```

The cell maintains learnable weights for input-hidden and hidden-hidden connections, initialized uniformly. It's designed for manual loop-based sequence processing rather than handling full sequences automatically.
