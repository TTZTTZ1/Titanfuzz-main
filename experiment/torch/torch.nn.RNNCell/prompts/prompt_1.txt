`torch.nn.RNNCell` is a single Elman RNN cell that processes one time step at a time. It applies either a tanh or ReLU activation to compute the next hidden state from the current input and previous hidden state.  

**Key parameters:**  
- `input_size`: Number of features in the input  
- `hidden_size`: Number of features in the hidden state  
- `nonlinearity`: `'tanh'` (default) or `'relu'`  
- `bias`: Whether to include bias terms  

**Inputs:**  
- `input`: Tensor of shape `(batch, input_size)` or `(input_size)`  
- `hidden`: Initial hidden state of shape `(batch, hidden_size)` (optional, defaults to zero)  

**Output:**  
- Next hidden state with shape matching the input hidden state  

**Weights:**  
- `weight_ih`, `weight_hh`, and optional biases are initialized uniformly from `±sqrt(1/hidden_size)`.  

**Usage example:**  
```python
rnn = nn.RNNCell(10, 20)
hx = torch.randn(3, 20)
for i in range(seq_len):
    hx = rnn(input[i], hx)
```
This is useful for custom RNN loops where manual control over each time step is needed.
