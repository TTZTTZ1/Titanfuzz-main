`torch.nn.RNNCell` is a single Elman RNN cell that processes one time step at a time. It applies either a tanh or ReLU activation to compute the next hidden state from the current input and previous hidden state.  

**Key details:**  
- **Parameters:** `input_size`, `hidden_size`, `bias` (default True), `nonlinearity` ('tanh' or 'relu').  
- **Inputs:** `input` (batch, input_size) and `hidden` (batch, hidden_size). If hidden is not provided, it defaults to zero.  
- **Output:** Next hidden state `h'` with shape (batch, hidden_size).  
- **Weights:** `weight_ih`, `weight_hh`, and optional biases `bias_ih`, `bias_hh`.  
- **Initialization:** Weights are uniformly sampled from \( \mathcal{U}(-\sqrt{k}, \sqrt{k}) \) where \( k = 1 / \text{hidden\_size} \).  

**Usage example:**  
```python
rnn = nn.RNNCell(10, 20)
hx = torch.randn(3, 20)
for i in range(seq_len):
    hx = rnn(input[i], hx)
```  
This allows manual looping over sequence steps, unlike `nn.RNN` which processes the full sequence at once.
