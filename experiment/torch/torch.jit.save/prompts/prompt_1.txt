`torch.jit.save` serializes a TorchScript module (`ScriptModule`) to a file or buffer, enabling it to be loaded and executed independently in either Python (`torch.jit.load`) or C++ (`torch::jit::load`). The saved module includes all methods, submodules, parameters, and attributes.

**Key points:**
- The module must be a `ScriptModule` with no calls to native Python functions.
- Saved modules are always loaded onto the CPU (unlike `torch.load`).
- Supports saving to a filename, file-like object, or with extra files via `_extra_files`.
- Operator behavior is preserved based on the PyTorch version used during saving.

**Example usage:**
```python
torch.jit.save(scripted_module, 'model.pt')
```
