# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
`torch.jit.save` serializes a PyTorch `ScriptModule` for offline use, allowing it to be loaded in separate processes via Python (`torch.jit.load`) or C++ (`torch::jit::load`). It saves the module’s methods, submodules, parameters, and attributes.

**Key points:**
- The module must be a `ScriptModule` with no calls to native Python functions.
- Saved modules are always loaded onto the CPU (unlike `torch.load`).
- Operator behavior is preserved based on the PyTorch version used during saving.
- Supports saving to a file path, file-like object, or with extra files.

**Example:**
```python
m = torch.jit.script(MyModule())
torch.jit.save(m, 'model.pt')
```
