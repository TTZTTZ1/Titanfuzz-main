The `torch.optim.LBFGS` class implements the L-BFGS optimization algorithm, a memory-efficient quasi-Newton method suitable for smooth, unconstrained optimization problems.

**Key Characteristics:**
- **Memory Intensive**: Requires additional memory proportional to `param_bytes * (history_size + 1)`.
- **Single Device/Group**: All parameters must be on one device; parameter groups are not supported.
- **Closure-based**: Requires a closure function that recomputes the loss and gradients each step.

**Main Parameters:**
- `lr` (float, default=1): Learning rate (often used as a scaling factor).
- `max_iter` (int, default=20): Maximum iterations per optimization step.
- `max_eval` (int, default=`max_iter * 1.25`): Maximum function evaluations per step.
- `tolerance_grad` (float, default=1e-7): Gradient-based termination tolerance.
- `tolerance_change` (float, default=1e-9): Tolerance for function/parameter changes.
- `history_size` (int, default=100): Number of past updates stored (affects memory).
- `line_search_fn` (str, optional): Can be `'strong_wolfe'` or `None`.

**Essential Methods:**
- `step(closure)`: Performs optimization using a closure that returns the loss.
- `zero_grad(set_to_none=True)`: Resets gradients (default sets to `None` for efficiency).
- `state_dict()` / `load_state_dict()`: Save and load optimizer state.

**Usage Notes:**
- Particularly useful for problems with many parameters where Hessian approximation is beneficial.
- Not compatible with learning rate schedulers in the standard way due to its internal line search.
- Consider reducing `history_size` if memory is constrained.
