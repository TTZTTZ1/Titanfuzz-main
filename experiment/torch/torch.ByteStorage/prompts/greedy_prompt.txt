### **Key Points:**

*   **Purpose:** Stores the actual bytes of a tensor's data. Multiple tensors (e.g., views created via `.view()` or slicing) can share the same storage.
*   **Access:** Get a tensor's storage via `tensor.untyped_storage()`.

*   **Manipulation:** Provides low-level methods like `.fill_()`, `.clone()`, `.copy_()`, and `.pin_memory()`. However, directly modifying storage is **not recommended** for typical use; prefer high-level tensor operations.

*   **Special Cases:** Tensors on the `"meta"` device or `FakeTensor`s have no actual storage. A tensor with gradients has separate storages for its data and its gradient.
*   **Legacy Note:** Older `torch.TypedStorage` and dtype-specific classes (e.g., `torch.FloatStorage`) are **deprecated**. `torch.UntypedStorage` is the modern, unified interface.

### **When to Use:**
Most users should **avoid** directly manipulating `UntypedStorage`. It's a specialized API for advanced scenarios like custom memory management, interoperability, or low-level performance tuning. Standard tensor operations are safer and more efficient.
