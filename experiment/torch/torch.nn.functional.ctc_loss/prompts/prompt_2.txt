`torch.nn.functional.ctc_loss` computes the Connectionist Temporal Classification (CTC) loss, which is used for sequence-to-sequence tasks like speech or handwriting recognition. It calculates the loss between a sequence of output probabilities and a target sequence, handling variable input and target lengths.

**Key Parameters:**
- `log_probs`: Log probabilities of shape `(T, N, C)` (sequence length, batch size, alphabet size including blank).
- `targets`: Target sequences of shape `(N, S)` or concatenated as a 1D tensor.
- `input_lengths`: Lengths of each input sequence in the batch.
- `target_lengths`: Lengths of each target sequence in the batch.
- `blank`: Index of the blank label (default 0).
- `reduction`: Specifies loss reduction (`'none'`, `'mean'`, `'sum'`).
- `zero_infinity`: Whether to zero infinite losses (occurs when inputs are too short).

**Notes:**
- May be nondeterministic on CUDA with CuDNN; set `torch.backends.cudnn.deterministic = True` for reproducibility.
- Accepts both batched and unbatched inputs.
- Infinite losses can arise if inputs are shorter than required for alignment.

**Example:**
