**Key Components:**

1. **Extension Classes:**
   - `CppExtension`: Creates a `setuptools.Extension` for C++ code.

2. **Build Extension:**
   - `BuildExtension`: A custom `setuptools.build_ext` subclass that manages compiler flags and mixed C++/CUDA/SYCL compilation. It uses Ninja by default for faster builds.

3. **JIT Loading Functions:**
   - `load()`: Compiles and loads an extension from source files.
   - `load_inline()`: Compiles and loads an extension from string sources, automatically generating bindings if needed.

   - `get_compiler_abi_compatibility_and_version()`: Checks compiler compatibility with PyTorch.

**Common Use Cases:**
- **Ahead-of-time:** Use `CppExtension`/`CUDAExtension` with `setup.py` and `BuildExtension` for distributing extensions.
- **Just-in-time:** Use `load()` or `load_inline()` for rapid prototyping and development.

**Important Notes:**
- Extensions can mix C++, CUDA (`.cu`), and SYCL (`.sycl`) sources.
- Ninja is recommended for faster builds (default in `BuildExtension`).
- For CUDA, compute capabilities can be specified via `TORCH_CUDA_ARCH_LIST`.
- The `py_limited_api=True` flag is not supported when using PyTorch's Python bindings.
