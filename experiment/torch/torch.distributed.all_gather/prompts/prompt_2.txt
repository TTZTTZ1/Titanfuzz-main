**Key Features:**

- **Backends:** Choose based on hardware:

- **Debugging:** Tools like `monitored_barrier()`, `TORCH_DISTRIBUTED_DEBUG`, and `torch.distributed.breakpoint()` help diagnose hangs/desynchronization.
- **Launch Utilities:** `torch.distributed.launch` (deprecated in favor of `torchrun`) and `torch.multiprocessing.spawn` for multi‑process startup.

**Typical Workflow:**
1. Initialize the process group with `init_process_group()`.
2. Use collective operations or `DistributedDataParallel` for model training.
3. Ensure proper GPU device setting (`torch.cuda.set_device`) when using NCCL.
4. Clean up with `destroy_process_group()`.

The package is designed for SPMD (single‑program‑multiple‑data) execution, where the same script runs on all ranks, and requires explicit launch of separate processes per node.
