`torch.nn.AvgPool2d` performs 2D average pooling over an input tensor, reducing its spatial dimensions by computing the average value within sliding windows.

**Key parameters:**
- `kernel_size`: Size of the pooling window (int or tuple)
- `stride`: Step size for sliding window (defaults to kernel_size)
- `padding`: Zero-padding added to input borders
- `ceil_mode`: Uses ceiling instead of floor for output size calculation
- `count_include_pad`: Includes padding in average computation when True
- `divisor_override`: Optional custom divisor instead of pooling region size

**Input/Output shape:**
- Input: `(N, C, H_in, W_in)` or `(C, H_in, W_in)`
- Output: `(N, C, H_out, W_out)` or `(C, H_out, W_out)`

**Functionality:**  
Applies a 2D averaging operation where each output element is the mean of values in the corresponding `kernel_size` window of the input. With padding, the input is implicitly zero-padded. When `ceil_mode=True`, windows may extend beyond boundaries if they start within valid regions.

**Example:**
```python
m = nn.AvgPool2d(3, stride=2)  # 3x3 window, stride 2
output = m(input)  # Reduces spatial dimensions
```
