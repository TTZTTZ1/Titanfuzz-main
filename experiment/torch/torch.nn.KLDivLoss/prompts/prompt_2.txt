The `torch.nn.KLDivLoss` computes the Kullback-Leibler divergence loss between two probability distributions.  

**Key points:**  
- **Input (`input`)**: Expected to be in **log-space** (e.g., log-probabilities) to avoid underflow.  
- **Target (`target`)**: Can be in probability space (default) or log-space if `log_target=True`.  
- **Reduction options**:  
  - `"batchmean"` (recommended): Mathematically correct KL divergence (sum divided by batch size).  
  - `"mean"`: Averages over all elements (does **not** give true KL value).  
  - `"sum"`: Sums all elements.  
  - `"none"`: Returns per-element loss.  

**Warning:**  
- Use `reduction="batchmean"` for correct KL divergence values, not `"mean"`.  

**Example usage:**  
```python
kl_loss = nn.KLDivLoss(reduction="batchmean")
input = F.log_softmax(model_output, dim=1)
target = F.softmax(target_data, dim=1)
loss = kl_loss(input, target)
```
