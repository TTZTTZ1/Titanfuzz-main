`ConvTranspose3d` is a PyTorch module that performs a 3D transposed convolution, often used for upsampling in tasks like 3D segmentation or generation. It expands spatial dimensions by applying a learnable kernel to the input, effectively reversing the spatial downsampling of a regular 3D convolution.

**Key parameters:**
- `in_channels`, `out_channels`: Number of input/output channels.
- `kernel_size`, `stride`, `padding`, `output_padding`, `dilation`: Control output shape via the formula:
  ```
  D_out = (D_in - 1) × stride[0] − 2 × padding[0] + dilation[0] × (kernel_size[0] − 1) + output_padding[0] + 1
  ```
  (similar for height and width).
- `groups`: Splits input/output channels into independent groups.
- `bias`: Whether to add a learnable bias term.

**Notes:**
- Not a true deconvolution; it's the gradient of `Conv3d` w.r.t. its input.
- `output_padding` resolves ambiguity when `stride > 1` to match desired output size.
- Supports CUDA, TensorFloat32, and has deterministic options via CuDNN settings.

**Example:**
```python
m = nn.ConvTranspose3d(16, 33, 3, stride=2)
input = torch.randn(20, 16, 10, 50, 100)
output = m(input)  # Upsamples spatial dimensions
```
