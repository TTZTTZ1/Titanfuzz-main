3. Clean up with `destroy_process_group()`

## Backend Selection Guidelines
- **CUDA GPU training**: Use NCCL (best performance)
- **XPU GPU training**: Use XCCL
- **CPU training**: Use Gloo
- **InfiniBand with GPU**: Use NCCL (only backend supporting GPUDirect)

- `TORCH_DISTRIBUTED_DEBUG` for enhanced debugging

## Important Notes
- Supports Linux (stable), macOS (stable), Windows (prototype)
- Each process maintains its own optimizer, eliminating parameter broadcast overhead
- Multiple Python interpreters avoid GIL contention
- Initialization is not thread-safe
- Object collectives have performance limitations and should be used cautiously

The package is essential for scaling PyTorch models across multiple GPUs and machines while maintaining training efficiency.
