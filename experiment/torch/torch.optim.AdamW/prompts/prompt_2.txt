The `torch.optim.AdamW` class implements the **AdamW optimizer**, a variant of Adam that decouples weight decay from gradient-based updates, preventing weight decay from accumulating in momentum and variance terms.

- **Decoupled weight decay**: Weight decay is applied directly to parameters before momentum/variance updates.
- **Supports multiple variants**: Includes AMSGrad (`amsgrad=True`) and maximization (`maximize=True`) options.
- **Performance optimizations**: Offers `foreach` (CUDA-optimized) and `fused` (theoretically fastest) implementations.

- AMSGrad (`amsgrad`): False
- Maximize (`maximize`): False

- `state_dict()` / `load_state_dict()`: Save/load optimizer state
- `add_param_group()`: Add new parameter groups dynamically

**Implementation Notes:**
- Automatically selects `foreach` implementation on CUDA for better performance
- Fused implementation available for specific dtypes (float64, float32, float16, bfloat16)
- Supports MPS devices for float32 and float16 dtypes
- Provides extensive hook system for customizing state dict and step behavior

The optimizer follows the algorithm described in "Decoupled Weight Decay Regularization" (Loshchilov & Hutter, 2019), separating weight decay from the adaptive learning rate mechanism of Adam.
