- **Purpose:** Reverses `MaxPool3d` using stored indices to place max values back, filling non‑max positions with zeros.
- **Inputs:** Requires both the pooled `input` tensor and the `indices` from `MaxPool3d`. An optional `output_size` can resolve ambiguous output dimensions.
- **Output shape:** Typically computed via the formula:

- **Note:** Non‑deterministic if indices have repeated values; not fully invertible since non‑maximal values are lost during pooling.
