- **ElasticAgent**: Abstract base class for managing workers. Implementations like `LocalElasticAgent` handle local processes.
- **WorkerSpec**: Blueprint defining worker configuration (role, entrypoint, world size, etc.).
- **WorkerGroup**: A set of `Worker` instances managed by an agent.
- **WorkerState**: Tracks the state of a worker group (e.g., `HEALTHY`, `FAILED`, `STOPPED`).

1. **Distributed Initialization**: Sets up `torch.distributed.init_process_group()` for workers.
2. **Fault Tolerance**: Monitors workers and restarts them upon failure (up to `max_restarts`).
3. **Elasticity**: Adjusts to membership changes (e.g., node scaling) by restarting workers with updated ranks.

- **SimpleElasticAgent**: Base class for custom agents; requires implementing `_start_workers`, `_stop_workers`, and `_monitor_workers`.

### **Key Concepts:**
- Workers are homogeneous per role across nodes.
- The agent ensures all-or-nothing execution: a run succeeds only if **all** workers succeed.
- Supports both function and binary entrypoints.

The Elastic Agent simplifies distributed training by automating process management, resilience, and scalability.
