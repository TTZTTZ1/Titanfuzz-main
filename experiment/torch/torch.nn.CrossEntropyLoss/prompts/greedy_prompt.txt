`CrossEntropyLoss` is a PyTorch loss function for classification tasks with C classes. It computes cross-entropy between input logits (unnormalized scores) and target labels.

**Key Features:**
- **Input**: Logits (shape: `(C)`, `(N, C)`, or `(N, C, d1, d2, ...)` for K-dimensional cases like images).
- **Target**: Can be either:
  1. **Class indices** (integer labels in `[0, C)`), which is more computationally efficient.
  2. **Class probabilities** (soft labels, shape same as input), useful for label smoothing or blended labels.
- **Options**:
  - `weight`: Class weighting for imbalanced datasets.
  - `ignore_index`: Ignore specific class indices in loss calculation.
  - `reduction`: `'none'`, `'mean'`, or `'sum'` to aggregate loss.
  - `label_smoothing`: Smooths target labels (float in `[0.0, 1.0]`).

**Important Notes:**
- Equivalent to applying `LogSoftmax` followed by `NLLLoss`.
- For probability targets, ensure values are in `[0, 1]` and sum to 1 per sample (e.g., via `.softmax(dim=1)`). PyTorch does not validate this automatically.
- Deprecated arguments: `size_average` and `reduce` (use `reduction` instead).

**Example Usage:**
```python
loss_fn = nn.CrossEntropyLoss()
# With class indices
target = torch.tensor([1, 0, 3])  # dtype=torch.long
# With class probabilities
target = torch.randn(3, 5).softmax(dim=1)  # dtype=torch.float
output = loss_fn(input_logits, target)
```
