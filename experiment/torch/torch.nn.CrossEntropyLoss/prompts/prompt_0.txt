**CrossEntropyLoss** is a PyTorch loss function for classification tasks with C classes. It computes cross-entropy between input logits (unnormalized scores) and target labels.

**Key Features:**
- **Input**: Logits (no softmax needed) of shape `(C)`, `(N, C)`, or `(N, C, d1, d2, ...)` for K-dimensional data (e.g., per-pixel loss in images).
- **Target**: Can be either:
  1. **Class indices** (integer labels in `[0, C)`), which is computationally efficient.
  2. **Class probabilities** (soft labels), useful for label smoothing or multi-label scenarios.
- **Optional Parameters**:
  - `weight`: Tensor of size C to assign class weights (handles class imbalance).
  - `ignore_index`: Specifies a target value to ignore in loss computation (for class indices only).
  - `reduction`: `'none'`, `'mean'` (default), or `'sum'` to aggregate the loss.
  - `label_smoothing`: Smooths target labels (float in `[0.0, 1.0]`).

**Mathematical Behavior**:
- For class indices: Equivalent to applying `LogSoftmax` followed by `NLLLoss`.
- For class probabilities: Directly computes cross-entropy between predicted and target distributions.

**Important Notes**:
- When using class probabilities, ensure targets are valid probability distributions (values in `[0, 1]`, sum to 1 per sample). PyTorch does not enforce this, and invalid inputs may produce misleading results.
- Performance is optimized for class indices; use probabilities only when necessary (e.g., blended labels).

**Example Usage**:
```python
loss_fn = nn.CrossEntropyLoss()
# With class indices
input = torch.randn(3, 5, requires_grad=True)
target = torch.empty(3, dtype=torch.long).random_(5)
output = loss_fn(input, target)
output.backward()

# With class probabilities
target_probs = torch.randn(3, 5).softmax(dim=1)
output = loss_fn(input, target_probs)
```
