`torch.nn.functional.grid_sample` performs spatial sampling of an input tensor using a grid of normalized coordinates. It is commonly used for tasks like image warping and spatial transformer networks.

**Key Points:**
- **Input:** A 4‑D (spatial) or 5‑D (volumetric) tensor.
- **Grid:** A tensor of normalized coordinates (typically in `[-1, 1]`) that define where to sample from the input.
- **Output:** A tensor with the same batch and channel dimensions as the input, but with spatial dimensions determined by the grid.

**Parameters:**
- `mode`: Interpolation method (`'bilinear'`, `'nearest'`, or `'bicubic'` for 4‑D only).
- `padding_mode`: How to handle out‑of‑bounds grid points (`'zeros'`, `'border'`, `'reflection'`).
- `align_corners`: If `True`, grid coordinates align with pixel centers; if `False`, they align with pixel corners (default is `False` for consistency with `interpolate`).

**Notes:**
- Grid values outside `[-1, 1]` are handled according to `padding_mode`.
- NaN values in the grid are treated as `-1`.
- When using CUDA, the backward pass may be nondeterministic.
- For bicubic mode, results may exceed the input value range—clamping is recommended.
