### **Key Features:**
- **Publishing Models:** Add a `hubconf.py` file to a GitHub repo to define entrypoints (e.g., model functions). Each entrypoint can return a model (`nn.Module`) or auxiliary tools (e.g., tokenizers).
- **Loading Models:** Use `torch.hub.load()` to fetch models from GitHub or local directories. Other functions include:

- **Dependency Management:** Specify required packages in a `dependencies` list within `hubconf.py`.

1. **Explore:** Use `torch.hub.list()` to see available models in a repo.
2. **Inspect:** Check model details with `torch.hub.help()`.
3. **Load:** Instantiate a model with `torch.hub.load()`, passing necessary arguments.
4. **Run:** Use `dir(model)` and `help(model.method)` to understand model capabilities.

### **Storage:**
Models are cached in a hub directory, determined by (in order):
1. `hub.set_dir(<PATH>)`
2. `$TORCH_HOME/hub`
3. `$XDG_CACHE_HOME/torch/hub`
4. `~/.cache/torch/hub`
