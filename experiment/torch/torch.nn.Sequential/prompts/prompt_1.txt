# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
`torch.nn.Sequential` is a container that chains multiple PyTorch modules in sequence. It simplifies building models by automatically passing the output of one module as the input to the next.

**Key Features:**
- **Ordered Execution:** Modules run in the exact order they are provided.
- **Flexible Input:** Accepts modules as positional arguments or as an `OrderedDict` with named layers.
- **Container Operations:** Supports appending, inserting, extending, and popping modules like a list.
- **Single Module Treatment:** Allows applying operations (e.g., moving to GPU) to the entire sequence at once.

**Example:**
```python
model = nn.Sequential(
    nn.Conv2d(1, 20, 5),
    nn.ReLU(),
    nn.Conv2d(20, 64, 5),
    nn.ReLU()
)
```
**Difference from `ModuleList`:** Unlike `ModuleList` (a simple list of modules), `Sequential` automatically connects layers in a forward pass.
