---GENERAL SUMMARY START---
`torch.nn.Sequential` is a container that chains multiple neural network modules in a sequential order. It simplifies model construction by treating the entire sequence as a single module, allowing inputs to pass through each submodule automatically.

**Key Features:**
- **Ordered Execution**: Modules are executed in the order they are provided (via constructor arguments or an `OrderedDict`).
- **Single Forward Pass**: The `forward()` method passes input through each module sequentially, returning the final output.
- **Module Management**: Supports appending, inserting, extending, and popping modules dynamically.

**Example Usage:**
```python
model = nn.Sequential(
    nn.Conv2d(1, 20, 5),
    nn.ReLU(),
    nn.Conv2d(20, 64, 5),
    nn.ReLU()
)
```

**Methods:**
- `append(module)`: Adds a module to the end.
- `extend(sequential)`: Appends modules from another `Sequential`.
- `insert(index, module)`: Inserts a module at a specified index.
- `pop(key)`: Removes a module by index or name.

**Difference from `ModuleList`:**
Unlike `ModuleList` (a simple list of modules), `Sequential` automatically connects outputs to inputs in a cascading manner, enabling direct forward propagation.
---GENERAL SUMMARY END---
