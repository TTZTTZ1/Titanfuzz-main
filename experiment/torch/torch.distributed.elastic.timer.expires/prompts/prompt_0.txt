The PyTorch Elastic Expiration Timers provide a mechanism to detect and handle stuck workers in distributed training by setting time limits on code blocks.

**Core Concept**:  
A timer server runs alongside the training agent, while client timers are used within worker scripts. If a worker enters a protected code block and fails to exit before the timer expires, the server can terminate that worker, allowing the agent to restart it.

**Basic Usage**:
1. Start a timer server (e.g., `LocalTimerServer` or `FileTimerServer`) in the agent process.
2. In the worker function, configure a matching timer client (e.g., `LocalTimerClient` or `FileTimerClient`).
3. Wrap potentially stuck operations with `timer.expires(after=seconds)`.

**Key Components**:
- **TimerClient**: Worker-side API to acquire/release timers.
- **TimerServer**: Monitors timers and reaps expired workers.
- **TimerRequest**: Message object for client-server communication.

**Provided Implementations**:
- **LocalTimerServer/Client**: Uses `multiprocessing.Queue` for communication.
- **FileTimerServer/Client**: Uses a named pipe (FIFO file).

**Customization**:  
You can implement custom servers/clients by extending the abstract `TimerServer` and `TimerClient` base classes.
