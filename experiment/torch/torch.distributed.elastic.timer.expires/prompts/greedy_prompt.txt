## **Purpose**
Prevent worker processes from hanging indefinitely by setting time limits on critical code blocks. If a worker exceeds the allotted time, it gets terminated and automatically restarted.

- `expires(after=seconds)`: Context manager that sets a countdown timer
- Must call `configure()` once before using timers
- Automatically releases timer when code block completes successfully

### **Timer Server** (Agent-side)
- Monitors active timers and terminates workers with expired timers
- Runs as a daemon thread alongside the training agent

## **Key Features**
- Automatic worker reaping on timeout
- Supports custom server/client implementations
- Per-host timer management (each host runs its own server)
- Debug logging capabilities for expired timers

## **When to Use**
- Distributed training operations that could hang (e.g., collective operations)
- Any code block with potential deadlock or indefinite waiting scenarios
- Ensuring job resilience against stuck workers

The system ensures training jobs can continue despite individual worker failures by automatically restarting timed-out workers.
