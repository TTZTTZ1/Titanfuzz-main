- Each scheduler becomes active at predefined milestone points
- Useful for implementing complex learning rate schedules with distinct phases

**Parameters:**
- `optimizer`: The optimizer to wrap
- `schedulers`: List of schedulers to apply sequentially
- `milestones`: List of epoch indices where transitions occur (length = len(schedulers)-1)

    milestones=[20],  # Switch at epoch 20

**Key Methods:**
- `step()`: Updates learning rate at each epoch
- `get_last_lr()`: Returns current learning rates
- `state_dict()` / `load_state_dict()`: Save/load scheduler state

**Behavior:**
- Before first milestone: uses first scheduler
- At milestone[i]: switches to scheduler[i+1]
- Each scheduler only operates during its assigned interval
- Maintains continuity of learning rate transitions between schedulers
