`torch.jit.script` compiles Python code into TorchScript, a static subset of Python optimized for PyTorch models. It can be applied to functions, `nn.Module` classes, dictionaries, and lists, converting them into serializable and optimizable formats (`ScriptFunction`, `ScriptModule`, `ScriptDict`, or `ScriptList`).

**Key Features:**
- **For functions:** Use as a decorator (`@torch.jit.script`) or directly to compile the function body.
- **For modules:** By default compiles the `forward` method and its dependencies; use `@torch.jit.export` to compile other methods or `@torch.jit.ignore` to exclude them.
- **Type annotation:** Optional `example_inputs` parameter allows explicit argument annotation for functions or modules.
- **Zero-copy overhead:** Scripted dictionaries and lists enable efficient data sharing between Python and TorchScript.

**Use Case:** Enables model deployment in non-Python environments (e.g., mobile, C++) and provides optimization opportunities via TorchScript's static graph.
