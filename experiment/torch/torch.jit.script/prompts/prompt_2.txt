- **For functions:** Use as a decorator (`@torch.jit.script`) or call directly to create a `ScriptFunction`.

- **Selective compilation:** Use `@torch.jit.export` to compile specific methods or `@torch.jit.ignore` to exclude Python-only code.

**Use Cases:**
- Deploying models without Python dependencies
- Performance optimization via graph-based execution
- Serialization for production environments

**Output Types:**
- `nn.Module` → `ScriptModule`
- Function → `ScriptFunction`
- Dictionary/list → TorchScript container objects
