The `torch.nn.GRU` class implements a multi-layer Gated Recurrent Unit (GRU) for sequence processing. It applies the standard GRU equations for reset, update, and new gates to compute hidden states over time.

**Key Parameters:**
- `input_size`: Number of features in input
- `hidden_size`: Number of features in hidden state
- `num_layers`: Number of stacked GRU layers (default: 1)
- `batch_first`: If True, input/output shape is (batch, seq, feature) instead of (seq, batch, feature)
- `bidirectional`: If True, becomes bidirectional GRU (default: False)
- `dropout`: Dropout probability between layers (except last layer)

**Input/Output Shapes:**
- Input: `(L, N, H_in)` or `(N, L, H_in)` if `batch_first=True`
- Hidden state: `(D*num_layers, N, H_out)` 
- Output: `(L, N, D*H_out)` or `(N, L, D*H_out)` if `batch_first=True`
- Where: L=sequence length, N=batch size, D=2 if bidirectional else 1

**Key Notes:**
- Supports packed sequences for variable-length inputs
- Weight initialization: Uniform distribution U(-√k, √k) where k=1/hidden_size
- PyTorch's GRU implementation differs slightly from the original paper for efficiency
- Optimized for GPU with cuDNN when using float16 on V100 GPUs

**Example:**
```python
rnn = nn.GRU(10, 20, 2)  # 2-layer GRU
input = torch.randn(5, 3, 10)  # (seq_len, batch, input_size)
h0 = torch.randn(2, 3, 20)  # (num_layers, batch, hidden_size)
output, hn = rnn(input, h0)
```
