The `torch.nn.GRU` module implements a multi-layer Gated Recurrent Unit (GRU) for sequence processing. It applies the GRU equations to compute hidden states over time, supporting stacked layers, dropout, and bidirectional processing.

**Key Parameters:**
- `input_size`: Number of input features.
- `hidden_size`: Number of hidden state features.
- `num_layers`: Number of stacked GRU layers (default: 1).
- `batch_first`: If `True`, input/output tensors have shape (batch, seq, feature) instead of (seq, batch, feature).
- `bidirectional`: Enables bidirectional processing (default: `False`).
- `dropout`: Dropout probability between layers (except last).

**Inputs:**
- `input`: Sequence tensor of shape `(L, N, H_in)` (or `(N, L, H_in)` if `batch_first=True`).
- `h_0`: Initial hidden state (optional); defaults to zeros.

**Outputs:**
- `output`: Output features for each time step from the last layer.
- `h_n`: Final hidden state for the sequence.

**Notes:**
- Weights are initialized uniformly from `U(-√k, √k)` where `k = 1/hidden_size`.
- PyTorch’s GRU implementation differs slightly from the original paper for efficiency (order of operations in the new gate calculation).
- Supports packed sequences for variable-length inputs via `torch.nn.utils.rnn.pack_padded_sequence()`.

**Example:**
```python
rnn = nn.GRU(10, 20, 2)  # input_size=10, hidden_size=20, num_layers=2
input = torch.randn(5, 3, 10)  # (seq_len, batch, input_size)
h0 = torch.randn(2, 3, 20)     # (num_layers, batch, hidden_size)
output, hn = rnn(input, h0)
```
