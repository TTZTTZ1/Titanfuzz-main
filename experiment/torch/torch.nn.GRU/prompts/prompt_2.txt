The `torch.nn.GRU` module implements a multi-layer Gated Recurrent Unit (GRU) for sequence processing. It applies the GRU equations to compute hidden states step-by-step, supporting stacked layers, dropout, and bidirectional processing.

**Key Parameters:**
- `input_size`: Number of features in input
- `hidden_size`: Number of features in hidden state
- `num_layers`: Number of stacked GRU layers (default: 1)
- `batch_first`: If True, input/output shape is (batch, seq, feature) instead of (seq, batch, feature)
- `bidirectional`: Enables bidirectional processing (default: False)
- `dropout`: Dropout probability between layers (default: 0)

**Input/Output Shapes:**
- Input: `(L, N, H_in)` or `(N, L, H_in)` if `batch_first=True`
- Hidden state: `(D*num_layers, N, H_out)` (optional, zeros if not provided)
- Output: `(L, N, D*H_out)` or `(N, L, D*H_out)` if `batch_first=True`
- Final hidden state: `(D*num_layers, N, H_out)`

Where:
- `N`: batch size, `L`: sequence length
- `D`: 2 if bidirectional else 1
- `H_in`: input_size, `H_out`: hidden_size

**Important Notes:**
- Weights initialized from uniform distribution U(-√k, √k) where k = 1/hidden_size
- PyTorch's GRU implementation differs slightly from the original paper for efficiency
- Supports packed sequences for variable-length inputs
- Can leverage cuDNN optimizations for GPU acceleration

**Example:**
```python
rnn = nn.GRU(10, 20, 2)  # 10 input features, 20 hidden features, 2 layers
input = torch.randn(5, 3, 10)  # (seq_len, batch, input_size)
h0 = torch.randn(2, 3, 20)  # (num_layers, batch, hidden_size)
output, hn = rnn(input, h0)
```
