`Dropout3d` randomly zeroes entire 3D feature map channels (not individual elements) during training to prevent overfitting. It is designed for 3D convolutional inputs where spatial correlations are strong, promoting independence between channels.

**Key details:**
- **Input shape:** `(N, C, D, H, W)` or `(C, D, H, W)`
- **Output shape:** Same as input
- **Parameters:**  
  - `p` (float, default 0.5): Probability of zeroing a channel  
  - `inplace` (bool, default False): Perform operation in-place

**Use case:**  
Use with `nn.Conv3d` outputs instead of standard dropout when adjacent voxels/pixels are highly correlated (e.g., early convolution layers). This avoids merely reducing the learning rate and better regularizes activations.

**Example:**
```python
m = nn.Dropout3d(p=0.2)
input = torch.randn(20, 16, 4, 32, 32)  # (batch, channels, depth, height, width)
output = m(input)  # Random channels zeroed
```
