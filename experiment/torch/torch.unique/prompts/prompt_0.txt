`torch.unique` returns the distinct elements from an input tensor, optionally sorting them and providing additional mapping information.  

**Key parameters:**  
- `sorted`: If `True`, output is sorted in ascending order (note: currently always sorted internally).  
- `return_inverse`: Returns indices mapping original elements to unique output positions.  
- `return_counts`: Returns frequency counts for each unique element.  
- `dim`: If specified, finds unique slices along that dimension; otherwise flattens the tensor.  

**Returns:**  
- `output`: Unique elements (or slices).  
- `inverse_indices` (optional): Same shape as input, showing where each original element appears in `output`.  
- `counts` (optional): Number of occurrences per unique element.  

**Note:**  
- Removes all duplicate values, not just consecutive ones (unlike `torch.unique_consecutive`).  
- Internally sorts the tensor even if `sorted=False` in current implementations, so for pre‑sorted data, `torch.unique_consecutive` is more efficient.  

**Example use:**  
```python
output, inv, counts = torch.unique(tensor, return_inverse=True, return_counts=True)
```
