`torch.nn.LSTMCell` implements a single LSTM cell for processing sequential data step-by-step. It takes an input tensor and the previous hidden/cell states, then computes the next hidden and cell states using the standard LSTM equations with sigmoid and tanh activations.

**Key Parameters:**
- `input_size`: Number of features in the input
- `hidden_size`: Number of features in the hidden state
- `bias`: Whether to include bias terms (default: True)

**Input/Output:**
- **Input:** `(input, (h_0, c_0))` where dimensions can be batched or single
- **Output:** `(h_1, c_1)` - updated hidden and cell states
- If initial states are not provided, they default to zero

**Internal Weights:**
- `weight_ih`: Input-hidden weights (shape: `[4*hidden_size, input_size]`)
- `weight_hh`: Hidden-hidden weights (shape: `[4*hidden_size, hidden_size]`)
- Corresponding bias terms if `bias=True`

**Usage Example:**
```python
rnn = nn.LSTMCell(10, 20)  # 10 input features, 20 hidden features
for input_step in sequence:
    hx, cx = rnn(input_step, (hx, cx))
```

This module is designed for manual iteration over sequence steps, unlike `nn.LSTM` which processes entire sequences automatically.
