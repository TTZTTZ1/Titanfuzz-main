**Key Components:**
- **`torch.profiler.profile`**: Main context manager for profiling, supporting configurable activities (CPU/CUDA/XPU), memory tracking, shape recording, and FLOPs estimation.
- **`schedule()`**: Controls profiling cycles with wait, warmup, active, and repeat phases for long-running jobs.
- **`tensorboard_trace_handler()`**: Exports traces for visualization in TensorBoard.
- **`_KinetoProfile`**: Lower-level API wrapping autograd profiling.

**Core Features:**
- Tracks operator execution time and device kernel activity
- Records tensor shapes and memory allocation/deallocation
- Captures stack traces and module hierarchies (TorchScript only)
- Estimates FLOPs for specific operations
- Supports export to Chrome JSON format and memory timelines

**Usage Example:**
```python
with torch.profiler.profile(activities=[ProfilerActivity.CPU, ProfilerActivity.CUDA]) as prof:
    # Code to profile
    model(input)

```

**Additional Tools:**
- **ITT APIs**: Intel Instrumentation and Tracing Technology for marking events and ranges
- **Execution Trace Observer**: Enables graph-based workload representation

The profiler replaces legacy `torch.autograd.profiler` and provides more detailed insights into PyTorch model performance.
