1. **`torch.profiler.profile`** – Main context manager for profiling
   - Supports CPU, CUDA, and XPU activities
   - Configurable via `schedule()` for periodic profiling
   - Can export traces for TensorBoard with `tensorboard_trace_handler()`

2. **`_KinetoProfile`** – Lower-level profiler wrapper (experimental)

3. **Profiling Features:**
   - Record operator input shapes (`record_shapes=True`)

   - Module hierarchy recording (`with_modules=True`)

4. **Output Methods:**
   - `export_chrome_trace()` – Chrome JSON format
   - `export_stacks()` – Save stack traces
   - `key_averages()` – Generate summary tables

5. **ITT APIs** – Intel Instrumentation and Tracing Technology for marking events and ranges

**Basic Usage:**
```python
with torch.profiler.profile(activities=[ProfilerActivity.CPU, ProfilerActivity.CUDA]) as prof:
    # Code to profile
    model(input)
    
print(prof.key_averages().table(sort_by="cuda_time_total"))
```

The profiler replaces legacy `torch.autograd.profiler` and provides both simple continuous profiling and advanced scheduled profiling for long-running jobs.
