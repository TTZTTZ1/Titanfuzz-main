The `torch.distributed` package provides PyTorch's distributed communication primitives for multi-process parallelism across multiple machines. It supports synchronous distributed training and is essential for scaling deep learning models.

## Key Features
- **Backends**: Supports Gloo (CPU/GPU), NCCL (CUDA GPU), XCCL (XPU GPU), and MPI (CPU/GPU with CUDA support if available)
- **Initialization**: Multiple methods including TCP, shared file-system, and environment variables
- **Communication**: Point-to-point (send/recv) and collective operations (broadcast, all_reduce, etc.)
- **Groups**: Fine-grained communication groups via `new_group()`
- **DeviceMesh**: Higher-level abstraction for managing process groups in multi-dimensional device topologies
- **Key-Value Store**: Distributed store implementations (TCPStore, FileStore, HashStore)

## Common Usage Patterns

### Initialization
```python
import torch.distributed as dist

# Environment variable initialization (default)
dist.init_process_group(backend="nccl")  # For GPU training
dist.init_process_group(backend="gloo")  # For CPU training

# Or specify initialization explicitly
dist.init_process_group(
    backend="nccl",
    init_method="tcp://10.1.1.20:23456",
    rank=args.rank,
    world_size=4
)
```

### Collective Operations
```python
# Synchronous all-reduce
dist.all_reduce(tensor, op=dist.ReduceOp.SUM)

# Asynchronous all-gather
work = dist.all_gather(tensor_list, tensor, async_op=True)
work.wait()

# Broadcast
dist.broadcast(tensor, src=0)
```

### Point-to-Point Communication
```python
# Synchronous send/recv
dist.send(tensor, dst=1)
dist.recv(tensor, src=0)

# Asynchronous
req = dist.isend(tensor, dst=1)
req.wait()
```

### DeviceMesh (for advanced parallelism)
```python
from torch.distributed.device_mesh import init_device_mesh

# Create a 2D device mesh
mesh = init_device_mesh(
    "cuda",
    mesh_shape=(2, 4),
    mesh_dim_names=("dp", "tp")
)
```

## Best Practices
1. **Backend Selection**:
   - Use NCCL for CUDA GPU training
   - Use Gloo for CPU training
   - Use XCCL for XPU GPU training

2. **Initialization**: Always call `dist.init_process_group()` before any distributed operations

3. **Cleanup**: Call `dist.destroy_process_group()` to clean up resources

4. **Debugging**: Use `TORCH_DISTRIBUTED_DEBUG` environment variable for debugging

5. **Performance**: Use asynchronous operations (`async_op=True`) to overlap communication with computation

## Environment Variables
- `MASTER_ADDR`, `MASTER_PORT`: For TCP initialization
- `WORLD_SIZE`, `RANK`: Process group configuration
- `NCCL_SOCKET_IFNAME`, `GLOO_SOCKET_IFNAME`: Network interface selection
- `TORCH_DISTRIBUTED_DEBUG`: Debugging level (OFF/INFO/DETAIL)

The package enables efficient distributed training with support for various network configurations and hardware setups, making it suitable for both single-machine multi-GPU and multi-machine distributed training scenarios.
