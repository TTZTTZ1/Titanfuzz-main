The `torch.distributed` package provides PyTorch support for distributed training across multiple processes and machines. It enables synchronous distributed training with primitives for communication and coordination.

## Key Components

### Backends
- **Gloo**: Recommended for CPU training, supports Ethernet/InfiniBand
- **NCCL**: Recommended for CUDA GPU training, optimized for NVIDIA GPUs
- **XCCL**: For XPU GPU training
- **MPI**: Optional backend requiring MPI installation

### Initialization Methods
1. **Environment Variables** (default): `MASTER_ADDR`, `MASTER_PORT`, `WORLD_SIZE`, `RANK`
2. **TCP**: Specify IP address and port
3. **Shared File System**: Use a shared file for coordination

### Core Functions
- **Process Management**: `init_process_group()`, `destroy_process_group()`
- **Collective Operations**: `all_reduce()`, `broadcast()`, `all_gather()`, `scatter()`, etc.
- **Point-to-Point**: `send()`, `recv()`, `isend()`, `irecv()`
- **Groups**: Create subgroups with `new_group()`
- **DeviceMesh**: Higher-level abstraction for multi-dimensional device layouts

### Key Features
- **Synchronous & Asynchronous Operations**: Most collectives support both modes
- **Distributed DataParallel**: Wrapper for synchronous distributed training
- **Key-Value Stores**: `TCPStore`, `FileStore`, `HashStore` for coordination
- **Debugging Tools**: `monitored_barrier()`, `TORCH_DISTRIBUTED_DEBUG`, Python breakpoint support

### Best Practices
- Use NCCL for GPU training, Gloo for CPU training
- Call `destroy_process_group()` for cleanup
- Set appropriate environment variables for network interface selection
- Use `async_op=True` for overlapping communication with computation

The package supports multi-node training with automatic rank assignment, process group management, and various communication patterns essential for scaling PyTorch models across multiple devices and machines.
