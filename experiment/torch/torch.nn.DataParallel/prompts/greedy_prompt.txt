# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
`torch.nn.DataParallel` enables data parallelism by splitting input batches across multiple GPUs. It replicates the model on each device, processes data in parallel during the forward pass, and aggregates gradients during backpropagation.

**Key Points:**
- **Usage:** Wrap a model to parallelize across specified GPUs (default: all available).
- **Input Handling:** Tensors are split along the batch dimension; other data types are shallow-copied or shared.
- **Requirements:** The batch size must exceed the number of GPUs, and the model's parameters must reside on the primary device (`device_ids[0]`).
- **Limitations:** 
  - Updates to module state during forward passes are lost (except for in-place updates on the primary device).
  - Hooks are executed per device, without guaranteed global order.
  - Returns a vector of outputs if the model outputs a scalar.
- **Recommendation:** Prefer `DistributedDataParallel` for better performance and scalability, even on single nodes.

**Example:**
```python
net = torch.nn.DataParallel(model, device_ids=[0, 1, 2])
output = net(input_var)  # input_var can be on CPU or GPU
```
---GENERAL SUMMARY END---
