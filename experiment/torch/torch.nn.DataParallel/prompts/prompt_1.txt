---GENERAL SUMMARY START---
`torch.nn.DataParallel` is a container that enables data parallelism by splitting input across multiple GPUs along the batch dimension. It replicates the module on each specified device, processes portions of the input in parallel during the forward pass, and aggregates gradients during backpropagation.

**Key Points:**
- **Usage:** Wrap any module to parallelize across GPUs (e.g., `DataParallel(model, device_ids=[0, 1, 2])`).
- **Input Handling:** Tensors are scattered on a given dimension (default dim=0); other types (tuples, lists, dicts) are shallow-copied.
- **Requirements:** The module’s parameters and buffers must reside on `device_ids[0]` before wrapping.
- **Limitations & Warnings:**
  - Recommended to use `DistributedDataParallel` instead for better performance.
  - Batch size should exceed the number of GPUs.
  - In-place updates in `forward()` are lost except for parameters/buffers on device 0.
  - Hooks are invoked per device, and order isn’t guaranteed across devices.
  - Returns a vector of outputs if the module outputs a scalar.

**Example:**
```python
net = torch.nn.DataParallel(model, device_ids=[0, 1, 2])
output = net(input_var)  # input_var can be on CPU or GPU
```
