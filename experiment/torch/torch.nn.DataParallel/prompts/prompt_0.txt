`torch.nn.DataParallel` is a container that enables data parallelism by splitting input across multiple GPUs along the batch dimension. It replicates the module on each specified device, processes portions of the input in parallel, and aggregates gradients during backpropagation.

**Key Features:**
- Automatically distributes input tensors across GPUs (default along dim=0).
- Replicates the model on each device; only in-place updates on the primary device (`device_ids[0]`) persist.
- Supports arbitrary input types (tensors are scattered, sequences/dicts are shallow copied).

**Important Warnings:**
- **Not recommended for production:** Prefer `DistributedDataParallel` for better performance and scalability.
- Module state updates in `forward` are lost unless performed in-place on the primary device.
- Hooks are invoked per device, with no guaranteed global execution order.
- Returns a vector of scalars if the module outputs a 0‑D tensor.

**Usage:**
```python
net = torch.nn.DataParallel(model, device_ids=[0, 1, 2])
output = net(input_var)  # input_var can be on CPU or any GPU
```

**Requirements:**
- Batch size must exceed the number of GPUs.
- Module parameters and buffers must reside on `device_ids[0]` before wrapping.
