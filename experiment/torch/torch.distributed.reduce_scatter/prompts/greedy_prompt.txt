- **Initialization**: Must call `init_process_group()` or `init_device_mesh()` before any distributed operations.

## **Key Features**
- **Collective Operations**: `broadcast`, `all_reduce`, `all_gather`, `scatter`, `reduce_scatter`, `all_to_all`, `barrier`

- **Key-Value Stores**: `TCPStore`, `FileStore`, `HashStore` for process coordination

## **Best Practices**
- Use NCCL backend for CUDA GPU training
- Use Gloo backend for CPU training
- Call `destroy_process_group()` for cleanup
- Set `async_op=True` for asynchronous operations to overlap computation/communication
- Use `DeviceMesh` for complex multi-dimensional parallelism

## **Debugging Tools**
- `TORCH_DISTRIBUTED_DEBUG`: Enable detailed logging (`INFO`/`DETAIL`)
- `monitored_barrier()`: Synchronization with timeout detection
- `breakpoint()`: Debug single rank while others wait
- HTTP debug server for runtime inspection

## **Performance Optimizations**
- **Copy Engine Collectives**: Offload communication to GPU DMA engines (NCCL 2.28+)
- **Environment Variables**: `NCCL_SOCKET_IFNAME`, `GLOO_SOCKET_IFNAME` for network tuning
- **Asynchronous Operations**: Overlap communication with computation

## **Important Notes**
- Supports Linux (stable), macOS (stable), Windows (prototype)
- Each process maintains independent Python interpreter and optimizer
- Object collectives (pickle-based) have performance limitations
- Third-party backends can be registered via `Backend.register_backend()`

This package forms the foundation for distributed training wrappers like `DistributedDataParallel` and enables scalable model training across multiple devices and nodes.
