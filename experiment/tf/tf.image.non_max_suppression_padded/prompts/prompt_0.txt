`tf.image.non_max_suppression_padded` performs greedy non-maximum suppression (NMS) to select high-scoring bounding boxes while suppressing overlapping ones. **Key Features:**
- **Input:** Batched or single set of boxes (`[..., num_boxes, 4]`) and scores (`[..., num_boxes]`).
- **Output:** A tuple (`indices_padded`, `num_valid`), where:
  - `indices_padded`: Padded indices of selected boxes (shape `[..., max_output_size]`).
  - `num_valid`: Number of valid indices per batch item (shape `[...]`).
- **Parameters:** Supports IoU threshold, score threshold, and options for sorted input, coordinate canonicalization, and tiling for parallelism.
- **Use Case:** Primarily for batched processing in object detection pipelines, ensuring consistent output dimensions across batch elements.

**Example Usage:**
```python
selected_padded, num_valid = tf.image.non_max_suppression_padded(
    boxes, scores, max_output_size=100, pad_to_max_output_size=True)
# Extract valid indices and corresponding boxes
selected_indices = tf.slice(selected_padded, [0], num_valid)
selected_boxes = tf.gather(boxes, selected_indices)
```

**Note:** `pad_to_max_output_size=True` is required for batched inputs to avoid errors.
