**Key features:**
- **Input:** A string tensor containing the raw image bytes.
- **Output:** A tensor of the specified `dtype` (default `tf.uint8`).
- **Shape behavior:**  
  - For BMP, JPEG, and PNG: Returns a 3D tensor `[height, width, channels]`.  
  - For GIFs: By default (`expand_animations=True`), returns a 4D tensor `[num_frames, height, width, 3]` to include all animation frames. If `expand_animations=False`, it returns a 3D tensor with only the first frame (same as static images).
- **Parameters:**  
  - `channels`: Optional number of color channels (e.g., 1 for grayscale, 3 for RGB).  
  - `dtype`: Desired output tensor data type.  
  - `expand_animations`: Controls whether animated GIFs are expanded to 4D or truncated to the first frame.

**Notes:**
- Animated GIFs that don’t fill the entire canvas are padded with zeros (black) in unoccupied areas for the first frame; subsequent frames reuse the previous frame to fill gaps.
- Raises a `ValueError` if the channel count is incompatible with the image.

**Use case:** Simplifies decoding multiple image formats in a single operation, especially useful when handling mixed image types in a pipeline.
