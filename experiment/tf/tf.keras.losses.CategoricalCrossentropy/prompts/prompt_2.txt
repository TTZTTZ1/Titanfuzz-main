`tf.keras.losses.CategoricalCrossentropy` is a loss function used for multi-class classification tasks where labels are provided in one-hot encoded format. It measures the cross-entropy between true labels and predicted probabilities.

**Key Features:**
- **Input Format**: Expects both `y_true` (true labels) and `y_pred` (predictions) to have shape `[batch_size, num_classes]`.
- **Parameters**:
  - `from_logits`: Set to `True` if `y_pred` are logits (unnormalized scores); default is `False` (probabilities).
  - `label_smoothing`: Smooths labels by redistributing a small value from true labels to other classes (range [0, 1]).
  - `axis`: Axis along which to compute cross-entropy (default is `-1`, the last axis).
  - `reduction`: Loss reduction type (e.g., `"sum_over_batch_size"`, `"sum"`, or `None`).

**Usage Example:**
```python
import tensorflow as tf

# One-hot encoded labels and predictions
y_true = [[0, 1, 0], [0, 0, 1]]
y_pred = [[0.05, 0.95, 0], [0.1, 0.8, 0.1]]

# Instantiate loss
cce = tf.keras.losses.CategoricalCrossentropy()
loss = cce(y_true, y_pred)  # Output: ~1.177
```

**Common Use Case**:  
Compile a model for multi-class classification:
```python

              loss=tf.keras.losses.CategoricalCrossentropy())
```

**Note**: For integer labels (not one-hot), use `SparseCategoricalCrossentropy` instead.
