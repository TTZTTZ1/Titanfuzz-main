`tf.data.experimental.index_table_from_dataset` creates a lookup table that maps keys from a given dataset to integer indices. It assigns a unique index to each vocabulary item in the dataset and can handle out-of-vocabulary (OOV) keys by hashing them into a specified number of OOV buckets or assigning a default value. This is useful for building vocabulary-based feature transformations in TensorFlow pipelines.

**Key parameters:**
- `dataset`: The dataset containing the vocabulary keys.
- `num_oov_buckets`: Number of buckets for OOV keys (if >0, OOV keys are hashed into these buckets).
- `default_value`: Value returned for OOV keys when `num_oov_buckets = 0` (default is -1).
- `vocab_size`: Optional known vocabulary size.
- `key_dtype`: Data type of the keys (string or integer).
