`CSVLogger` is a Keras callback that saves training metrics (e.g., loss, accuracy) to a CSV file after each epoch. It is useful for tracking and later analyzing model performance over time.

**Key Features:**
- Logs epoch-level metrics, including validation metrics (prefixed with `val_`).
- Supports custom separators (default: comma) and file appending for resuming training.
- Handles various data types (e.g., NumPy arrays) by converting them to strings.

**Basic Usage:**
```python
csv_logger = tf.keras.callbacks.CSVLogger(
    filename='training.log',
    separator=',',
    append=False  # Overwrite existing file
)
model.fit(X_train, y_train, callbacks=[csv_logger])
```

**Primary Methods:**
- `on_epoch_end()`: Writes metrics to the CSV file at the end of each epoch.
- `on_train_begin()`: Initializes the CSV file and writes the header.
- `on_train_end()`: Closes the file handler.

**Notes:**
- Only logs epoch-level results, not batch-level data.
- Inherits from `tf.keras.callbacks.Callback`, so it can be integrated seamlessly into the training loop.
