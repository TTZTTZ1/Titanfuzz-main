The `tf.keras.layers.Bidirectional` wrapper enables bidirectional processing for RNN layers (e.g., LSTM, GRU) by running the input sequence both forward and backward and merging the outputs.  

**Key features:**  
- **Merge modes:** Supports `"concat"` (default), `"sum"`, `"mul"`, `"ave"`, or `None` (returns separate outputs).  
- **Custom backward layer:** Allows specifying a separate backward RNN layer with matching properties.  
- **State handling:** When using `initial_state`, the first half is passed to the forward RNN, the second half to the backward RNN.  

**Important notes:**  
- The wrapped RNN must support `go_backwards`, `return_sequences`, `return_state`, and serialization.  
- Instantiating `Bidirectional` on an existing RNN does **not** reuse its weights—weights are freshly initialized.  

**Example usage:**  
```python
model = Sequential([
    Input(shape=(5, 10)),
    Bidirectional(LSTM(10, return_sequences=True)),
    Bidirectional(LSTM(10)),
    Dense(5, activation="softmax")
])
```
