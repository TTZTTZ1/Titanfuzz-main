- **Sensitivity (Recall)**: `TP / (TP + FN)` – proportion of actual positives correctly identified.
- **Specificity**: `TN / (TN + FP)` – proportion of actual negatives correctly identified.

**How It Works:**
1. Internally tracks true/false positives and negatives across multiple thresholds (`num_thresholds` defaults to 200).
2. Finds the threshold where specificity meets or exceeds the specified target.
3. Returns the sensitivity at that threshold.

**Main Parameters:**
- `specificity`: Target specificity value (float between 0 and 1).
- `num_thresholds`: Number of thresholds to evaluate (more → more precise).
- `class_id`: Optional class index for multi-class classification.
- `sample_weight`: Supports weighted examples.

**Usage Example:**
```python

print(metric.result())  # Get best sensitivity at ≥80% specificity
```

Useful when you need to **prioritize detecting positives** (high sensitivity) while maintaining a **minimum specificity requirement**, such as in medical diagnostics or fraud detection.
