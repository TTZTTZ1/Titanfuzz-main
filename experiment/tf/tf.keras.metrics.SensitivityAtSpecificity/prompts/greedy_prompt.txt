**Key Features:**
- **Purpose:** Finds the optimal trade-off between sensitivity (true positive rate) and specificity (true negative rate).
- **Core Mechanism:** Internally tracks true positives, true negatives, false positives, and false negatives across multiple thresholds to determine the sensitivity at the target specificity.
- **Parameters:**
  - `specificity`: Target specificity value (between 0 and 1).
  - `num_thresholds`: Number of thresholds to evaluate (default 200).
  - `class_id`: Optional class index for multi-class classification.
- **Usage:** Can be used directly via `update_state()` and `result()` methods, or integrated into a model's `compile()` step for automatic evaluation during training.
