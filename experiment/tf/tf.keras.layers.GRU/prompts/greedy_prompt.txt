- **Gating Mechanism:** Uses update and reset gates to control information flow, making it effective for learning long-term dependencies.
- **Performance Optimization:** Automatically selects between a fast cuDNN-based implementation (on GPU) and a backend-native version (on CPU) based on runtime hardware and parameter compatibility.
- **Flexible Outputs:** Configurable to return either the final output, the full output sequence, or the hidden state via `return_sequences` and `return_state` parameters.

**Main Parameters:**
- `units`: Dimensionality of the output space.
- `activation`/`recurrent_activation`: Activation functions for the output and recurrent steps (default: `tanh` and `sigmoid`).
- `dropout`/`recurrent_dropout`: Fraction of units to drop for regularization.
- `return_sequences`: Returns the full sequence if `True`, otherwise only the last output.
- `reset_after`: Determines the order of gate application (default `True` for cuDNN compatibility).

**Typical Use:**  
Processes 3D input tensors of shape `(batch_size, timesteps, features)` and outputs sequences or final states based on configuration. Commonly applied in tasks like time-series prediction, natural language processing, and sequential recommendation.
