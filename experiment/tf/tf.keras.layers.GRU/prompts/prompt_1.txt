---GENERAL SUMMARY START---
The `tf.keras.layers.GRU` layer implements a Gated Recurrent Unit (GRU), a type of recurrent neural network (RNN) designed to capture sequential dependencies while mitigating the vanishing gradient problem. It is commonly used for tasks involving sequential data like time series, text, and speech.

**Key Features:**
- **CuDNN Optimization:** Automatically uses a faster GPU-accelerated implementation when specific conditions are met (e.g., `activation='tanh'`, `recurrent_activation='sigmoid'`, no dropout).
- **Two Variants:** Supports the modern default (`reset_after=True`, CuDNN-compatible) and the original GRU formulation.
- **Flexible Output:** Can return either the final output, the full sequence of outputs, or the final hidden state via `return_sequences` and `return_state`.
- **Regularization:** Supports dropout and recurrent dropout to prevent overfitting.
- **State Management:** Allows stateful operation across batches for streaming data.

**Main Parameters:**
- `units`: Dimensionality of the output/hidden state.
- `activation`/`recurrent_activation`: Activation functions for the output and recurrent steps (default: `tanh` and `sigmoid`).
- `dropout`/`recurrent_dropout`: Fraction of units to drop for input and recurrent transformations.
- `return_sequences`: If `True`, returns all time steps; otherwise, returns only the last output.
- `return_state`: If `True`, returns the hidden state in addition to the output.
- `stateful`: If `True`, maintains the hidden state across batches.

**Input/Output:**
- **Input:** 3D tensor of shape `(batch_size, timesteps, features)`.
- **Output:** 
  - If `return_sequences=False`: 2D tensor `(batch_size, units)`.
  - If `return_sequences=True`: 3D tensor `(batch_size, timesteps, units)`.
  - If `return_state=True`: Also returns the final hidden state.

**Typical Use Cases:** Sequence modeling, time series prediction, natural language processing (e.g., machine translation, text generation), and recommendation systems.
---GENERAL SUMMARY END---
