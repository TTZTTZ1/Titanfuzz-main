`tf.keras.layers.GRUCell` is a single-step RNN cell implementing the Gated Recurrent Unit (GRU). It processes one timestep at a time, unlike `tf.keras.layers.GRU` which handles entire sequences.

**Key Features:**
- **Purpose:** Used within `tf.keras.layers.RNN` to process sequential data step-by-step.
- **Core Parameters:**
  - `units`: Dimensionality of the output space.
  - `activation`: Activation for the output (default: `tanh`).
  - `recurrent_activation`: Activation for recurrent gates (default: `sigmoid`).
  - `dropout` / `recurrent_dropout`: Optional dropout for inputs and recurrent states.
  - `reset_after`: GRU convention flag (default `True`, cuDNN compatible).

**Call Arguments:**
- `inputs`: 2D tensor of shape `(batch, features)`.
- `states`: Previous hidden state, shape `(batch, units)`.
- `training`: Boolean for dropout behavior.

**Example Usage:**
```python
cell = tf.keras.layers.GRUCell(units=4)
rnn = tf.keras.layers.RNN(cell)
output = rnn(inputs)  # Processes full sequence
```

**Main Methods:**
- `call()`: Performs one GRU step.
- `get_initial_state()`: Returns initial hidden state.
- Dropout mask utilities for training.

This cell is designed for custom RNN implementations where fine-grained control over timestep processing is needed.
