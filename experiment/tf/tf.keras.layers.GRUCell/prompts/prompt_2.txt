`tf.keras.layers.GRUCell` is a single-step RNN cell implementing the Gated Recurrent Unit (GRU) architecture. It processes one timestep at a time, making it suitable for use within `keras.layers.RNN` for sequential data handling.

**Key Features:**
- **Purpose:** Handles one timestep of input within a sequence, unlike `keras.layers.GRU` which processes entire sequences.
- **Core Parameters:** 
  - `units`: Dimensionality of the output space.
  - `activation`/`recurrent_activation`: Activation functions for the output and recurrent gates (default: `tanh` and `sigmoid`).
  - `dropout`/`recurrent_dropout`: Optional dropout rates for inputs and recurrent states.
  - `reset_after`: Determines gate order (default `True`, cuDNN-compatible).

**Usage:**  
Pass `GRUCell` to `keras.layers.RNN` to build a recurrent layer. It accepts 2D inputs `(batch, features)` and a previous state `(batch, units)`, returning the output and new state for the current timestep.

**Example:**
```python
cell = keras.layers.GRUCell(units=4)
rnn = keras.layers.RNN(cell, return_sequences=True)
output = rnn(inputs)  # Processes full sequence
```
