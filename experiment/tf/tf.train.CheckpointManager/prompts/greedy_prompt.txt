`tf.train.CheckpointManager` is a utility for managing TensorFlow training checkpoints. It automatically handles saving, keeping a specified number of recent checkpoints, and optionally preserving older checkpoints at regular time intervals.

**Key Features:**
- **Automatic Cleanup:** Keeps only the `max_to_keep` most recent checkpoints by default, deleting older ones.
- **Time-based Preservation:** Optionally preserves checkpoints every `keep_checkpoint_every_n_hours` for long-term archival.
- **Step-based Saving:** Can be configured with `step_counter` and `checkpoint_interval` to save only after a minimum number of steps.
- **Easy Restoration:** Provides `latest_checkpoint` property to easily resume training from the most recent checkpoint.
- **Custom Initialization:** Allows an `init_fn` for custom setup when no existing checkpoint is found.

**Core Methods:**
- `save()`: Creates and manages a new checkpoint.
- `restore_or_initialize()`: Restores from the latest checkpoint or runs `init_fn` if none exists.
- `sync()`: Waits for any ongoing save/restore operations.

**Typical Usage:**
```python
manager = tf.train.CheckpointManager(checkpoint, directory, max_to_keep=5)
manager.restore_or_initialize()  # Resume training
for step in range(num_steps):
    # Training loop...
    manager.save()  # Periodically save
```
