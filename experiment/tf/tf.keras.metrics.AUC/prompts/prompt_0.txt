`tf.keras.metrics.AUC` approximates the Area Under the Curve (AUC) for ROC or Precision-Recall (PR) curves, evaluating binary classifier performance across all operational thresholds. It discretizes predictions into buckets using a specified number of thresholds (default 200) and computes AUC via Riemann summation.

**Key Features:**
- **Curve Type:** Supports `'ROC'` (default) or `'PR'` curves.
- **Multilabel Support:** When `multi_label=True`, computes AUC per label and averages (weighted optionally via `label_weights`).
- **Logits Handling:** Set `from_logits=True` if predictions are logits (not probabilities).
- **Custom Thresholds:** Provide a list via `thresholds` to override automatic threshold generation.
- **Summation Methods:** Choose `'interpolation'` (default), `'minoring'`, or `'majoring'` to control approximation accuracy.

**Usage Example:**
```python
# For probability outputs
model.compile(loss='binary_crossentropy', metrics=[tf.keras.metrics.AUC()])

# For logit outputs
model.compile(
    loss=tf.keras.losses.BinaryCrossentropy(from_logits=True),
    metrics=[tf.keras.metrics.AUC(from_logits=True)]
)
```

**Important Notes:**
- Predictions should be uniformly distributed in `[0, 1]` (when `from_logits=False`) for accurate approximation.
- Not suitable for multi-class classification directly (use `multi_label=False` to flatten data).
- Automatically manages state variables (`true_positives`, `false_positives`, etc.) for accumulating statistics across batches.
