`tf.keras.metrics.AUC` approximates the area under the ROC or Precision-Recall curve for binary or multilabel classification. It discretizes predictions into thresholds to compute true/false positives/negatives, then calculates the area via Riemann summation.

**Key Parameters:**
- `num_thresholds`: Number of thresholds for discretization (default 200).
- `curve`: Type of curve (`'ROC'` or `'PR'`).
- `summation_method`: Riemann sum method (`'interpolation'`, `'minoring'`, `'majoring'`).
- `thresholds`: Custom thresholds (overrides `num_thresholds`).
- `multi_label`: If `True`, computes AUC per label and averages.
- `from_logits`: Whether predictions are logits (set to match loss).

**Usage:**
- Works with `compile()` for model evaluation.
- Handles sample weights and supports stateful accumulation across batches.

**Example:**
```python
metric = tf.keras.metrics.AUC(num_thresholds=3)
metric.update_state([0, 0, 1, 1], [0, 0.5, 0.3, 0.9])
print(metric.result())  # Output: 0.75
```

This metric is useful for evaluating classifier performance across all decision thresholds, especially with imbalanced data.
