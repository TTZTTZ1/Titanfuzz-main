The `tf.keras.metrics.AUC` class computes the **Area Under the Curve (AUC)** for **ROC** (default) or **Precision-Recall (PR)** curves, evaluating binary classifier performance across all decision thresholds.

**Key Features:**
- Approximates AUC using a Riemann sum over discretized thresholds.
- Handles both **single-label** and **multi-label** classification.
- Supports **probability outputs** or **logits** (via `from_logits`).
- Allows custom thresholds or automatic linear spacing (`num_thresholds`).
- Provides interpolation methods (`summation_method`) to control approximation accuracy.

**Main Parameters:**
- `num_thresholds`: Number of thresholds for discretization (default 200).
- `curve`: Type of curve—`'ROC'` or `'PR'`.
- `multi_label`: If `True`, computes AUC per label and averages; if `False`, flattens multi-label data.
- `thresholds`: Optional custom threshold list (overrides `num_thresholds`).
- `from_logits`: Set `True` if predictions are logits (not probabilities).

**Usage Example:**
```python
# Binary classification with probability output
model.compile(
    loss='binary_crossentropy',
    metrics=[tf.keras.metrics.AUC()]
)

# Multi-label classification
auc_metric = tf.keras.metrics.AUC(multi_label=True, num_labels=5)
```

**How It Works:**
1. Accumulates **true/false positives/negatives** across thresholds.
2. Computes AUC by interpolating between threshold points.
3. Returns a scalar AUC value representing classifier quality.

This metric is especially useful for **imbalanced datasets**, as it evaluates performance across all classification thresholds rather than at a single fixed point.
