`tf.data.experimental.OptimizationOptions` is a class used to configure optimization settings for TensorFlow datasets. It allows fine-grained control over various performance-enhancing transformations applied during dataset processing.

**Key Usage:**
- Accessed via `options.experimental_optimization` in `tf.data.Options`.
- Optimizations are applied when calling `dataset.with_options(options)`.

**Main Attributes (Defaults):**
- **Enabled by default:** `noop_elimination`, `map_and_batch_fusion`, `map_parallelization`, `parallel_batch`, `shuffle_and_repeat_fusion`, `inject_prefetch`.
- **Disabled by default:** `filter_fusion`, `filter_parallelization`, `map_and_filter_fusion`, `map_fusion`, `seq_interleave_prefetch`.
- `apply_default_optimizations`: When `False`, only explicitly enabled optimizations are applied.

**Common Optimizations:**
- **Fusion:** Combines operations like map+batch or shuffle+repeat to reduce overhead.
- **Parallelization:** Runs stateless map/filter operations concurrently.
- **Elimination:** Removes no-operation transformations.

**Example:**
```python
options = tf.data.Options()
options.experimental_optimization.noop_elimination = True
options.experimental_optimization.apply_default_optimizations = False

```

This provides a flexible way to tune dataset performance by selectively enabling or disabling specific optimizations.
