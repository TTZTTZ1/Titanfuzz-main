- **`value`**: 4-D tensor of shape `[batch, height, width, channels]`.
- **`pooling_ratio`**: List of floats ≥ 4, where the first and last elements must be 1.0 (no pooling on batch/channels), and intermediate values define pooling ratios for height and width (e.g., `[1.0, 1.44, 1.73, 1.0]`).
- **`pseudo_random`**: Controls randomness in pooling sequence generation.
- **`overlapping`**: If `True`, boundary values can be shared between adjacent pooling cells.
- **`deterministic`**: Ensures fixed pooling regions for reproducibility (e.g., in tests).

**Returns:**  
A tuple of three tensors:  
1. **`output`**: Pooled result (same type as `value`).  
2. **`row_pooling_sequence`**: `int64` tensor storing row pooling indices.  
3. **`col_pooling_sequence`**: `int64` tensor storing column pooling indices.

This operation is useful for downsampling with non-integer ratios while preserving channel and batch dimensions.
