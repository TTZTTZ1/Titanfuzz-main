The `tf.keras.callbacks.ModelCheckpoint` callback saves a Keras model or its weights during training, allowing you to resume training or deploy the best-performing model later. It provides flexible options to control when and what to save based on performance metrics.

**Key Features:**
- **Save Frequency:** Save at the end of each epoch or after a specified number of batches.
- **Best Model Only:** Optionally save only the best model according to a monitored metric (e.g., validation loss or accuracy).
- **Save Weights or Full Model:** Save only the model weights or the entire model (including architecture and optimizer state).
- **Customizable Naming:** Use formatted filenames that include epoch numbers and metric values.

**Common Usage:**
```python
checkpoint_callback = tf.keras.callbacks.ModelCheckpoint(
    filepath='model.{epoch:02d}-{val_loss:.2f}.keras',
    monitor='val_loss',
    save_best_only=True,
    mode='min'
)
model.fit(..., callbacks=[checkpoint_callback])
```

**Main Parameters:**
- `filepath`: Path to save the model (supports formatting like `{epoch}` and `{val_loss}`).
- `monitor`: Metric to track (e.g., `'val_accuracy'`).
- `save_best_only`: If `True`, only overwrites when performance improves.
- `mode`: `'min'` (for loss) or `'max'` (for accuracy) to determine improvement.
- `save_weights_only`: If `True`, saves only weights (as `.h5`); otherwise saves full model (as `.keras`).
- `save_freq`: `'epoch'` or an integer (number of batches).
- `initial_value_threshold`: Minimum/maximum metric value required for the first save.

This callback is essential for preventing data loss during long training sessions and for retaining optimal model versions.
