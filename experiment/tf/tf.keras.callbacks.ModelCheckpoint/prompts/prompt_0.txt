The `tf.keras.callbacks.ModelCheckpoint` callback saves a Keras model or its weights during training, allowing you to resume training or deploy the best model later.

**Key Features:**
- Saves the model or weights at specified intervals (per epoch or per batch).
- Can save only the best-performing model based on a monitored metric.
- Supports saving the full model or just the weights.

**Main Parameters:**
- `filepath`: Path to save the file, with optional formatting (e.g., `{epoch:02d}-{val_loss:.2f}.keras`).
- `monitor`: Metric to track (e.g., `'val_loss'`, `'val_accuracy'`).
- `save_best_only`: If `True`, saves only when the monitored metric improves.
- `mode`: `'min'` or `'max'` to determine improvement (e.g., minimize loss, maximize accuracy).
- `save_weights_only`: If `True`, saves only weights; otherwise saves the full model.
- `save_freq`: Save frequency (`'epoch'` or an integer for batches).
- `initial_value_threshold`: Minimum metric value to start saving when `save_best_only=True`.

**Usage Example:**
```python
checkpoint_callback = ModelCheckpoint(
    filepath='model.keras',
    monitor='val_accuracy',
    save_best_only=True,
    mode='max'
)
model.fit(..., callbacks=[checkpoint_callback])
```

This callback is essential for checkpointing during training, preventing data loss, and retaining optimal model versions.
