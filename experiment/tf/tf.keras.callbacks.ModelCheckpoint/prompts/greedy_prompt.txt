The `tf.keras.callbacks.ModelCheckpoint` callback saves a Keras model or its weights during training at specified intervals. It is primarily used with `model.fit()` to preserve training progress, allowing the model to be resumed or evaluated later.

**Key Features:**
- **Flexible Saving:** Can save the entire model or just the weights.
- **Conditional Saving:** Optionally saves only the "best" model based on a monitored metric (e.g., validation loss or accuracy).
- **Customizable Frequency:** Saves at the end of each epoch or after a fixed number of batches.
- **File Naming:** Supports dynamic naming with placeholders like `{epoch}` and metric values.

**Main Parameters:**
- `filepath`: Path to save the model (supports formatting like `{epoch:02d}-{val_loss:.2f}.keras`).
- `monitor`: Metric to track (e.g., `'val_loss'`, `'val_accuracy'`).
- `save_best_only`: If `True`, only saves when the monitored metric improves.
- `mode`: `'min'` or `'max'` to define improvement direction (e.g., minimize loss, maximize accuracy).
- `save_weights_only`: If `True`, saves only weights; otherwise saves the full model.
- `save_freq`: `'epoch'` or an integer (number of batches).
- `initial_value_threshold`: Sets a starting benchmark for `save_best_only`.

**Typical Use Case:**  
It is commonly employed to checkpoint models during long training runs, preventing data loss and enabling selection of the best-performing model based on validation metrics.
