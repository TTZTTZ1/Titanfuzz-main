The `tf.keras.regularizers.Regularizer` base class is used to apply penalties on layer parameters (kernels, biases) or layer outputs (activity) during model training, which are added to the loss function to help prevent overfitting.

**Key Features:**
- **Types of Regularizers:**  
  - `kernel_regularizer`: Penalizes layer weights.  
  - `bias_regularizer`: Penalizes layer biases.  
  - `activity_regularizer`: Penalizes layer outputs (scaled by batch size).  
- **Built-in Penalties:**  
  - `L1`, `L2`, and `L1L2` (combined) regularization.  
- **Usage:**  
  - Can be passed as arguments to Keras layers (e.g., `Dense`, `Conv2D`).  
  - Regularization losses are accessible via `layer.losses`.  
- **Custom Regularizers:**  
  - Can be created by extending `Regularizer` and implementing `__call__` and `get_config`.  
  - Any function taking a weight tensor and returning a scalar can also be used.  
- **Serialization:**  
  - Required for saving/loading models in HDF5, JSON, or cloning; implemented via `get_config` and `from_config`.

**Example:**
