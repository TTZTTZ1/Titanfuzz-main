The `tf.keras.callbacks.EarlyStopping` callback is used to automatically stop model training when a monitored metric (e.g., validation loss) stops improving, helping to prevent overfitting and save computational resources.

**Key Parameters:**
- **`monitor`**: Metric to monitor (default: `'val_loss'`).
- **`patience`**: Number of epochs with no improvement before stopping (default: `0`).
- **`min_delta`**: Minimum change required to qualify as an improvement (default: `0`).
- **`mode`**: Direction of improvement (`'auto'`, `'min'`, or `'max'`; default: `'auto'`).
- **`baseline`**: Threshold value; training stops if the metric doesnŌĆÖt surpass it.
- **`restore_best_weights`**: If `True`, restores model weights from the best epoch (default: `False`).
- **`start_from_epoch`**: Epoch to start monitoring improvement (default: `0`).

**How It Works:**
- Monitors the specified metric at the end of each epoch.
- If no improvement (beyond `min_delta`) occurs for `patience` consecutive epochs, training stops.
- Optionally restores the model to its best-performing state.

**Example:**
```python
early_stopping = tf.keras.callbacks.EarlyStopping(
    monitor='val_loss', patience=3, restore_best_weights=True
)
model.fit(..., callbacks=[early_stopping])
```
This stops training if validation loss doesnŌĆÖt improve for 3 epochs and reverts to the best model weights.
