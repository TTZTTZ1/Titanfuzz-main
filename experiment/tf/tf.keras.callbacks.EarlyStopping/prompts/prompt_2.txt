`tf.keras.callbacks.EarlyStopping` is a callback that halts training when a specified metric stops improving, helping to prevent overfitting and save computational resources.  

**Key parameters:**  
- `monitor`: Metric to watch (default: `'val_loss'`).  
- `min_delta`: Minimum change to qualify as improvement.  
- `patience`: Number of epochs with no improvement before stopping.  
- `mode`: Direction of improvement (`'min'`, `'max'`, or `'auto'`).  
- `restore_best_weights`: If `True`, restores model weights from the best epoch.  
- `start_from_epoch`: Epoch to start monitoring (allows warm-up).  

**How it works:**  
After each epoch, it checks whether the monitored metric has improved beyond `min_delta`. If no improvement occurs for `patience` consecutive epochs, training stops. Optionally, the model’s weights can revert to those from the best-performing epoch.

**Example usage:**  
```python
early_stopping = EarlyStopping(monitor='val_loss', patience=5, restore_best_weights=True)
model.fit(..., callbacks=[early_stopping])
```
