`tf.keras.callbacks.EarlyStopping` is a callback that halts training when a specified metric stops improving, helping to prevent overfitting and save computational resources.

**Key Parameters:**
- `monitor`: Metric to watch (default: `'val_loss'`).
- `patience`: Number of epochs with no improvement before stopping (default: 0).
- `min_delta`: Minimum change to qualify as improvement (default: 0).
- `mode`: Direction to optimize (`'auto'`, `'min'`, or `'max'`).
- `restore_best_weights`: If `True`, restores model weights from the best epoch (default: `False`).
- `baseline`: Target value; training stops if not surpassed.
- `start_from_epoch`: Epoch to start monitoring (default: 0).

**How It Works:**  
At the end of each epoch, it checks if the monitored metric has improved relative to `min_delta`. If no improvement occurs for `patience` consecutive epochs, training stops. Optionally, the model can revert to the best-performing weights.

**Example Usage:**
```python
early_stopping = EarlyStopping(monitor='val_loss', patience=3)
model.fit(..., callbacks=[early_stopping])
```
This stops training if validation loss doesn’t improve for 3 epochs.
