`tf.keras.callbacks.EarlyStopping` is a callback that stops training when a monitored metric stops improving, preventing overfitting and saving time.  

**Key parameters:**  
- `monitor`: Metric to watch (default: `"val_loss"`).  
- `patience`: Epochs to wait after no improvement before stopping.  
- `min_delta`: Minimum change to qualify as improvement.  
- `mode`: Direction to optimize (`"min"`, `"max"`, or `"auto"`).  
- `restore_best_weights`: If `True`, restores model weights from the best epoch.  
- `start_from_epoch`: Epoch to start monitoring (allows warm-up).  

**How it works:**  
- Tracks the monitored metric at the end of each epoch.  
- If no improvement is seen for `patience` epochs (within `min_delta`), training stops.  
- Optionally restores the best model weights when stopped.  

**Example usage:**  
```python
early_stopping = EarlyStopping(monitor='val_loss', patience=3)
model.fit(..., callbacks=[early_stopping])
```
