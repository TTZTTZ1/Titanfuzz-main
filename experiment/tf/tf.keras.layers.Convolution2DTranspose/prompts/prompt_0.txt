`tf.keras.layers.Conv2DTranspose` (also known as `Convolution2DTranspose`) is a **2D transposed convolutional layer** used for **upsampling** feature maps, commonly applied in tasks like image generation, autoencoders, and segmentation. It performs the reverse operation of a standard convolution, increasing spatial dimensions while learning to reconstruct or expand data.

### Key Parameters:
- **`filters`**: Number of output channels.
- **`kernel_size`**: Size of the convolution window.
- **`strides`**: Upsampling factor (e.g., `(2, 2)` doubles height/width).
- **`padding`**: `"valid"` (no padding) or `"same"` (output size matches input when `strides=1`).
- **`data_format`**: `"channels_last"` (default) or `"channels_first"`.
- **`activation`**: Optional activation function (e.g., `'relu'`).

### Input/Output Shapes:
- **Input**: 4D tensor `(batch, height, width, channels)` (channels-last) or `(batch, channels, height, width)` (channels-first).
- **Output**: Upsampled 4D tensor with new spatial dimensions based on `strides` and `padding`.

### Example:
```python
layer = Conv2DTranspose(filters=32, kernel_size=2, strides=2, activation='relu')
output = layer(input_tensor)  # Upsamples height/width by factor of 2.
```

### Common Use Cases:
- **Generative models** (VAEs, GANs)
- **Image-to-image translation** (e.g., pix2pix)
- **Upsampling in autoencoders or segmentation networks**.

### Notes:
- Incompatible with `strides > 1` and `dilation_rate > 1` simultaneously.
- Often paired with `Conv2D` layers to reverse their downsampling effects.
