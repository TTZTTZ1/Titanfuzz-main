`tf.train.Checkpoint` is a TensorFlow class for saving and restoring the state of trackable objects (like `tf.Variable`, `tf.keras.Model`, or `tf.keras.Optimizer`) to disk. It uses object-based checkpointing, which saves dependencies between Python objects, making it more robust to code changes compared to name-based approaches.

**Key Features:**
- **Object-Based:** Tracks dependencies between objects (e.g., layers, variables) rather than relying on variable names.
- **Automatic Dependency Management:** Objects like `tf.keras.Model` automatically track their variables, simplifying checkpoint creation.
- **Incremental Saving:** Includes a `save_counter` to number checkpoints when using `save()`.
- **Flexible Restoration:** Supports deferred restoration (variables restored when created) and can load TensorFlow 1.x name-based checkpoints or SavedModel directories.

**Basic Usage:**
```python

save_path = checkpoint.save('/tmp/ckpt')  # Saves with auto-incremented suffix
checkpoint.restore(save_path)             # Restores state
```

**Main Methods:**
- `save(file_prefix)`: Saves a checkpoint, increments `save_counter`, and returns the file path.
- `restore(save_path)`: Restores from a checkpoint (created by `save()`). Returns a status object for verification (e.g., `assert_consumed()`).
- `write(file_prefix)`: Low-level save without counter management (use with `read()`).
- `read(save_path)`: Restores from checkpoints created by `write()`.

**Use Cases:**
- Training model checkpoints.
- Loading pre-trained models (including from SavedModel).
- Distributed training with shared filesystems.

**Note:** For advanced checkpoint management (e.g., garbage collection), use `tf.train.CheckpointManager`.
