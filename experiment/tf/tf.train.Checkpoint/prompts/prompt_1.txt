`tf.train.Checkpoint` is a TensorFlow class for saving and restoring the state of trackable objects (like variables, models, and optimizers) to disk. It uses object-based checkpointing, which saves dependencies between Python objects, making it more robust to code changes compared to name-based approaches.

**Key Features:**
- **Object-based:** Tracks dependencies between objects (e.g., layers, variables) rather than relying on variable names.
- **Automatic dependency management:** Objects like `tf.keras.Model` automatically track their variables.
- **Incremental saving:** Includes a `save_counter` to number checkpoints when using `save()`.
- **Flexible restoration:** Can restore values immediately or defer until variables are created.
- **Multi-worker support:** Checkpoints from distributed training are merged without copying all data to one worker.

**Main Methods:**
- **`save(file_prefix)`:** Saves a checkpoint, incrementing the `save_counter`. Returns the file path.
- **`restore(save_path)`:** Restores from a checkpoint created by `save()`. Returns a status object for verification (e.g., `assert_consumed()`).
- **`write(file_prefix)`:** Low-level save without counter updates; use with `read()` for custom checkpoint management.
- **`read(save_path)`:** Restores from checkpoints created by `write()`.

**Common Usage:**
```python
checkpoint = tf.train.Checkpoint(model=model, optimizer=optimizer)
checkpoint.restore(tf.train.latest_checkpoint(checkpoint_dir))
checkpoint.save(file_prefix)
```

**Note:** Differs from `model.save_weights()` by numbering checkpoint files (e.g., `ckpt-1`, `ckpt-2`). It also supports loading from SavedModel directories and legacy TensorFlow 1.x checkpoints.
