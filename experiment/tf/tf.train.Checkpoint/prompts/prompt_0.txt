`tf.train.Checkpoint` is a TensorFlow class for saving and restoring the state of trackable objects (like variables, layers, models, and optimizers) to disk. It uses object-based checkpointing, which tracks dependencies between Python objects, making it more robust to code changes compared to name-based approaches.

**Key Features:**
- **Object-Based:** Saves a graph of dependencies, matching objects by structure rather than variable names.
- **Automatic Dependency Tracking:** Objects like `tf.keras.Model` automatically track their variables and layers.
- **Incremental Saving:** Includes a `save_counter` to number checkpoints when using `save()`.
- **Flexible Restoration:** Can restore values immediately or defer until dependent objects are created.
- **Multi-Worker Support:** Checkpoints from distributed training are merged without copying all data to one worker.

**Basic Usage:**
```python
checkpoint = tf.train.Checkpoint(model=model, optimizer=optimizer)
save_path = checkpoint.save('/tmp/ckpt')  # Saves as /tmp/ckpt-1, etc.
checkpoint.restore(save_path)             # Restores state
```

**Main Methods:**
- `save(file_prefix)`: Saves a checkpoint, increments `save_counter`, and updates latest checkpoint metadata.
- `restore(save_path)`: Restores from a checkpoint created by `save()`.
- `write(file_prefix)`: Writes a checkpoint without numbering or metadata updates (use with `read()`).
- `read(save_path)`: Reads a checkpoint created by `write()` (no `save_counter` expected).

**Notes:**
- Differs from `model.save_weights()` by numbering checkpoint files (e.g., `ckpt-1`, `ckpt-2`).
- Can load TensorFlow 1.x name-based checkpoints and SavedModel directories.
- Provides status methods (`assert_consumed()`, `expect_partial()`, etc.) to verify restoration.

Use `tf.train.Checkpoint` for robust state persistence in training loops, especially with custom models and distributed setups.
