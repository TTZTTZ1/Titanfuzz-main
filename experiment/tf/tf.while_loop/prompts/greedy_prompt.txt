`tf.while_loop` is a TensorFlow operation that repeatedly executes a loop body while a specified condition remains true. It is commonly used to implement dynamic loops in TensorFlow graphs, especially within `tf.function` where Python loops are automatically converted.

### Key Features:
- **Condition and Body**:  
  Takes two callables:  
  - `cond`: Returns a boolean scalar tensor indicating whether to continue looping.  
  - `body`: Returns updated loop variables (same structure and types as input `loop_vars`).

- **Loop Variables**:  
  A tuple, list, or namedtuple of tensors/TensorArrays passed to both `cond` and `body`.

- **Shape Invariants**:  
  Enforces shape consistency across iterations. If shapes vary, `shape_invariants` can be specified to allow more flexibility (e.g., dynamic dimensions).

- **Parallel Execution**:  
  Supports non-strict semantics, allowing multiple iterations to run in parallel (controlled by `parallel_iterations`).

- **Memory Management**:  
  The `swap_memory` option enables GPU-CPU memory swapping to handle large models or long sequences.

- **Gradient Control**:  
  The deprecated `back_prop=False` is replaced with `tf.stop_gradient` for disabling backpropagation.

- **Early Stopping**:  
  Optional `maximum_iterations` limits the number of iterations, overriding `cond` if reached.

### Common Use Cases:
- Dynamic iteration counts (e.g., while `i < n`).
- Recurrent models (RNNs, custom loops).
- Loops with tensor-dependent termination conditions.

### Example:
```python
i = tf.constant(0)
cond = lambda i: i < 10
body = lambda i: i + 1
result = tf.while_loop(cond, body, [i])[0]  # Returns 10
```

### Note:
In `tf.function`, Python `while` loops are automatically converted to `tf.while_loop`, making explicit use optional unless fine-grained control is needed.
