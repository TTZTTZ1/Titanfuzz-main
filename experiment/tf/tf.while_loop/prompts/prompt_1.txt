`tf.while_loop` is a TensorFlow operation that repeatedly executes a loop body while a specified condition remains true. It is commonly used to implement dynamic loops within TensorFlow graphs, especially when the number of iterations depends on runtime values.

### Key Features:
- **Automatic Conversion**: Inside `tf.function`, Python `while` loops are automatically converted to `tf.while_loop` unless `autograph=False` is set.
- **Strict Shape Invariants**: Enforces shape consistency across loop iterations unless custom `shape_invariants` are provided.
- **Parallel Execution**: Supports non-strict semantics, allowing multiple iterations to run in parallel (controlled by `parallel_iterations`).
- **Memory Management**: Includes `swap_memory` option to swap tensors between GPU and CPU, useful for handling large models or long sequences.
- **Gradient Control**: Backpropagation can be disabled via `tf.stop_gradient` instead of the deprecated `back_prop=False`.

### Basic Usage:
```python
i = tf.constant(0)
cond = lambda i: i < 10
body = lambda i: (i + 1,)
result = tf.while_loop(cond, body, [i])
```

### Important Parameters:
- `cond`: Callable returning a boolean scalar tensor (loop termination condition).
- `body`: Callable returning updated loop variables (same structure as `loop_vars`).
- `loop_vars`: Initial loop variables (tensors, TensorArrays, or nested structures).
- `maximum_iterations`: Optional cap on loop iterations.
- `shape_invariants`: Allows variable-shaped tensors across iterations.

### Common Use Cases:
- Dynamic RNNs
- Iterative algorithms with data-dependent stopping conditions
- Loops requiring gradient computation in custom training loops

The operation ensures correct tensor flow across iterations and integrates seamlessly with TensorFlow's autodiff system.
