`tf.keras.layers.RNN` is the base class for recurrent neural network layers in TensorFlow/Keras. It wraps an RNN cell (or a stack of cells) to process sequential data over multiple timesteps.

**Key Features:**
- **Cell-based design:** Accepts a single RNN cell or a list of cells (stacked RNN).
- **Flexible output:** Can return the final output only, the full output sequence, and/or the final hidden state(s).
- **Bidirectional support:** Can process sequences forward or backward (`go_backwards`).
- **Stateful mode:** Retains hidden states across batches for continuous sequences.
- **Masking support:** Handles variable-length sequences via masking.

**Main Parameters:**
- `cell`: The RNN cell instance(s) defining the recurrent computation.
- `return_sequences`: Whether to output the last timestep or all timesteps.
- `return_state`: Whether to return the final state(s) in addition to the output.
- `stateful`: If `True`, states persist between batches (requires fixed batch size).
- `unroll`: Unrolls the RNN loop for speed (suitable for short sequences).

**Input/Output:**
- **Input shape:** `(batch_size, timesteps, features)`
- **Output shape:** 
  - Default: `(batch_size, output_size)`
  - If `return_sequences`: `(batch_size, timesteps, output_size)`
  - If `return_state`: List of tensors [output, state1, state2, ...]

**Common Use Cases:**
- Time series forecasting
- Sequence classification/generation
- Stacked or custom RNN architectures

The layer supports masking, state management, and custom RNN cell implementations, providing a flexible foundation for recurrent models.
