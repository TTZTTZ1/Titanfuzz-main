The `tf.keras.optimizers.RMSprop` optimizer implements the RMSprop algorithm, which adapts the learning rate for each parameter by maintaining a moving average of squared gradients. It divides the gradient by the root of this average to normalize updates, making it effective for non-stationary objectives and recurrent neural networks.

**Key Features:**
- **Adaptive Learning Rates:** Adjusts per-parameter learning rates based on recent gradient magnitudes.
- **Momentum Support:** Optional plain momentum (not Nesterov) to accelerate convergence.
- **Centered Variant:** Optionally uses a moving average of gradients to estimate variance for more stable normalization.
- **Additional Optimizations:** Supports weight decay, gradient clipping, exponential moving averages (EMA), loss scaling, and gradient accumulation.

**Common Use Cases:**
- Training recurrent neural networks (RNNs).
- Problems with noisy or sparse gradients.
- Scenarios where adaptive learning rate methods like Adam may not generalize well.

**Example Usage:**
```python
opt = keras.optimizers.RMSprop(learning_rate=0.01, rho=0.9)
model.compile(optimizer=opt, loss='mse')
```

**Parameters of Note:**
- `learning_rate`: Base step size (default 0.001).
- `rho`: Discount factor for gradient history (default 0.9).
- `momentum`: Acceleration factor (default 0.0).
- `centered`: Whether to normalize by gradient variance estimate (default False).
- `epsilon`: Small constant for numerical stability (default 1e-7).

RMSprop is particularly useful when dealing with unstable gradients or when training deep networks where learning rates need per-parameter adjustments.
