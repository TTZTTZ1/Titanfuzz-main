`tf.nn.nce_loss` computes the noise-contrastive estimation (NCE) training loss, which is an efficient method for training models with a large number of output classes (e.g., in language modeling or recommendation systems). It approximates the full softmax loss by sampling a small number of negative classes per batch, reducing computational cost.

**Key Parameters:**
- `weights` and `biases`: Embeddings and biases for all classes.
- `labels`: Target classes for each example.
- `inputs`: Forward activations from the network.
- `num_sampled`: Number of negative classes to sample per batch.
- `num_classes`: Total number of possible classes.

**Usage Notes:**
- Typically used during training, while the full sigmoid cross-entropy loss is used for evaluation.
- Labels should be sorted by decreasing frequency for optimal sampling (log-uniform/Zipfian distribution is used by default).
- Supports multiple target classes per example (`num_true > 1`), with target probabilities normalized to sum to 1.
- Set `remove_accidental_hits=True` to ignore cases where a sampled negative class matches a target class (converting the loss to "Sampled Logistic").

**Output:** Returns a 1-D tensor of per-example NCE losses with shape `[batch_size]`.
---GENERAL SUMMARY END---
