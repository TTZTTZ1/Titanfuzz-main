`tf.nn.nce_loss` computes the noise-contrastive estimation training loss, which is an efficient method for learning embeddings when the number of classes is large. It approximates the full softmax by sampling a subset of negative classes per batch, making it suitable for training tasks like word embedding models.

**Key parameters:**
- `weights` and `biases`: Embedding matrix and biases for all classes.
- `labels`: Target classes for each batch example.
- `inputs`: Forward activations from the network.
- `num_sampled`: Number of negative samples per batch.
- `num_classes`: Total number of possible classes.

**Usage notes:**
- Typically used during training, while full sigmoid cross-entropy may be used for evaluation.
- By default, it uses a log-uniform (Zipfian) sampling distribution, so labels should be sorted by decreasing frequency for optimal results.
- Supports removal of accidental hits (where a sampled class matches a target) via `remove_accidental_hits`.

This loss is designed to scale efficiently with large vocabularies by avoiding computation over all classes.
