# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
`tf.scan` sequentially applies a function `fn` across elements of `elems` along their first dimension, accumulating results. It is similar to a fold/reduce operation but returns all intermediate accumulator values rather than just the final one.

**Key behavior:**
- **Inputs:** `fn` takes two arguments: the accumulated value (initially `initializer`) and the current element from `elems`.
- **Output:** Returns a sequence of accumulated values, one for each step.
- **Without `initializer`:** The first element of `elems` is used as the starting accumulator.
- **With `initializer`:** Specifies the starting accumulator and determines the output structure/dtype.
- **Multi-arity support:** Both `elems` and `initializer` can be nested structures (lists/tuples of tensors), requiring matching structures in `fn`.
- **Reverse option:** If `reverse=True`, scanning proceeds from last to first element.

**Example:**
```python
import tensorflow as tf
elems = tf.constant([1, 2, 3, 4, 5])
scan_result = tf.scan(lambda a, x: a + x, elems)  # Result: [1, 3, 6, 10, 15]
```

**Common use cases:** Running sums, sequences like Fibonacci, or any cumulative operation across a sequence.
