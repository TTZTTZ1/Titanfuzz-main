The output tensors are ordered by partition index, and their first dimension corresponds to the number of elements assigned to that partition.

- **Inputs:** `data` (any tensor), `partitions` (int32 tensor with the same leading shape as `data`), `num_partitions` (integer ≥ 1).
- **Output:** A list of `num_partitions` tensors, each containing the slices of `data` assigned to that partition index.
- **Requirements:** `partitions` values must be in `[0, num_partitions)`, and the shape of `partitions` must match the leading dimensions of `data`.
- **Use case:** Useful for dynamically grouping data based on conditions, often paired with `tf.dynamic_stitch` to reconstruct the original tensor.

**Example:**
