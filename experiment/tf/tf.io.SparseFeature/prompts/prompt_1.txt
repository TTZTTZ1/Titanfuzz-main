`tf.io.SparseFeature` is a configuration class used to parse sparse input features from TensorFlow `Example` protos into `SparseTensor` objects. It is designed for cases where sparse data is stored with separate features for indices and values.

### Key Parameters:
- **`index_key`**: Name(s) of the feature(s) containing indices (must be `int64`). Can be a single string for 1D sparse data or a list for higher dimensions.
- **`value_key`**: Name of the feature containing the values (data type matches `dtype`).
- **`dtype`**: Data type of the values (e.g., `tf.float32`).
- **`size`**: Dense shape of the resulting `SparseTensor` (a single int for 1D, or a list of ints matching the length of `index_key` for higher dimensions).
- **`already_sorted`** (optional): If `True`, assumes indices are pre-sorted to skip sorting (default is `False`).

### Usage Example:
For a 2D `SparseTensor` with indices `[[3, 1], [20, 0]]` and values `[0.5, -1.0]` (dense shape `[100, 3]`), the corresponding `Example` and `SparseFeature` would be:

```python

    index_key=["ix0", "ix1"],  # Features for each dimension
    value_key="val",           # Feature for values
    dtype=tf.float32,
    size=[100, 3]
)
```

### Note:
Prefer `tf.io.VarLenFeature` for simpler use cases, especially with `SequenceExample`. `SparseFeature` is more explicit and suited for multi-dimensional sparse data where indices are stored separately.
