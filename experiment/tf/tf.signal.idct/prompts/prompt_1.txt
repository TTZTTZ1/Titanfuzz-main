The `tf.signal.idct` function computes the 1D Inverse Discrete Cosine Transform (IDCT) of an input tensor. It supports Types I, II, III, and IV transforms, where Type III is the inverse of Type II (and vice versa).

**Key parameters:**
- `input`: A float tensor with shape `[..., samples]`.
- `type`: The IDCT type (1, 2, 3, or 4).
- `norm`: Normalization mode—`None` for no normalization or `'ortho'` for orthonormal normalization. Without `'ortho'`, rescaling by `1/(2n)` is needed for exact inversion.
- `n`, `axis`: Currently restricted to `None` and `-1`, respectively, for future compatibility.

**Important note:**  
When `norm='ortho'`, the transform is perfectly invertible:  
`signal == idct(dct(signal, norm='ortho'), norm='ortho')`.  
Otherwise, manual rescaling is required:  
`signal == idct(dct(signal)) * 0.5 / signal.shape[-1]`.

The function matches SciPy’s `scipy.fftpack.idct` behavior for all supported types.
