The `tf.keras.metrics.CategoricalCrossentropy` metric computes the cross-entropy loss between true labels and predictions for multi-class classification tasks. It assumes labels are one-hot encoded and predictions represent probability distributions across classes.

**Key Parameters:**
- `from_logits`: Set to `True` if predictions are raw logits (unscaled), `False` if they are probabilities (default).
- `label_smoothing`: Smooths labels by redistributing a small amount of probability mass (e.g., 0.1 for smoothing=0.2).
- `axis`: The axis along which to compute entropy (defaults to -1, the last dimension).

**Usage Example:**
```python
metric = tf.keras.metrics.CategoricalCrossentropy()
metric.update_state([[0, 1, 0], [0, 0, 1]], 
                    [[0.05, 0.95, 0], [0.1, 0.8, 0.1]])
print(metric.result())  # Output: ~1.177
```

**Integration:**
Can be passed directly to `model.compile()` as a metric to monitor during training:
```python
model.compile(optimizer='sgd',
              loss='categorical_crossentropy',
              metrics=[tf.keras.metrics.CategoricalCrossentropy()])
```

**Inheritance:**  
Extends `MeanMetricWrapper`, so it computes the mean cross-entropy across samples, optionally weighted by `sample_weight`.
