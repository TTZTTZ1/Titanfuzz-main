**`tf.keras.losses.SparseCategoricalCrossentropy`** is a loss function for multi-class classification tasks where labels are provided as integer indices (e.g., `[0, 2, 1]`) rather than one-hot encoded vectors. It computes the cross-entropy between true labels and predicted class probabilities.

---

### **Key Features:**
- **Input format:**  
  - `y_true`: Integer labels with shape `[batch_size]`.  
  - `y_pred`: Predicted probabilities/logits with shape `[batch_size, num_classes]`.
- **Use case:** Suitable for classification with ≥2 classes when labels are integers.
- **Parameters:**
  - `from_logits`: Set `True` if `y_pred` are raw logits (not probabilities).
  - `reduction`: Loss aggregation method (`"sum_over_batch_size"` by default).
  - `ignore_class`: Optional integer class index to ignore in loss calculation.

---

### **Example:**
```python
import tensorflow as tf

loss_fn = tf.keras.losses.SparseCategoricalCrossentropy()
y_true = [1, 2]  # Integer labels
y_pred = [[0.05, 0.95, 0], [0.1, 0.8, 0.1]]  # Probabilities for 3 classes
loss = loss_fn(y_true, y_pred)  # Output: ~1.177
```

---

### **Usage with `compile()`:**
```python
model.compile(
    optimizer='adam',
    loss=tf.keras.losses.SparseCategoricalCrossentropy()
)
```

---

### **Notes:**
- For one-hot encoded labels, use `CategoricalCrossentropy` instead.
- Supports sample weighting and different reduction strategies (`"sum"`, `"none"`, etc.).
