`tf.keras.models.clone_model` creates a copy of a Keras Functional or Sequential model, producing new layers and weights rather than sharing existing ones. It is useful for replicating a model's architecture with fresh weight initializations or for customizing the cloned structure.

**Key Parameters:**
- `model`: The original Functional or Sequential model instance.
- `input_tensors`: Optional new input tensors; if not provided, new inputs are created.
- `clone_function`: Custom function to clone each layer (default uses layer configs).
- `call_function`: Custom function to modify layer calls (Functional models only).
- `recursive`: Whether to recursively clone inner models.

**Key Notes:**
- Does not preserve shared objects (e.g., variables used in multiple layers become separate).
- Subclassed models are not supported unless they implement `get_config()`/`from_config()`.
- Useful for weight resetting, layer modifications, or adding/removing components without altering the original model.

**Example:**
```python
new_model = clone_model(model)  # Fresh copy with new weights
```
