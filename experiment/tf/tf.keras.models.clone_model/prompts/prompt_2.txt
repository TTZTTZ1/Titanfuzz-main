`tf.keras.models.clone_model` creates a copy of a Keras Sequential or Functional model, producing new layers and weights rather than reusing existing ones. It does not preserve shared objects (e.g., a variable used in multiple layers becomes separate copies).

**Key parameters:**
- `model`: The original Sequential or Functional model.
- `input_tensors`: Optional new inputs; defaults to new `Input` layers.
- `clone_function`: Customizes layer cloning (default uses `from_config`).
- `call_function`: Customizes layer calls (Functional models only).
- `recursive`: If `True`, recursively clones inner models.

**Use cases:**
- Replicate a model with fresh weights.
- Modify architecture via `clone_function` (e.g., replace layer types).
- Insert layers during calls via `call_function`.

**Limitations:**
- Subclassed models require `get_config`/`from_config` instead.
- Shared objects lose uniqueness in the clone.

**Example:**
```python
new_model = clone_model(model)  # Basic clone with new weights
```
