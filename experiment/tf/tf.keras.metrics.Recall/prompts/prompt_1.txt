`tf.keras.metrics.Recall` is a metric that calculates the recall (true positive rate) for binary or multi-class classification tasks. It measures the proportion of actual positive cases that are correctly identified.

**Key Features:**
- Computes recall as: `true_positives / (true_positives + false_negatives)`
- Supports optional parameters:
  - `thresholds`: Custom decision thresholds (default 0.5)
  - `top_k`: Consider only top-k predictions
  - `class_id`: Calculate recall for a specific class only
- Handles sample weighting and masking
- Compatible with logits-based models (use `thresholds=0`)

**Basic Usage:**
```python
metric = tf.keras.metrics.Recall()
metric.update_state([0, 1, 1, 1], [1, 0, 1, 1])  # y_true, y_pred
print(metric.result())  # Output: 0.6667
```

**Integration:**
```python
model.compile(
    optimizer='adam',
    loss='binary_crossentropy',
    metrics=[tf.keras.metrics.Recall()]
)
```

The metric maintains internal state variables (`true_positives`, `false_negatives`) that accumulate across batches and can be reset using `reset_state()`.
