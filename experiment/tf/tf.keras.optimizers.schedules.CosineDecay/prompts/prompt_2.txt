`tf.keras.optimizers.schedules.CosineDecay` implements a learning rate schedule that combines an optional linear warmup phase with a cosine decay phase.

**Key Features:**
- **Cosine Decay:** After warmup (or immediately if no warmup), the learning rate follows a cosine curve from the starting value down to a minimum (`alpha * initial_learning_rate`).
- **Optional Warmup:** Can linearly increase the learning rate from `initial_learning_rate` to a `warmup_target` over `warmup_steps` before starting decay.
- **Step-based:** Computes the learning rate based on the current optimizer step.

**Main Parameters:**
- `initial_learning_rate`: Starting LR (float).
- `decay_steps`: Total steps over which decay occurs (int).
- `alpha`: Minimum LR as a fraction of the initial LR (float, default 0.0).
- `warmup_target`: Target LR at end of warmup (float or None to skip warmup).
- `warmup_steps`: Number of steps for warmup (int, default 0).

**Returns:** A callable schedule that takes the current `step` and returns the corresponding learning rate.

**Typical Use:** Pass the schedule directly to a Keras optimizer's `learning_rate` argument. It is serializable via `serialize`/`deserialize`.
