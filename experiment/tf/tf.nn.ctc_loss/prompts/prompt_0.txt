`tf.nn.ctc_loss` computes the Connectionist Temporal Classification (CTC) loss, which is used for training sequence models (like RNNs/LSTMs) on tasks with variable-length input and output sequences, such as speech or handwriting recognition.

**Key features:**
- Automatically applies softmax to `logits`.
- Supports both dense padded tensors and sparse tensors for `labels` (sparse is faster on CPU/GPU, but only dense is supported on TPU).
- Default blank label index is `0`, but can be customized via `blank_index`.
- Input `logits` are time-major by default (`[frames, batch_size, num_labels]`), but can be batch-major.

**Arguments:**
- `labels`: Ground truth sequences, either dense (padded) or sparse.
- `logits`: Model predictions (pre-softmax).
- `label_length`: Lengths of each label sequence.
- `logit_length`: Lengths of each input sequence.
- `logits_time_major`: Specifies the layout of `logits`.
- `blank_index`: Optional index for the blank class (default 0).
- `unique`: Optional precomputed unique labels for TPU optimization.

**Returns:**
- A 1-D tensor of negative log probabilities (loss per batch element).

**Note:** If using sparse `labels`, `blank_index` must be explicitly provided.
