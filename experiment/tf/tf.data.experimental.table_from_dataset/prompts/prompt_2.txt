`tf.data.experimental.table_from_dataset` creates a lookup table from a dataset of key-value pairs. It maps known keys to their corresponding values, while handling out-of-vocabulary (OOV) keys by assigning them either to hashed buckets (if `num_oov_buckets > 0`) or to a specified `default_value`. The function is useful for building vocabulary-based mappings directly from TensorFlow datasets.

**Key parameters:**
- `dataset`: Dataset of (key, value) pairs.
- `num_oov_buckets`: Number of buckets for OOV keys (optional).
- `default_value`: Fallback value for OOV keys (defaults to -1).
- `key_dtype`: Data type of keys (string or integer).

**Example:**
```python
keys = tf.data.Dataset.range(3)
values = keys.map(lambda x: tf.strings.as_string(x * 2))
ds = tf.data.Dataset.zip((keys, values))
table = tf.data.experimental.table_from_dataset(ds, key_dtype=tf.int64)
table.lookup(tf.constant([0, 1, 2], dtype=tf.int64))  # Returns [b'0', b'2', b'4']
```
