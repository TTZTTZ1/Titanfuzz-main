`tf.keras.layers.LSTMCell` is a low-level RNN cell that implements a single time step of a Long Short-Term Memory (LSTM) layer. It is designed to be used within `tf.keras.layers.RNN` to process sequential data step-by-step, unlike `tf.keras.layers.LSTM` which processes entire sequences at once.

**Key Features:**
- **Purpose:** Handles one time step of input, making it suitable for custom RNN loops or when integrating with `tf.keras.layers.RNN`.
- **Core Parameters:**  
  - `units`: Dimensionality of the output space.  
  - `activation`: Activation for the cell state (default: `tanh`).  
  - `recurrent_activation`: Activation for recurrent gates (default: `sigmoid`).  
  - `dropout` / `recurrent_dropout`: Dropout rates for inputs and recurrent connections.  
  - `unit_forget_bias`: If `True`, initializes forget gate bias to 1 (recommended).

**Usage Example:**
```python
cell = tf.keras.layers.LSTMCell(units=4)
rnn = tf.keras.layers.RNN(cell, return_sequences=True)
output = rnn(inputs)  # Processes the full sequence
```

**Main Methods:**
- `call()`: Processes a single time step given inputs and previous states.
- `get_initial_state()`: Returns the initial state tensor.
- Dropout mask utilities (`get_dropout_mask`, `reset_dropout_mask`, etc.) for managing dropout during training.

**Use Case:** Ideal for custom RNN implementations, dynamic sequence handling, or when fine-grained control over LSTM behavior is needed. For standard sequence modeling, the higher-level `LSTM` layer is often more convenient.
