The `tf.keras.layers.MultiHeadAttention` layer implements multi-headed attention as described in the "Attention is All You Need" paper. It projects query, key, and value tensors into multiple heads, computes scaled dot-product attention, and combines the results.

**Key Parameters:**
- `num_heads`: Number of parallel attention heads.
- `key_dim`: Dimensionality of each attention head for query and key.
- `value_dim`: (Optional) Dimensionality for value heads; defaults to `key_dim`.
- `dropout`: Dropout rate applied to attention scores.
- `use_bias`: Whether to use bias in dense projections.
- `attention_axes`: Axes over which attention is applied (defaults to all except batch, heads, and features).

**Call Arguments:**
- `query`, `value`, `key` (optional): Input tensors. If `key` is not provided, `value` is used for both.
- `attention_mask`: Boolean mask to prevent attention to certain positions.
- `return_attention_scores`: If `True`, returns both output and attention scores.
- `use_causal_mask`: Applies a causal mask to prevent attending to future tokens (useful in decoder).

**Returns:**
- `attention_output`: Tensor of shape matching query except the last dimension, which is projected based on `output_shape`.
- `attention_scores`: (Optional) Attention coefficients if `return_attention_scores=True`.

**Common Use Cases:**
- Self-attention (when query, key, value are the same).
- Cross-attention (e.g., in encoder-decoder architectures).
- Transformer models for tasks like machine translation, image captioning, and more.

This layer is highly configurable with options for initialization, regularization, and constraints, making it suitable for various attention-based neural network designs.
