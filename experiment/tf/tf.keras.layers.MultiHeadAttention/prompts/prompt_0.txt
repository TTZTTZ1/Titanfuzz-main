The `tf.keras.layers.MultiHeadAttention` layer implements multi-headed attention as described in the "Attention Is All You Need" paper. It projects query, key, and value tensors into multiple heads, computes scaled dot-product attention, and concatenates the results.

**Key Parameters:**
- `num_heads`: Number of attention heads.
- `key_dim`: Feature dimension for each head in query and key.
- `value_dim`: Feature dimension for each head in value (defaults to `key_dim`).
- `dropout`: Dropout rate applied to attention scores.
- `use_bias`: Whether to use bias in dense projections.
- `attention_axes`: Axes over which attention is applied (defaults to all except batch and heads).

**Call Arguments:**
- `query`, `value`, `key` (optional): Input tensors. If `key` is not provided, `value` is used for both.
- `attention_mask`: Boolean mask to prevent attention to certain positions.
- `return_attention_scores`: If `True`, returns both output and attention scores.
- `use_causal_mask`: Applies a causal mask to prevent attending to future tokens (e.g., for decoder layers).

**Returns:**
- `attention_output`: Tensor of shape `(batch_size, target_seq_len, feature_dim)`.
- `attention_scores` (optional): Attention coefficients if `return_attention_scores=True`.

**Common Use Cases:**
- Self-attention (when query, key, value are the same).
- Cross-attention (e.g., in encoder-decoder architectures).
- Transformer models for tasks like machine translation, image captioning, and more.

The layer supports config serialization via `get_config` and `from_config`, and includes regularization, constraint, and initialization options for its dense kernels.
