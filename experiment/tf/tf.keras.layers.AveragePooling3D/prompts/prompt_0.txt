The `tf.keras.layers.AveragePooling3D` layer performs **average pooling** on 3D data (such as volumetric or spatio-temporal data). It downsamples input along its spatial dimensions (depth, height, width) by computing the **average value** within a sliding window of size `pool_size`.

**Key Parameters:**
- `pool_size`: Size of the pooling window (int or tuple of 3 integers).
- `strides`: Step size for sliding the window (defaults to `pool_size`).
- `padding`: `"valid"` (no padding) or `"same"` (padding to preserve spatial dimensions).
- `data_format`: `"channels_last"` (default) or `"channels_first"` for dimension ordering.

**Input Shape:**  
5D tensor:  
- `(batch, depth, height, width, channels)` for `channels_last`  
- `(batch, channels, depth, height, width)` for `channels_first`

**Output Shape:**  
Same 5D structure, with spatial dimensions reduced based on `pool_size`, `strides`, and `padding`.

**Example:**
```python
inputs = keras.Input(shape=(30, 30, 30, 3))
outputs = keras.layers.AveragePooling3D(pool_size=3)(inputs)  # Output shape: (batch, 10, 10, 10, 3)
```

This layer is commonly used in **3D convolutional neural networks** to reduce spatial dimensions while retaining channel information, helping to decrease computational cost and control overfitting.
