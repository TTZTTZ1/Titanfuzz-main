`tf.keras.metrics.Hinge` computes the hinge loss metric for binary classification tasks, where true labels (`y_true`) are expected to be -1 or 1 (binary 0/1 labels are automatically converted). It inherits from `MeanMetricWrapper`, meaning it averages the hinge loss across samples.

**Key Features:**
- **Formula**: Hinge loss = `max(1 - y_true * y_pred, 0)`, averaged over all samples.
- **Inputs**:  
  - `y_true`: Ground truth labels (-1 or 1).  
  - `y_pred`: Predicted values (logits or probabilities).
- **Usage**:  
  - Initialize the metric: `m = keras.metrics.Hinge()`  
  - Update state: `m.update_state(y_true, y_pred)`  
  - Get result: `m.result()` (returns average hinge loss).
- **Supports sample weighting** and maintains state across batches (state is reset via `reset_state()`).

**Example:**
```python
m = keras.metrics.Hinge()
m.update_state([[0, 1], [0, 0]], [[0.6, 0.4], [0.4, 0.6]])
print(m.result())  # Output: 1.3
```
