`tf.debugging.experimental.enable_dump_debug_info` enables the dumping of debugging information from a TensorFlow program to a specified directory (`dump_root`). This information includes details about function construction, op execution, stack traces, and tensor values, which can be used by debugger UIs.

### Key Features:
- **Dump Directory**: All debugging data is written to the provided `dump_root` path.
- **Tensor Debug Modes**: Controls the level of tensor value information captured (e.g., shape, health stats like NaN/inf counts, or full tensor values). Default is `"NO_TENSOR"`.
- **Filters**: Optionally restrict dumping using `op_regex` (for op types) and `tensor_dtypes` (for specific data types).
- **Circular Buffer**: Uses a buffer (default size 1000) to reduce overhead by storing recent execution events before writing to disk.
- **Idempotent**: Calling multiple times with the same `dump_root` has no effect; changing `dump_root` replaces the previous setting.

### Usage Example:
```python

```

### Important Notes:
- Disable with `tf.debugging.experimental.disable_dump_debug_info()`.
- On TPUs, enable soft device placement first: `tf.config.set_soft_device_placement(True)`.
- Returns a `DebugEventsWriter` instance for manual flushing of debug files.
