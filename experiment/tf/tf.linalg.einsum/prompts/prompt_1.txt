`tf.linalg.einsum` (aliased as `tf.einsum`) performs tensor contractions and outer products using Einstein summation notation. It allows flexible element-wise computations by specifying an equation string that defines how input indices map to output indices.

**Key features:**
- **Equation format:** Follows NumPy's `einsum` syntax (e.g., `'ij,jk->ik'` for matrix multiplication).
- **Automatic summation:** Repeated indices not in the output are summed over.
- **Common operations:** Supports matrix multiplication, dot products, outer products, transpose, diagonal extraction, trace, and batch operations.
- **Ellipsis support:** Uses `...` for broadcasting over arbitrary dimensions (e.g., `'...ij,...jk->...ik'`).
- **No broadcasting on named axes:** Matching labels must have the same length; use `tf.squeeze` or `tf.reshape` for length‑1 axes.
- **Optimization:** Optional contraction path optimization via `optimize` argument.

**Example:**
```python
# Matrix multiplication
result = tf.einsum('ij,jk->ik', A, B)  # C[i,k] = sum_j A[i,j] * B[j,k]
```

**Args:**
- `equation`: String describing the contraction.
- `*inputs`: Tensors consistent with the equation.
- `optimize`: Optimization strategy ('greedy', 'optimal', etc.).
- `name`: Operation name.

**Returns:** Contracted tensor with shape determined by the equation.
