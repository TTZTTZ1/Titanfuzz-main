`tf.linalg.einsum` (also available as `tf.einsum`) performs tensor contractions and outer products using Einstein summation notation. It allows you to define complex tensor operations concisely via an equation string.

**Key Features:**
- **Equation Format:** Uses a string like `'ij,jk->ik'` to specify input indices and output indices, following the same rules as NumPy's `einsum`.
- **Automatic Summation:** Repeated indices that don’t appear in the output are summed over.
- **Common Operations:** Supports matrix multiplication, dot products, outer products, transpose, diagonal extraction, trace, batch operations, and more.
- **Ellipsis Support:** Allows `'...'` to represent arbitrary dimensions, enabling broadcasting over those axes.
- **Optimization:** Optional contraction path optimization via `optimize` keyword.

**Example Usage:**
```python
# Matrix multiplication
C = tf.einsum('ij,jk->ik', A, B)

# Dot product
dot = tf.einsum('i,i->', u, v)

# Batch matrix multiplication with broadcasting
result = tf.einsum('...ij,...jk->...ik', batch1, batch2)
```

**Limitations:**
- Does not support broadcasting on named axes; matching labels must have the same length.
- Length‑1 dimensions should be removed explicitly (e.g., with `tf.squeeze`).

**Arguments:**
- `equation`: Einstein summation string.
- `*inputs`: Tensors to contract.
- `optimize`: Contraction path strategy (`'greedy'`, `'optimal'`, etc.).
- `name`: Optional operation name.

**Returns:** Contracted tensor with shape determined by the equation.
