`tf.linalg.einsum` (also available as `tf.einsum`) performs tensor contractions and outer products using Einstein summation notation. It allows you to define element-wise computations concisely via an equation string.

**Key Features:**
- **Equation Format:** Follows NumPy's `einsum` syntax (e.g., `'ij,jk->ik'` for matrix multiplication).
- **Automatic Summation:** Repeated indices not in the output are summed over.
- **Common Operations:** Supports matrix multiplication, dot products, outer products, transpose, diagonal extraction, trace, and batch operations.
- **Ellipsis Support:** Uses `...` for broadcasting over unspecified dimensions (e.g., `'...ij,...jk->...ik'`).
- **No Broadcasting on Named Axes:** Matching labels must have the same length; use `tf.squeeze` or `tf.reshape` for length‑1 axes.
- **Optimization:** Optional contraction path optimization (`optimize` keyword).

**Example:**
```python
# Matrix multiplication
result = tf.einsum('ij,jk->ik', A, B)  # C[i,k] = sum_j A[i,j] * B[j,k]
```

**Arguments:**
- `equation` (str): Einstein summation string.
- `*inputs` (Tensor): Tensors to contract.
- `optimize` (str): Contraction path strategy ('greedy', 'optimal', etc., default 'greedy').
- `name` (str): Optional operation name.

**Returns:** Contracted tensor with shape determined by `equation`.

**Raises:** `ValueError` for incorrect equation format or shape mismatches.
