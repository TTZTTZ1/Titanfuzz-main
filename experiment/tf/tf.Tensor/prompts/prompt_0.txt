The `tf.Tensor` class in TensorFlow represents a multidimensional array with elements of a single data type (e.g., float32, int32). It is the core object manipulated in TensorFlow programs and has two key properties: **data type** (`dtype`) and **shape**.

### Key Features:
- **Eager vs. Graph Execution**: In eager mode (default), operations execute immediately; in graph mode (via `tf.function`), a computational graph is built for later evaluation.
- **Shape Handling**: Shapes may be fully known or partially unknown (e.g., `None` dimensions in `tf.function`). Use `tf.shape(tensor)` for runtime shape or `tensor.shape`/`get_shape()` for static shape.
- **Specialized Tensors**: Includes `tf.Variable`, `tf.constant`, `tf.sparse.SparseTensor`, and `tf.RaggedTensor`.

### Important Methods:
- **`eval(feed_dict=None, session=None)`**: Evaluates the tensor in a session (legacy, mainly for graph execution).
- **`get_shape()`**: Returns a `tf.TensorShape` representing the tensor's shape.
- **`set_shape(shape)`**: Updates the shape (use `tf.ensure_shape` for better error checking).
- **`ref()`**: Returns a hashable reference for use in sets/dictionaries (since tensors are unhashable).
- **`numpy()`**: Converts an eager tensor to a NumPy array.

### Operator Overloading:
Supports common Python operators (e.g., `+`, `-`, `*`, `@` for matrix multiplication) and comparisons, enabling intuitive syntax similar to NumPy.

### Caveats:
- **Buffer Reuse**: Constructing a tensor from a NumPy array may share the underlying buffer (changes to the array affect the tensor).
- **Unhashable**: Tensors cannot be directly used as dictionary keys or in sets; use `tensor.ref()` instead.
- **No Boolean Conversion**: Tensors cannot be used as booleans (e.g., in `if` statements); this raises a `TypeError`.

### Example:
```python
c = tf.constant([[1.0, 2.0], [3.0, 4.0]])
d = tf.constant([[1.0, 1.0], [0.0, 1.0]])
e = c @ d  # Matrix multiplication
print(e.numpy())  # Convert to NumPy array
```

In summary, `tf.Tensor` is the fundamental data structure in TensorFlow, providing a flexible interface for numerical computations with support for both eager and graph execution modes.
