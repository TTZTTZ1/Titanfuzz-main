**Key parameters:**
- `allow_partial_checkpoint`: Allows checkpoint mismatches with the loaded object (default: `False`).
- `experimental_io_device`: Specifies the device (e.g., CPU:0) for filesystem access in distributed settings.
- `experimental_skip_checkpoint`: Skips loading checkpoints, usually resulting in an unusable model (default: `False`).
- `experimental_variable_policy`: Sets the variable loading policy (e.g., for resource variables).
- `experimental_load_function_aliases`: Adds a `function_aliases` attribute to the loaded model (default: `False`).

These options provide control over checkpoint handling, distributed loading, variable policies, and function aliases when restoring SavedModels.
