**`tf.keras.metrics.TruePositives`** is a metric that calculates the number of true positive predictions in binary or multi-label classification tasks.

### **Key Features:**
- **Purpose:** Counts predictions where both the true label and predicted label are positive.
- **Threshold Handling:**  
  - Default threshold is `0.5`.  
  - Supports custom thresholds (single float or list) to binarize predictions.  
  - Set `thresholds=0` if using logits (`from_logits=True`).
- **Weighted Calculation:**  
  - Supports `sample_weight` to weigh individual predictions.  
  - Without weights, defaults to counting each true positive as 1.
- **Stateful:** Internally tracks the cumulative count across batches via a `true_positives` variable.

### **Basic Usage:**
```python

metric.update_state([0, 1, 1, 1], [1, 0, 1, 1])  # y_true, y_pred

```

### **Main Methods:**
- **`update_state(y_true, y_pred, sample_weight=None)`**: Accumulates counts.
- **`result()`**: Returns the current true positives count.
- **`reset_state()`**: Clears the accumulated state.

### **Use Case:**
Useful for monitoring model performance, especially in imbalanced classification scenarios (see [Classification on imbalanced data](https://tensorflow.google.cn/tutorials/structured_data/imbalanced_data)).
