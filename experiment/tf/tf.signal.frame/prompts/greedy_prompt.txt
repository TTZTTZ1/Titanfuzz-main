`tf.signal.frame` is a TensorFlow function that segments a signal into overlapping or non-overlapping frames by sliding a fixed-length window along a specified axis. It transforms the input tensor by expanding the chosen axis into two dimensions: the number of frames and the frame length.

**Key Parameters:**
- `signal`: Input tensor with at least one dimension.
- `frame_length`: Length of each frame (window size).
- `frame_step`: Step size between consecutive frames (stride).
- `pad_end`: If `True`, pads the signal end with `pad_value` to include partial frames; otherwise, discards incomplete frames.
- `axis`: Axis along which to apply framing (defaults to the last axis).

**Output:** A tensor where the specified axis is replaced by `[num_frames, frame_length]`, adding one dimension.

**Example:**  
For an audio tensor of shape `[3, 9152]`, framing with `frame_length=512` and `frame_step=180` (without padding) yields `[3, 49, 512]`, where 49 frames are generated. With `pad_end=True`, it produces `[3, 51, 512]`, including zero-padded frames at the end.

**Use Case:** Commonly used in audio and signal processing to prepare data for spectral analysis or machine learning models.
