---GENERAL SUMMARY START---
`tf.signal.frame` converts a signal into overlapping frames by sliding a fixed-length window along a specified axis. It is commonly used for audio or time-series analysis.

**Key Parameters:**
- `signal`: Input tensor (rank ≥ 1).
- `frame_length`: Length of each frame (in samples).
- `frame_step`: Step size between frame starts.
- `pad_end`: If `True`, pads the end of the signal with `pad_value` to include incomplete final frames.
- `axis`: Axis along which to frame (defaults to last axis).

**Output:**  
A tensor where the specified axis is replaced with two dimensions: `[..., num_frames, frame_length, ...]`.

**Example:**
```python
audio = tf.random.normal([3, 9152])
frames = tf.signal.frame(audio, 512, 180, pad_end=True)
# Result shape: [3, 51, 512] (3 batches, 51 frames of length 512)
```

**Use Case:**  
Primarily applied in audio preprocessing to split continuous signals into fixed-length segments for tasks like spectrogram extraction or feature framing.
