# First, here is a general summary of the API's functionality and usage:
---GENERAL SUMMARY START---
`tf.saved_model.save` exports a TensorFlow model (any `tf.Module` or trackable object) to the SavedModel format for deployment or sharing. It serializes the model's architecture, variables, and computation graphs.

**Key features:**
- **Signatures:** Define input/output specifications for serving. Can be a single `@tf.function` with input signature, a concrete function, or a dictionary mapping keys to functions.
- **Automatic signature detection:** If no signature is provided, it exports all traced `@tf.function` methods.
- **Variable handling:** Variables must be assigned as attributes of tracked objects (automatically done by Keras layers/optimizers).
- **Cross-device compatibility:** Generally works across devices (e.g., GPU-trained models served on CPU), except for hard-coded device annotations or device-specific ops.

**Basic usage:**
```python
tf.saved_model.save(obj, export_dir, signatures=None, options=None)
```

**Example:**
```python
class Adder(tf.Module):
    @tf.function(input_signature=[tf.TensorSpec(shape=None, dtype=tf.float32)])
    def add(self, x):
        return x + x

model = Adder()
tf.saved_model.save(model, '/tmp/adder')  # Exports with signature "add"
```

**Notes:**
- The SavedModel can be loaded with `tf.saved_model.load`.
- Keras models can be exported, with or without explicit signatures.
- Not recommended for use inside TensorFlow 1.x graph mode loops.
---GENERAL SUMMARY END---
