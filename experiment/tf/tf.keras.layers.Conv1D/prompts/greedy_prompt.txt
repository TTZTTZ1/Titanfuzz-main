The `tf.keras.layers.Conv1D` layer performs 1D convolution, primarily used for temporal or sequential data. It applies a convolution kernel across a single spatial/temporal dimension to extract features, optionally adding bias and applying an activation function.

**Key Parameters:**
- `filters`: Number of output channels.
- `kernel_size`: Size of the convolution window.
- `strides`: Step size for the convolution (incompatible with `dilation_rate > 1`).
- `padding`: `"valid"` (no padding), `"same"` (output size matches input when strides=1), or `"causal"` (for temporal order preservation).
- `data_format`: `"channels_last"` (default, shape: `[batch, steps, channels]`) or `"channels_first"` (shape: `[batch, channels, steps]`).
- `dilation_rate`: Controls kernel spacing for dilated convolutions.
- `groups`: Splits input channels into groups for grouped convolution.
- `activation`, `use_bias`, initializers, regularizers, and constraints are also configurable.

**Input/Output Shapes:**
- **Input:** 3D tensor with shape depending on `data_format`.
- **Output:** 3D tensor with `steps` dimension adjusted based on `padding` and `strides`, and channels replaced by `filters`.

**Example:**
```python
x = np.random.rand(4, 10, 128)  # 4 samples, 10 timesteps, 128 features
y = Conv1D(32, 3, activation='relu')(x)  # Output shape: (4, 8, 32)
```

**Use Cases:** Time-series forecasting, text processing, audio signal modeling (e.g., WaveNet with `padding="causal"`).
