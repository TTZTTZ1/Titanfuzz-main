`tf.keras.layers.Conv2D` is a 2D convolution layer used for processing spatial data like images. It applies a convolution kernel to the input, producing a tensor of outputs, optionally adds a bias, and applies an activation function.

**Key Parameters:**
- **`filters`**: Number of output filters (dimensionality of the output space).
- **`kernel_size`**: Size of the convolution window (int or tuple of 2 ints).
- **`strides`**: Stride length of the convolution (default: `(1, 1)`).
- **`padding`**: `"valid"` (no padding) or `"same"` (padding to maintain input size when `strides=1`).
- **`data_format`**: `"channels_last"` (default, shape: `(batch, height, width, channels)`) or `"channels_first"` (shape: `(batch, channels, height, width)`).
- **`activation`**: Activation function (e.g., `'relu'`); defaults to `None`.
- **`use_bias`**: Boolean to add a bias vector (default: `True`).
- **`dilation_rate`**: Dilation rate for dilated convolution (default: `(1, 1)`).
- **`groups`**: Number of groups for grouped convolution (input and filters must be divisible by groups).

**Input/Output Shapes:**
- **Input**: 4D tensor with shape depending on `data_format`.
- **Output**: 4D tensor with spatial dimensions adjusted based on `kernel_size`, `strides`, and `padding`, and channel dimension equal to `filters`.

**Example:**
```python
x = np.random.rand(4, 10, 10, 128)
y = keras.layers.Conv2D(32, 3, activation='relu')(x)  # Output shape: (4, 8, 8, 32)
```

**Notes:**
- Raises `ValueError` if both `strides > 1` and `dilation_rate > 1`.
- Supports optional kernel/bias regularizers, constraints, and initializers.
- Commonly used in CNNs for tasks like image classification, object detection, and generative modeling.
