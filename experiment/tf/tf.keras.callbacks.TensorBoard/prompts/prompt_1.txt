The `tf.keras.callbacks.TensorBoard` callback enables TensorFlow's TensorBoard visualization tool for Keras models. It logs training metrics, model graphs, weight histograms, and profiling data during model training and evaluation.

**Key Features:**
- **Metrics Logging:** Tracks loss and metrics at customizable intervals (epoch/batch level)
- **Visualizations:** Supports weight histograms, model graphs (currently unsupported), and embedding visualizations
- **Profiling:** Can profile specific batches to analyze performance
- **Flexible Logging:** Configurable update frequency (epoch, batch, or custom interval)

**Basic Usage:**
```python
tensorboard_callback = tf.keras.callbacks.TensorBoard(log_dir="./logs")
model.fit(x_train, y_train, epochs=2, callbacks=[tensorboard_callback])
```

**Key Parameters:**
- `log_dir`: Directory for TensorBoard log files
- `histogram_freq`: Frequency (in epochs) for weight histogram computation
- `update_freq`: Logging frequency ('epoch', 'batch', or integer)
- `profile_batch`: Batch(es) to profile for performance analysis
- `embeddings_freq`: Frequency for embedding layer visualization

**Visualization:** After training, launch TensorBoard from command line:
```bash
tensorboard --logdir=path_to_your_logs
```

The callback integrates seamlessly with Keras training workflows and supports both Functional API and subclassed models, including custom batch-level summaries via `tf.summary` operations.
