The `tf.keras.callbacks.TensorBoard` callback enables TensorBoard visualizations for Keras model training and evaluation. It logs key information including:

- **Metrics summary plots** (loss, accuracy, etc.)
- **Training graph visualization** (when `write_graph=True`)
- **Weight histograms** (frequency controlled by `histogram_freq`)
- **Sampled profiling** (via `profile_batch`)
- **Embedding visualizations** (via `embeddings_freq` and `embeddings_metadata`)

### Key Parameters:
- `log_dir`: Directory where TensorBoard log files are saved.
- `histogram_freq`: How often (in epochs) to log weight histograms (0 to disable).
- `update_freq`: Logging frequency (`"epoch"`, `"batch"`, or an integer number of batches).
- `write_steps_per_second`: Whether to log training speed.
- `profile_batch`: Batch or batch range to profile for performance insights.

### Basic Usage:
```python
tensorboard_callback = tf.keras.callbacks.TensorBoard(log_dir="./logs")
model.fit(x_train, y_train, epochs=5, callbacks=[tensorboard_callback])
```
After training, launch TensorBoard from the command line:
```bash
tensorboard --logdir=./logs
```

### Advanced Features:
- **Batch-level logging**: Use `update_freq=1` (or an integer) and add `tf.summary` calls in your model to log custom batch-level summaries.
- **Profiling**: Profile specific batches (e.g., `profile_batch=5` or `profile_batch=(10,20)`) to analyze performance.

This callback is essential for visualizing training progress, debugging, and optimizing model performance.
