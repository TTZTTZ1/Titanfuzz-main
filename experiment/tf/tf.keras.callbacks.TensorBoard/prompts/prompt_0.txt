The `tf.keras.callbacks.TensorBoard` callback enables TensorBoard visualizations for Keras model training and evaluation. It logs key information such as metrics, weight histograms, and computational graphs, which can be viewed in TensorFlow's TensorBoard tool.

### **Key Features:**
- **Logs Directory**: Saves logs to a specified `log_dir` for TensorBoard parsing.
- **Metrics Visualization**: Tracks training/evaluation metrics over time.
- **Histograms**: Logs weight distributions per layer at a specified `histogram_freq` (epoch-based).
- **Update Frequency**: Controls logging frequency via `update_freq` (batch, epoch, or custom interval).
- **Custom Summaries**: Supports batch-level logging via `tf.summary` within model code.
- **Embeddings Visualization**: Logs embedding layer data periodically if `embeddings_freq` > 0.

### **Basic Usage:**
```python
tensorboard_callback = tf.keras.callbacks.TensorBoard(log_dir="./logs")
model.fit(x_train, y_train, epochs=5, callbacks=[tensorboard_callback])
```
After training, launch TensorBoard from the command line:
```bash
tensorboard --logdir=./logs
```

### **Important Notes:**
- Requires TensorFlow installation.
- Avoid reusing the log directory across different runs.
- High logging frequency (e.g., batch-level) may slow training, especially in distributed setups.
- Some features like `write_graph` and `profile_batch` may not be fully supported in all TensorFlow versions.

This callback is essential for monitoring, debugging, and visualizing model performance during the machine learning workflow.
