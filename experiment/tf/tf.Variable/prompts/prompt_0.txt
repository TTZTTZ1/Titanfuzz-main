`tf.Variable` is a TensorFlow class used to create mutable tensors that hold persistent, shared state in a program. It is primarily employed to store trainable model parameters (like weights and biases) in machine learning models.

### Key Features:
- **Mutable State**: Unlike regular tensors, variables can be updated using methods like `assign()`, `assign_add()`, and `assign_sub()`.
- **Initialization**: Requires an initial value (a tensor or convertible object) that defines its shape and data type.
- **Trainable**: By default, variables are trainable, meaning they are automatically tracked by `tf.GradientTape` for gradient computation during training.
- **Integration with Modules**: Variables assigned as attributes of `tf.Module` subclasses are automatically tracked, facilitating checkpointing and model serialization.
- **Compatibility with `tf.function`**: Variables work seamlessly inside TensorFlow’s graph-based functions, though they must be created outside the function’s scope to ensure proper ownership.

### Common Use Cases:
- Storing and updating model parameters in neural networks.
- Maintaining persistent counters or state across training steps.
- Enabling automatic differentiation for optimization.

### Important Methods:
- **`assign(value)`**: Updates the variable with a new value.
- **`assign_add(delta)` / `assign_sub(delta)`**: Increments or decrements the variable by a given amount.
- **`read_value()`**: Retrieves the current value of the variable in the current context.
- **`value()`**: Returns a snapshot of the variable’s value (non-mutable).

### Example:
```python
import tensorflow as tf

# Create a trainable variable
weight = tf.Variable(1.0)
weight.assign(2.0)  # Update value
print(weight.numpy())  # Output: 2.0
```

Variables are foundational in TensorFlow for building and training models, providing a mechanism to store and update learnable parameters efficiently.
