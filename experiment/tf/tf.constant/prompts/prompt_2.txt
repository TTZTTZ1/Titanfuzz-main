`tf.constant` creates an immutable constant tensor from a given value, list, or array. It optionally allows specifying the data type (`dtype`) and reshaping the output (`shape`). Unlike `tf.Variable`, the resulting tensor is fixed and cannot be changed after creation.  

Key points:  
- If `dtype` is not provided, it is inferred from the input.  
- If `shape` is given, the input is reshaped accordingly (scalars are broadcast).  
- It embeds the value directly into the computation graph (unlike `tf.fill`, which creates a runtime operation).  
- It cannot accept symbolic tensors (e.g., TensorFlow 1.x placeholders) as input—use `tf.convert_to_tensor` instead for such cases.  
- Gradients can flow through `tf.constant` when used with eager tensors.
