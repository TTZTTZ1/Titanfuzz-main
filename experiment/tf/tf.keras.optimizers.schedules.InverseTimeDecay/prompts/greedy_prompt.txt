`InverseTimeDecay` is a TensorFlow/Keras learning rate schedule that reduces the learning rate inversely proportional to the training step. It's useful for gradually lowering the learning rate during training to improve convergence.

**Key Features:**
- **Formula:** `initial_learning_rate / (1 + decay_rate * step / decay_steps)` (continuous) or with `floor(step / decay_steps)` if `staircase=True`
- **Purpose:** Automatically decays the learning rate as training progresses
- **Usage:** Pass directly to any Keras optimizer as the `learning_rate` argument

**Parameters:**
- `initial_learning_rate`: Starting learning rate (float)
- `decay_steps`: Frequency of decay application
- `decay_rate`: Decay rate (number)
- `staircase`: If `True`, applies decay in discrete steps instead of continuously
- `name`: Optional operation name

**Example:**
```python
lr_schedule = InverseTimeDecay(
    initial_learning_rate=0.1,
    decay_steps=1.0,
    decay_rate=0.5
)
optimizer = SGD(learning_rate=lr_schedule)
```

The schedule is callable, taking the current step as input and returning the decayed learning rate. It also supports serialization via `get_config()` and `from_config()`.
