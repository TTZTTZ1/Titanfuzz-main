**Summary: `tf.keras.optimizers.schedules.InverseTimeDecay`**

This is a learning rate schedule that decays the learning rate inversely proportional to the training step. It gradually reduces the learning rate during training, which can help improve convergence and model performance.

**Key Features:**
- **Decay Formula:**  
  Continuous: `initial_learning_rate / (1 + decay_rate * step / decay_steps)`  
  Staircase (if `staircase=True`): Uses `floor(step / decay_steps)` for discrete decay steps.
- **Purpose:** Automatically lowers the learning rate as training progresses, avoiding large updates later in training.
- **Usage:** Pass the schedule directly to an optimizer's `learning_rate` argument.

**Parameters:**
- `initial_learning_rate`: Starting learning rate (float).
- `decay_steps`: Frequency of decay application.
- `decay_rate`: Decay rate (float).
- `staircase`: If `True`, decay occurs at discrete intervals; otherwise, continuous.
- `name`: Optional operation name.

**Returns:**  
A callable that takes the current optimizer `step` and returns the decayed learning rate.

**Example:**
```python
lr_schedule = InverseTimeDecay(
    initial_learning_rate=0.1,
    decay_steps=1.0,
    decay_rate=0.5
)
optimizer = tf.keras.optimizers.SGD(learning_rate=lr_schedule)
```
