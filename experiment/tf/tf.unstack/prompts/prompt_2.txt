`tf.unstack` splits a tensor along a specified axis into a list of smaller tensors, each with one dimension removed. It is the inverse operation of `tf.stack`.

**Key features:**
- **Input:** A rank‑R tensor (R > 0).
- **Output:** A list of rank‑(R‑1) tensors. The number of tensors equals the size of the original tensor along the chosen axis.
- **Parameters:**
  - `value`: The tensor to split.
  - `axis`: The dimension to unpack (default 0). Supports negative indexing.
  - `num`: Optional integer specifying the length of the axis; required if the axis length is unknown (e.g., inside a `@tf.function` with dynamic shapes).
  - `name`: Optional operation name.

**Important notes:**
- In eager execution, Python iterable unpacking (e.g., `a, b, c = t`) works for the 0th axis, but inside a `@tf.function` you must use `tf.unstack` because symbolic tensors are not iterable.
- If the axis length is unknown in graph mode, you must provide `num` explicitly; otherwise, use a `tf.TensorArray` with a loop for dynamic splitting.

**Example:**
```python
x = tf.reshape(tf.range(12), (3, 4))   # shape (3, 4)
a, b, c = tf.unstack(x)                # 3 tensors, each shape (4,)
```
